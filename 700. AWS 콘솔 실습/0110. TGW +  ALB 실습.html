단일 VPC 내 TGW 기반 중앙 집중형 네트워크 구성 가이드

단일 VPC(Virtual Private Cloud, 가상 사설 클라우드) 내에서 TGW(Transit Gateway, 전송 게이트웨이)를 거쳐 외부 접속(Inbound)과 인터넷 출구(Outbound)를 동시에 구현하는 구성입니다.

그림 1. 단일 VPC 내 중앙 집중형 트래픽 라우팅 개요

1. 주요 구성 요소 및 메뉴 설정 (GUI)

※ AWS Management Console(아마존 웹 서비스 관리 콘솔)의 GUI(Graphical User Interface, 그래픽 사용자 인터페이스)를 기준으로 설명합니다.

1. TGW 생성 및 연결:

  • 메뉴: VPC 콘솔 > Transit Gateways > Create Transit Gateway
  • 연결: Transit Gateway Attachments > Create Transit Gateway Attachment
  • 핵심 설정: Appliance Mode(어플라이언스 모드)1를 반드시 'Enable(활성화)'로 체크 (비대칭 라우팅 방지).
⚡ 실전 팁 & 자주 실수하는 부분

Appliance Mode(어플라이언스 모드)를 활성화하지 않으면, 트래픽이 들어온 AZ(Availability Zone, 가용 영역)와 나가는 AZ가 달라집니다. 이 경우 상태 저장(Stateful) 방화벽 등에서 세션을 인식하지 못해 트래픽이 차단되는 비대칭 라우팅(Asymmetric Routing) 이슈가 발생하므로 반드시 활성화해야 합니다.

2. ALB(Application Load Balancer) 생성:

  • 메뉴: EC2 콘솔 > Load Balancers > Create Load Balancer
  • 유형: Internet-facing(인터넷 연결형) 선택.
  • 네트워크: Public Subnet(퍼블릭 서브넷, 공개 부분망) 선택 및 프라이빗 서버 IP(Internet Protocol, 인터넷 규약)를 타겟 그룹으로 지정.

TGW 기반 라우팅 흐름 (Routing Flow)

프라이빗 서브넷 라우팅 테이블
(Private Subnet RTb)

대상(Destination)타겟(Target)
10.0.0.0/16local (VPC 내부)
0.0.0.0/0tgw-123ab... (TGW로 전달)

전송 게이트웨이 라우팅 테이블
(Transit Gateway Route Table)

대상(Destination)타겟(Attachment)
0.0.0.0/0vpc-attach-public (NAT 방향)
10.0.0.0/16vpc-attach-private (서버 방향)

그림 2. 라우팅 테이블(RTb)을 통한 서브넷 간 트래픽 제어

2. 라우팅 테이블(RTb) 상세 설정

(이 섹션에서는 각 서브넷에 연결된 RTb(Routing Table, 라우팅 테이블)의 대상(Destination)과 타겟(Target)을 정의하여 TGW(Transit Gateway, 전송 게이트웨이) ENI(Elastic Network Interface, 탄력적 네트워크 인터페이스)로 트래픽을 정확히 유도해야 합니다.)

3. 구성 확인 및 테스트 방법

1. 아웃바운드(내부 → 외부):

SSM(Systems Manager, 시스템 관리자) 접속 후 curl -I https://www.google.com 실행 → 200 OK 확인.

2. 인바운드(외부 → 내부):

로컬 브라우저에서 ALB(Application Load Balancer, 애플리케이션 로드 밸런서)의 DNS Name(Domain Name System Name, 도메인 이름) 접속 → 웹 페이지 표시 확인.

전문 용어 설명

  • • Inbound(인바운드): 외부에서 내부로 들어오는 트래픽.
  • • Outbound(아웃바운드): 내부에서 외부로 나가는 트래픽.
  • • Appliance Mode(어플라이언스 모드): 트래픽의 송수신 경로를 동일 AZ(Availability Zone, 가용 영역)로 고정하는 기능.
  • • DNS Name(Domain Name System Name, 도메인 이름): IP(Internet Protocol, 인터넷 주소) 대신 사용하는 주소.
[과금 주의 - 유료 서비스]

TGW(Transit Gateway, 전송 게이트웨이)는 시간당 연결 비용과 GB(Gigabyte, 기가바이트)당 데이터 처리 비용이 발생하므로 NFW(Network Firewall, 네트워크 방화벽) 도입 등 중앙 통제 목적일 때 권장함.
출처: AWS Transit Gateway 요금 가이드

[주석 1] Appliance Mode(어플라이언스 모드): AWS(Amazon Web Services, 아마존 웹 서비스) Transit Gateway에서 보안 어플라이언스(방화벽, IPS 등)가 트래픽의 양방향(Inbound/Outbound) 흐름을 정상적으로 검사할 수 있도록, 트래픽이 항상 동일한 가용 영역(AZ)을 통해 라우팅되도록 보장하는 심화 기능입니다. 이를 통해 비대칭 라우팅으로 인한 패킷 드롭(Packet Drop)을 방지합니다.
