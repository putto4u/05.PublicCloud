개발 생산성을 극대화하는
3대 외부 연동 시스템

Visual Studio Code(비주얼 스튜디오 코드)는 단순한 편집기를 넘어, 클라우드 인프라와 버전 관리 시스템을 하나로 묶는 허브 역할을 합니다. 본 가이드는 개발 현장에서 가장 많이 사용하는 연동 기술을 단계별로 안내합니다.

코드 관리
클라우드 배포
인프라 코딩

IaC(Infrastructure as Code, 코드형 인프라)[^1] 도구 시장 점유율

GitHub (깃허브) 연동 과정

Version Control System (버전 관리 시스템)[^2] 연동을 통한 협업 환경 구축

1

Git 엔진 설치 및 환경 변수 설정

로컬 PC에 Git(깃) 엔진이 설치되어 있어야 합니다. 공식 사이트에서 설치 후 터미널에 명령어를 입력하여 확인합니다.

git --version
2

VS Code 계정 인증 (OAuth)

왼쪽 하단 사람 모양 아이콘을 클릭하고 'Sign in with GitHub'을 선택합니다. 웹 브라우저에서 승인(Authorize) 버튼을 눌러 연동을 완료합니다.

3

Repository (저장소)[^3] 연결 및 Commit/Push

Source Control (소스 제어) 탭에서 'Publish to GitHub' 또는 'Clone Repository'를 통해 프로젝트를 동기화합니다.

브랜치 전략 사용 통계

AWS (아마존 웹 서비스) 연동 과정

Cloud Computing (클라우드 컴퓨팅)[^4] 리소스의 직접 관리 및 모니터링

1

AWS Toolkit 확장 프로그램 설치

Extensions(확장 프로그램) 마켓플레이스에서 'AWS Toolkit'을 검색하여 설치합니다.

[Image: AWS Toolkit 설치 화면]
2

IAM(자격 증명) 생성 및 키 발급

AWS 콘솔에서 IAM(Identity and Access Management)[^5] 사용자를 생성하고 Access Key(액세스 키)와 Secret Key(비밀 키)를 발급받습니다.

3

자격 증명 연결 (Connect to AWS)

AWS 아이콘 클릭 후 'Add New Connection'을 통해 발급받은 키를 입력하여 연결을 마무리합니다.

실무 과금 및 보안 주의사항

  • 유료 서비스 경보: 연동 후 EC2(가상 서버)나 RDS(데이터베이스) 생성 시 시간당 비용이 발생합니다.
  • 보안 경고: 발급받은 액세스 키가 GitHub에 노출되면 해킹 및 요금 폭탄의 주원인이 됩니다. .gitignore[^6] 설정이 필수입니다.
  • 프리 티어(Free Tier)[^7] 체크: 무료 할당량을 초과하면 즉시 유료로 전환되므로 Billing Dashboard(결제 대시보드)를 자주 확인하세요.

Terraform (테라폼) 연동 과정

Infrastructure as Code (코드형 인프라)[^1] 자동화 도구 연동

1

Terraform Binary(이진 파일) 설치

Terraform 공식 사이트에서 OS에 맞는 파일을 다운로드하여 환경 변수(Path)에 추가합니다.

2

HashiCorp Terraform 확장 설치

VS Code에서 HCL(HashiCorp Configuration Language)[^8] 문법 강조 및 자동 완성을 위해 전용 확장 프로그램을 설치합니다.

실무 예제: main.tf (AWS 리소스 생성 코드) 

provider "aws" {
  region = "ap-northeast-2" # 서울 리전
}

resource "aws_s3_bucket" "example" {
  bucket = "my-terraform-bucket-2024"
}
terraform init

공급자(Provider) 플러그인 초기화

terraform plan

변경 사항 미리보기 (실행 계획 확인)

terraform apply

전문가의 조언

이 모든 도구를 연동하는 목적은 '자동화'와 '안정성'입니다. 코드로 인프라를 정의(Terraform)하고, 버전으로 관리(GitHub)하며, 클라우드(AWS)에 배포하는 방식은 현대 개발자의 필수 역량입니다. 연동이 완료된 후에는 각 도구의 에러 메시지를 읽는 습관을 들여보세요.

주석 및 용어 설명

[^1] IaC (Infrastructure as Code): 서버, 네트워크 등 인프라를 수동 설정이 아닌 코드로 정의하고 관리하는 방식.

[^2] VCS (Version Control System): 파일의 변경 내역을 기록하여 특정 시점으로 되돌리거나 비교할 수 있는 시스템.

[^3] Repository (저장소): 프로젝트의 모든 파일과 변경 이력이 저장되는 디지털 보관소.

[^4] Cloud Computing: 인터넷을 통해 서버, 저장소, DB 등의 컴퓨팅 자원을 빌려 쓰는 서비스.

[^5] IAM (Identity and Access Management): AWS 리소스에 대한 개별 사용자의 접근 권한을 안전하게 관리하는 서비스.

[^6] .gitignore: Git 관리에서 제외할 파일이나 폴더를 지정하는 설정 파일 (보안 키 누출 방지용).

[^7] Free Tier: 신규 가입자에게 일정 기간/사양 내에서 서비스를 무료로 제공하는 정책.

[^8] HCL (HashiCorp Configuration Language): Terraform에서 사용하는 독자적인 인프라 정의 언어.
