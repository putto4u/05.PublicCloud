## 2일차: 온프레미스 중심 글로벌 네트워크 설계 및 하이브리드 대응 구축 (Project: awskr01)

오늘의 목표는 온프레미스 대역(**10.10.0.0/16**)과 충돌하지 않도록 AWS 글로벌 인프라 대역을 **10.20.0.0/16**부터 새롭게 할당하고, 향후 데이터베이스를 온프레미스로 이전하거나 복제할 때를 대비한 보안 및 네트워크 구조를 미리 반영하는 것입니다.

---

### 1. 글로벌 IP 할당 및 서브넷 분할 전략

온프레미스와 AWS 각 리전 간의 VPN/Direct Connect(전용선) 연결 시 IP 충돌을 방지하기 위해 대역을 명확히 분리합니다.

| 위치 (Role) | 전체 네트워크 대역 (CIDR) | 비고 |
| --- | --- | --- |
| **온프레미스 (On-prem)** | **10.10.0.0/16** | **(기준)** 현재 사무실 및 IDC 대역 |
| **한국 리전 (apne2)** | **10.20.0.0/16** | 글로벌 허브 및 첫 번째 스포크 위치 |
| **미국 리전 (use1)** | **10.30.0.0/16** | 향후 확장 예정인 미국 동부 스포크 |

#### [세부 설계] 한국 리전 스포크 내부 서브넷 (10.20.0.0/16)

| 서브넷 용도 | 가용 영역(AZ) A | 가용 영역(AZ) C | 비고 |
| --- | --- | --- | --- |
| **Public** | 10.20.1.0/24 | 10.20.2.0/24 | ALB, NAT Gateway 위치 |
| **Private EKS** | 10.20.10.0/24 | 10.20.11.0/24 | 워커 노드 및 애플리케이션 Pod |
| **Private DB** | **10.20.100.0/24** | **10.20.101.0/24** | RDS Master 및 Read Replica |

### 2. 모듈 작성: 하이브리드 대응형 스포크 네트워크 (Terraform)

**파일 1: `infrastructure/modules/aws-network-spoke/variables.tf**`
온프레미스(10.10)를 1순위로 두고, AWS 대역을 10.20으로 시작하도록 설정합니다.

```terraform
variable "vpc_cidr" {
  description = "VPC의 IP 대역"
  type        = string
  default     = "10.20.0.0/16"
}

variable "enable_nat" {
  description = "NAT Gateway 생성 여부 (허브 연결 시 false로 변경)"
  type        = bool
  default     = true
}

variable "public_subnets" {
  type    = list(string)
  default = ["10.20.1.0/24", "10.20.2.0/24"]
}

variable "private_subnets" {
  type    = list(string)
  default = ["10.20.10.0/24", "10.20.11.0/24"]
}

variable "database_subnets" {
  type    = list(string)
  default = ["10.20.100.0/24", "10.20.101.0/24"]
}

# 온프레미스 이전을 위한 IDC 대역 변수 추가
variable "onprem_cidr" {
  description = "온프레미스 IDC 네트워크 대역"
  type        = string
  default     = "10.10.0.0/16"
}

```

**파일 2: `infrastructure/modules/aws-network-spoke/main.tf**`
DB 이전을 고려한 보안 그룹(Security Group)과 표준 VPC 로직을 통합합니다.

```terraform
# 1. 표준 VPC 구성
module "vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "5.0.0"

  name = "awskr01-spoke-vpc"
  cidr = var.vpc_cidr

  azs              = ["ap-northeast-2a", "ap-northeast-2c"]
  public_subnets   = var.public_subnets
  private_subnets  = var.private_subnets
  database_subnets = var.database_subnets

  create_database_subnet_group = true
  enable_nat_gateway           = var.enable_nat
  single_nat_gateway           = true

  tags = { Project = "awskr01", Terraform = "true", Environment = "spoke" }
}

# 2. [추가] DB 이전을 위한 하이브리드 보안 그룹
# 향후 온프레미스(10.10.x.x)로의 복제 및 이관을 위해 미리 정의
resource "aws_security_group" "db_migration_sg" {
  name        = "awskr01-db-migration-sg"
  description = "Allow DB replication and migration traffic from On-premises"
  vpc_id      = module.vpc.vpc_id

  # 온프레미스 IDC 대역에서 DB 포트(MySQL 3306) 접근 허용
  ingress {
    from_port   = 3306
    to_port     = 3306
    protocol    = "tcp"
    cidr_blocks = [var.onprem_cidr]
    description = "Replication traffic from IDC"
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = { Name = "awskr01-db-migration-sg" }
}

```
