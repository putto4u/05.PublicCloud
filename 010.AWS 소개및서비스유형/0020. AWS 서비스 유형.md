AWS는 사용자가 인프라를 직접 관리하는 수준에 따라 크게 세 가지 서비스 유형으로 분류됩니다. 이를 **클라우드 서비스 모델**이라고 하며, 각 모델에 따라 고객과 AWS가 책임지는 영역이 달라집니다.

---

## 1. 클라우드 서비스 모델의 3단계

클라우드 서비스는 관리의 편의성과 자유도에 따라 **IaaS, PaaS, SaaS**로 나뉩니다.

### **① IaaS (Infrastructure as a Service)**

**가상화된 물리적 자원**을 제공하는 서비스입니다. OS를 직접 설치하고 네트워크, 방화벽, 스토리지를 직접 설정해야 합니다.

* **특징:** 자유도가장 높음, 관리 부담 가장 큼.
* **대표 서비스:** **Amazon EC2** (가상 서버), **Amazon VPC** (네트워크), **Amazon EBS** (저장소).
* **비유:** 땅을 빌려 내가 직접 설계한 집을 짓는 것.

### **② PaaS (Platform as a Service)**

애플리케이션 개발에 필요한 **플랫폼과 환경**을 제공합니다. 개발자는 서버 사양이나 OS 업데이트를 신경 쓰지 않고 코드만 배포하면 됩니다.

* **특징:** 개발에 집중 가능, 인프라 제어권은 낮음.
* **대표 서비스:** **AWS Elastic Beanstalk**, **AWS Lambda** (Serverless).
* **비유:** 빌트인 가전과 가구가 완비된 집을 빌려 생활만 하는 것.

### **③ SaaS (Software as a Service)**

인터넷을 통해 **완성된 소프트웨어**를 제공받는 방식입니다. 모든 인프라와 소프트웨어 업데이트는 공급업체가 관리합니다.

* **특징:** 즉시 사용 가능, 커스터마이징이 거의 불가능.
* **대표 서비스:** **Amazon Chime** (화상 회의), **AWS Marketplace**에서 제공되는 타사 소프트웨어.
* **비유:** 호텔이나 에어비앤비를 예약하여 몸만 가서 머무는 것.

---

## 2. 관리 범위 비교 (Shared Responsibility)

각 서비스 유형별로 사용자가 직접 관리해야 하는 영역(파란색)과 AWS가 관리해 주는 영역(주황색)을 비교하면 다음과 같습니다.

| 관리 항목 | On-Premise (직접 운영) | IaaS (EC2 등) | PaaS (Beanstalk 등) | SaaS (Chime 등) |
| --- | --- | --- | --- | --- |
| **애플리케이션/데이터** | 사용자 | **사용자** | **사용자** | AWS |
| **런타임 / 미들웨어** | 사용자 | **사용자** | AWS | AWS |
| **운영 체제 (OS)** | 사용자 | **사용자** | AWS | AWS |
| **가상화 / 서버 하드웨어** | 사용자 | AWS | AWS | AWS |
| **스토리지 / 네트워킹** | 사용자 | AWS | AWS | AWS |

---

## 3. 서버리스(Serverless): 차세대 서비스 유형

최근에는 PaaS를 넘어 **Serverless** 모델이 각광받고 있습니다. 서버의 존재 자체를 신경 쓸 필요 없이 이벤트가 발생할 때만 코드가 실행되는 방식입니다.

* **핵심 서비스:** **AWS Lambda**, **Amazon Fargate**.
* **장점:** 코드가 실행된 시간만큼만 비용이 발생하여 경제적이며, 트래픽에 따라 자동으로 무한 확장됩니다.

---

## 4. 실전 팁: 상황별 서비스 선택 가이드

1. **세밀한 튜닝이 필요한 경우:** 커스텀 커널 설정이나 특정 보안 패키지 설치가 필수라면 **EC2 (IaaS)**를 선택하세요.
2. **빠른 시장 출시가 우선인 경우:** 인프라 관리 인력이 부족하다면 **Elastic Beanstalk (PaaS)**이나 **Lambda**를 사용하세요.
3. **비용 최적화가 중요한 경우:** 상시 가동되지 않는 배치 작업 등은 **Serverless**가 가장 저렴합니다.

---

> [!WARNING] **과금 및 유료 전환 주의**
> * **EC2 (IaaS):** 인스턴스를 중지하지 않고 켜두면 시간당 비용이 계속 발생합니다. 프리 티어 범위를 넘기지 않도록 주의하세요.
> * **Lambda (Serverless):** 호출 횟수와 실행 시간에 따라 과금되지만, 무한 루프 코드를 작성할 경우 예기치 못한 막대한 비용이 청구될 수 있습니다.
