Why Cloud Computing?

On-Premise (기존)

  • 높은 초기 구축 비용 (CAPEX)
  • 하드웨어 유지보수 부담
  • 확장의 어려움 (수동 증설)
  • 데이터센터 물리적 보안 필요

AWS Cloud

  • 사용한 만큼만 지불 (OPEX)
  • 즉각적인 인프라 생성
  • 무한에 가까운 탄력적 확장
  • 글로벌 배포 용이

Core Services Ecosystem

Amazon EC2

크기 조절이 가능한 클라우드 상의 가상 서버

AWS Lambda

서버 관리 없이 코드를 실행하는 이벤트 중심 컴퓨팅

Amazon S3

확장성, 데이터 가용성 및 보안을 제공하는 객체 스토리지

Amazon VPC

AWS 클라우드 내 논리적으로 격리된 가상 네트워크

* 주의: 각 서비스는 사용량(시간, 용량, 요청 수)에 따라 별도의 요금이 부과됩니다.

Security & Compliance

책임 공유 모델

AWS의 책임 (Security 'OF' the Cloud)

물리적 시설, 하드웨어, 네트워크, 하이퍼바이저 등 글로벌 인프라 자체의 보안

고객의 책임 (Security 'IN' the Cloud)

데이터 암호화, OS 패치, 방화벽 설정, 계정 및 권한 관리 (IAM)
