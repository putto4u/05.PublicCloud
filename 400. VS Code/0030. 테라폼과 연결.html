IDE INTEGRATION STRATEGY

VS Code와 테라폼 연동 가이드

효율적인 글로벌 네트워크 자산 관리를 위한
IaC (Infrastructure as Code, 코드형 인프라) 개발 환경 최적화

이 과정의 주요 학습 포인트

  • 1. IDE (Integrated Development Environment, 통합 개발 환경) 설정
  • 2. 테라폼 바이너리 설치 및 시스템 환경 변수 구성
  • 3. HCL (HashiCorp Configuration Language, 하시코프 설정 언어) 최적 확장 도구 활용
01

01. CORE CONCEPTS

도구의 역할과 필요성

수작업으로 리소스를 생성하는 대신 코드로 관리함으로써 인적 실수를 방지하고 동일한 환경을 미국, 유럽 리전에 복제할 수 있습니다.

VS Code

가장 강력한 오픈소스 코드 편집기로, 테라폼 코드 작성 시 실시간 문법 검사 및 자동 완성을 지원합니다.

Terraform

작성된 코드를 읽어 실제 AWS 리소스(VPC, TGW 등)로 변환하고 상태를 추적하는 엔진 역할을 합니다.

연동의 핵심: Terraform 확장 프로그램

VS Code 자체는 테라폼 언어를 이해하지 못합니다. HashiCorp에서 공식 제공하는 확장 프로그램을 통해 **LSP (Language Server Protocol, 언어 서버 프로토콜)** 기능을 활성화해야 합니다.

* IaC (Infrastructure as Code): 하드웨어나 클라우드 리소스를 물리적 설정 대신 코드를 통해 관리하고 프로비저닝(Provisioning)하는 방식입니다.

02. TERRAFORM INSTALLATION

테라폼 엔진(CLI) 설치

편집기와 연동하기 전, 운영체제(OS)에서 테라폼 명령어를 실행할 수 있는 바이너리가 설치되어 있어야 합니다.

1
패키지 관리자를 이용한 설치

수동 다운로드보다 업데이트가 용이한 패키지 관리자 도구 사용을 권장합니다.

# macOS (Homebrew 사용 시) brew tap hashicorp/tap brew install hashicorp/tap/terraform # Windows (Chocolatey 사용 시) choco install terraform

2
설치 확인 및 버전 체크

terraform -version

03. VS CODE EXTENSION

공식 확장 프로그램 설치

VS Code 마켓플레이스에서 **HashiCorp Terraform** 확장 프로그램을 설치하여 지능형 편집 기능을 확보합니다.

주요 지원 기능

  • Syntax Highlighting (구문 강조): 코드의 가독성을 높여주는 색상 표시
  • IntelliSense (지능형 자동 완성): 리소스 타입 및 속성 자동 추천
  • Validation (코드 검증): 저장 시 문법 오류 실시간 탐지
04

04. AUTHENTICATION SETUP

AWS 계정 연동 (인증 구성)

테라폼이 개발자의 권한으로 AWS에 명령을 내릴 수 있도록 **Credentials (자격 증명)**를 설정해야 합니다.

중요 보안 주의사항: Access Key(액세스 키)와 Secret Key(비밀 키)는 절대 코드 파일에 직접 작성하지 마십시오. GitHub 등에 노출될 경우 심각한 무단 과금이 발생할 수 있습니다.

추천 방식: AWS CLI 프로파일 설정

# 터미널에서 실행 aws configure --profile my-terraform-user # 입력 요구 사항 AWS Access Key ID [None]: AKIA... AWS Secret Access Key [None]: wJalr... Default region name [None]: ap-northeast-2

인증 정보의 위치

Windows: C:\Users\사용자\.aws\credentials
Linux/macOS: ~/.aws/credentials

05. PROJECT STRUCTURE

테라폼 프로젝트 파일 구성

효율적인 관리를 위해 기능을 분리하여 파일을 구성하는 것이 관례입니다.

HCL (HashiCorp Configuration Language) 주석

HCL (HashiCorp Configuration Language): 사람이 읽기 쉬운 형태의 구성 언어로, JSON과 호환되지만 클라우드 인프라의 복잡한 계층 구조를 표현하는 데 최적화되어 있습니다.

06

06. PROVIDER DEFINITION

AWS 공급자(Provider) 설정

`main.tf` 상단에 어떤 클라우드를 사용할지 명시합니다. 이때 앞서 설정한 AWS 프로파일과 연결합니다.

# provider.tf 또는 main.tf 상단 terraform { required_providers { aws = { source = "hashicorp/aws" version = "~> 5.0" # 최신 버전 유지 권장 } } } provider "aws" { region = "ap-northeast-2" # 서울 리전 profile = "my-terraform-user" # AWS CLI 설정 프로파일 }
* Provider (공급자): 테라폼 엔진이 각 클라우드 API(AWS, Azure, GCP 등)와 통신할 수 있게 해주는 전용 플러그인입니다.
07

07. RESOURCE PROVISIONING

스포크 VPC 리소스 정의

앞선 교재에서 설계한 **KR-SPOKE-01** VPC를 코드로 구현합니다.

# Spoke VPC 생성 resource "aws_vpc" "spoke_01" { cidr_block = "10.10.16.0/20" enable_dns_hostnames = true tags = { Name = "KR-SPOKE-01" Env = "Production" } } # Private Subnet 생성 resource "aws_subnet" "pri_2a" { vpc_id = aws_vpc.spoke_01.id cidr_block = "10.10.16.0/24" availability_zone = "ap-northeast-2a" tags = { Name = "KR-SPOKE-01-PRI-2A" } }

08. WORKFLOW: INIT

테라폼 초기화 (Initialize)

작성된 코드를 실행하기 위해 필요한 플러그인(AWS Provider 등)을 내려받는 단계입니다.

$ terraform init # 결과 메시지 Initializing the backend... Initializing provider plugins... - Installing hashicorp/aws v5.x.x... Terraform has been successfully initialized!
09

09. WORKFLOW: PLAN

변경 사항 검토 (Plan)

실제 AWS에 리소스를 만들기 전, 어떤 리소스가 생성/수정/삭제될지 미리 확인하는 가장 중요한 단계입니다.

$ terraform plan # 결과 출력 (Dry Run) Terraform will perform the following actions: # aws_vpc.spoke_01 will be created + resource "aws_vpc" "spoke_01" { + cidr_block = "10.10.16.0/20" ... } Plan: 2 to add, 0 to change, 0 to destroy.

Plan의 중요성

실제 환경을 건드리지 않는 **Dry Run (시뮬레이션)** 과정입니다. 의도치 않은 삭제 작업(`- destroy`)이 포함되어 있는지 반드시 눈으로 확인해야 합니다.

10

10. WORKFLOW: APPLY

리소스 생성 실행 (Apply)

검토가 끝난 코드를 실제로 AWS API에 요청하여 리소스를 생성합니다.

$ terraform apply # 최종 승인 대기 Do you want to perform these actions? Enter a value: yes # 결과 aws_vpc.spoke_01: Creating... aws_vpc.spoke_01: Creation complete after 5s Apply complete! Resources: 2 added, 0 changed, 0 destroyed.
유료 리소스 시작: 이 명령을 수행하는 순간부터 생성된 리소스(NAT Gateway, Transit Gateway 등)에 대한 과금이 시작됩니다. VPC 자체 생성은 무료이나 부수적인 서비스는 유료임을 기억하십시오.
11

11. MANAGING STATE

상태 추적 (Terraform State)

테라폼은 `terraform.tfstate`라는 파일을 생성하여 실제 인프라의 현재 상태를 JSON 형식으로 기록합니다.

**주의:** `.tfstate` 파일은 테라폼의 뇌와 같습니다. 수동으로 편집하거나 삭제할 경우 실제 인프라와 동기화가 깨져 큰 장애를 초래할 수 있습니다.
12

12. CODE FORMATTING

코드 스타일 자동 정렬

팀 협업 시 코드 스타일을 통일하기 위해 VS Code의 **Format on Save (저장 시 자동 정렬)** 기능을 활성화합니다.

설정 방법 (VS Code)

1. `Ctrl + ,` (설정) 열기

2. "Format on Save" 검색 후 체크

3. "Terraform: Format" 옵션 확인

수동 정렬 명령어

$ terraform fmt

이 명령어는 지저분하게 작성된 코드의 인덴트(들여쓰기)를 테라폼 표준 스타일로 교정해 줍니다.

13. TROUBLE SHOOTING

디버깅 및 문제 해결

연동 과정에서 발생하는 흔한 에러와 해결 방법입니다.

오류 현상 원인 해결 방법
Terraform not found 시스템 경로(PATH) 설정 누락 테라폼 설치 폴더를 환경 변수에 추가
Unauthorized Operation AWS 인증 권한 부족 IAM 사용자의 정책(VPC 관리 등) 확인
Language Server Error 확장 프로그램 충돌 VS Code 재시작 또는 확장 재설치
