이 안내서는 VS Code를 처음 설치한 상태에서 AWS 클라우드 환경과 완벽하게 연결하고, 실제 개발 시 문제가 발생하지 않도록 최적화하는 과정을 단계별로 다룹니다.

---

## 1. VS Code 설치 및 초기 설정

1. **VS Code 설치**: [공식 웹사이트](https://code.visualstudio.com/)에서 운영체제에 맞는 버전을 다운로드하여 설치합니다.
2. **한글 팩 설치**:
* 좌측 사이드바의 **Extensions (확장)** 아이콘(테트리스 블록 모양) 클릭
* `Korean` 검색 후 **Korean (Language) Pack for Visual Studio Code** 설치
* 우측 하단 알림창에서 `Restart` 버튼을 눌러 재시작합니다.



---

## 2. AWS Toolkit 설치 및 인증 연결

AWS 리소스를 탐색하고 관리하기 위해 공식 확장을 설치해야 합니다.

1. **확장 설치**:
* 확장 탭에서 `AWS Toolkit` 검색 후 설치합니다.


2. **IAM User (식별 및 액세스 관리 사용자) 생성**:
* **AWS 웹 콘솔** 접속 → **IAM** 서비스 이동
* **사용자 생성**: 이름 설정 후 `AdministratorAccess` 권한 연결 (학습용)
* **보안 자격 증명** 탭 → **액세스 키 만들기** 클릭 → `CLI` 선택 → **Access Key**와 **Secret Access Key**를 별도로 저장합니다.


3. **VS Code 연결**:
* 왼쪽 사이드바에 생긴 **AWS 아이콘** 클릭
* `Connect to AWS` → `Edit Credentials` 선택
* 열리는 텍스트 파일에 아래 형식으로 정보를 입력하고 저장합니다.



```text
[default]
aws_access_key_id = YOUR_ACCESS_KEY
aws_secret_access_key = YOUR_SECRET_ACCESS_KEY
region = ap-northeast-2

```

## 3. EC2 원격 접속 최적화 (Remote - SSH)

로컬 PC가 아닌 성능이 좋은 AWS EC2 서버 환경에서 직접 코딩하도록 설정합니다.

1. **Remote - SSH 확장 설치**: 확장 탭에서 `Remote - SSH`를 설치합니다.
2. **SSH 설정 파일 편집**:
* `F1` 키를 누르고 `SSH: Open SSH Configuration File...` 입력
* 사용자 홈 폴더의 `.ssh/config` 파일을 선택하고 아래 내용을 작성합니다.



```text
Host my-aws-server
    HostName [EC2의 퍼블릭 IP]
    User ubuntu  (또는 ec2-user)
    IdentityFile "C:/keys/my-key-pair.pem" (내 키페어 경로)

```

## 4. 프리징 방지를 위한 서버 최적화 (CLI 작업)

`t2.micro` 같은 저사양 인스턴스에서 VS Code가 멈추는 것을 방지하기 위한 **필수 커맨드 작업**입니다. EC2에 SSH로 접속한 상태에서 진행하세요.

### 가. 가상 메모리(SWAP) 생성

RAM 부족으로 인한 시스템 다운을 방지합니다.

```bash
# 2GB 크기의 스왑 파일 생성
sudo dd if=/dev/zero of=/swapfile bs=128M count=16

# 권한 설정 (보안)
sudo chmod 600 /swapfile

# 스왑 영역으로 포맷
sudo mkswap /swapfile

# 스왑 활성화
sudo swapon /swapfile

# 재부팅 시 자동 적용을 위해 fstab 수정
echo '/swapfile swap swap defaults 0 0' | sudo tee -a /etc/fstab

```

### 나. 리눅스 커널 파일 감시 제한 확장

VS Code가 많은 파일을 감시할 때 발생하는 에러(`ENOSPC`)를 방지합니다.

```bash
# 파일 감시 개수를 524288개로 증가
echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

```

## 5. VS Code 내부 성능 최적화 설정

서버 성능을 아끼기 위해 VS Code 내부 설정을 변경합니다.

1. **설정 진입**: `Ctrl + ,` (쉼표) 클릭
2. **설정 값 변경**:
* `Files: Watcher Exclude` 검색 → `**/node_modules/**`, `**/.git/objects/**` 추가
* `Editor: Minimap` 검색 → `Enabled` 체크 해제 (가독성 및 리소스 절약)
* `Workbench: Tips` 검색 → `Enabled` 체크 해제



---

## 6. 보안 및 비용 관리 (주의사항)

* **Security Group (보안 그룹)**: SSH(22번 포트)는 반드시 `내 IP`로만 허용하세요. (**과금 발생 안 함**)
* **EIP (Elastic IP, 탄력적 IP)**: 인스턴스를 껐다 켜도 IP가 유지되도록 설정하되, **인스턴스가 종료된 상태에서는 과금이 발생**하므로 주의하세요.
* **사용하지 않을 때**: 인스턴스를 '중지' 상태로 두어야 비용을 아낄 수 있습니다.

---

### 주석 (Annotations)

* **IAM (Identity and Access Management, 식별 및 액세스 관리)**: AWS 서비스에 접근할 수 있는 권한을 유저별로 관리하는 도구입니다.
* **CLI (Command Line Interface, 명령줄 인터페이스)**: 터미널을 통해 텍스트로 명령어를 입력하여 시스템을 조작하는 방식입니다.
* **SWAP (스왑)**: 물리적인 RAM이 가득 찼을 때 하드 디스크의 일부 공간을 메모리처럼 사용하는 기술입니다.
