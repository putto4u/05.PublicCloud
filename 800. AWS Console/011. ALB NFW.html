중앙 집중식 보안 아키텍처 구축 가이드

본 가이드는 Nginx → VPC → TGW → NAT → ALB → NFW → IGW로 이어지는 복합 경로를 통해 외부 브라우저 접속이 가능한 고보안 웹 서비스를 구축하는 전문 직업훈련 과정의 핵심 교안입니다.

최고 수석 아키텍트 설계 보안 컴플라이언스 준수 엔터프라이즈 표준

1. 시스템 구성도 및 트래픽 흐름 (Traffic Flow)

외부 브라우저
→
IGW
(인터넷 게이트웨이)
→
NFW
(네트워크 방화벽)
→
ALB
(로드 밸런서)
→
TGW
(전송 게이트웨이)
→
Nginx
(웹 서버)

[전문 아키텍트 분석]: 위 경로는 외부 사용자가 접속하는 '인바운드(Inbound)' 흐름을 기준으로 설명합니다. `NAT`(Network Address Translation, 네트워크 주소 변환) 게이트웨이는 인스턴스가 패치나 업데이트를 위해 인터넷으로 나가는 '아웃바운드(Outbound)' 전용이므로, 서비스 응답 트래픽의 복귀 경로 혹은 서버 자체의 외부 통신 시에 경유하게 됩니다.

2. 상세 구축 절차 (Step-by-Step)

단계 1: VPC 및 기본 네트워크 설계

VPC 자체는 무료

서비스가 운영될 Service-VPC와 보안/인터넷 접점이 될 Inspection-VPC를 각각 생성합니다.

콘솔 메뉴: VPC 콘솔 > VPC 생성 > 이름: Service-VPC / CIDR: 10.1.0.0/16
콘솔 메뉴: VPC 콘솔 > VPC 생성 > 이름: Inspection-VPC / CIDR: 10.2.0.0/16

단계 2: TGW(전송 게이트웨이) 통합

시간당 점유 비용 + 데이터 처리 비용 발생

두 VPC를 연결하는 중앙 허브를 구축합니다.

콘솔 메뉴: VPC 콘솔 > Transit Gateways > Transit Gateway 생성
콘솔 메뉴: VPC 콘솔 > Transit Gateway Attachments > Attachment 생성 (Service-VPC 및 Inspection-VPC 연결)

단계 3: NFW(네트워크 방화벽) 배치

엔드포인트 비용 + 트래픽 처리 비용 (고가)

Inspection-VPC 내에 방화벽을 배치하여 외부 인입 트래픽의 패킷을 전수 검사합니다.

콘솔 메뉴: VPC 콘솔 > AWS Network Firewall > Firewall 생성
상세 선택: 보안 정책(Firewall Policy) 생성 후 `Suricata` 형식 또는 기본 규칙으로 80/443 포트 허용 설정

단계 4: ALB(애플리케이션 로드 밸런서) 및 Nginx 구성

ALB LCU 시간당 비용 발생

인터넷 가시성을 가진 `ALB`를 생성하고, `Service-VPC`의 프라이빗 서브넷에 있는 `Nginx` 서버를 대상 그룹으로 연결합니다.

콘솔 메뉴: EC2 콘솔 > 로드 밸런서 > ALB 생성 > Internet-facing 선택
콘솔 메뉴: EC2 콘솔 > 대상 그룹 > Target Group 생성 > Nginx 인스턴스 등록

단계 5: NAT 게이트웨이 및 아웃바운드 라우팅

시간당 비용 + 데이터 처리 비용 발생

서버의 보안 업데이트 및 ALB의 인터넷 통신을 위해 `NAT` 게이트웨이를 `Inspection-VPC`의 퍼블릭 서브넷에 배치합니다.

콘솔 메뉴: VPC 콘솔 > NAT Gateway > 생성 (탄력적 IP 할당 필요)

"이 아키텍처의 핵심: 라우팅 테이블(Routing Table) 설계"

① IGW 인바운드 라우팅 (Ingress)

외부 브라우저 접속이 ALB로 가기 전 NFW를 거치게 하는 강제 경로입니다.

Target: NFW Endpoint ID
Destination: ALB Subnet CIDR

② TGW 전파 라우팅 (Propagation)

VPC 간 통신 시 모든 트래픽이 보안 VPC의 TGW 어태치먼트로 향하게 합니다.

Target: TGW ID
Destination: 0.0.0.0/0

* 주석: 이 구조에서 `NFW`(Network Firewall, 네트워크 방화벽)는 반드시 '검사 전용 서브넷'에 위치해야 하며, VPC 라우팅 테이블에서 인터넷 게이트웨이로 향하는 모든 경로의 다음 홉(Next Hop)을 방화벽 엔드포인트로 설정하는 **VPC Ingress Routing** 기술이 적용되어야 외부 브라우저 접속이 안전하게 필터링됩니다.

3. 외부 접속 확인 (Verification)

  • 브라우저에 ALB의 DNS 이름을 입력하여 Nginx 초기 화면이 뜨는지 확인합니다.
  • NFW 로그(CloudWatch Logs)에서 해당 접속 IP의 패킷이 통과(Pass)되었는지 검증합니다.
  • Nginx 서버 내부 보안 그룹(Security Group)에서 ALB의 보안 그룹으로부터의 80포트 인입만 허용했는지 확인합니다.
