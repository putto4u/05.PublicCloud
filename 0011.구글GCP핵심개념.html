Core Architecture & Deep Dive
아키텍트를 위한 핵심 개념 가이드

Why GCP?

  • 글로벌 전용 네트워크 (Private Global Fiber)
  • 오픈소스 리더십 (Kubernetes, TensorFlow, Istio)
  • 데이터 분석 및 머신러닝의 강자 (BigQuery, Vertex AI)
  • 라이브 마이그레이션 (중단 없는 유지보수)

글로벌 인프라

Regions & Zones

전 세계 35개 이상의 리전(Region)과 100개 이상의 존(Zone)으로 구성됩니다. 각 리전은 독립적인 지리적 위치이며, 존은 리전 내의 격리된 하드웨어 영역입니다.

글로벌 백본망

GCP의 가장 큰 차별점은 구글이 직접 소유한 해저 광케이블망입니다. 데이터가 공용 인터넷이 아닌 구글 전용망을 통해 빠르고 안전하게 이동합니다.

리소스 계층 구조 (Resource Hierarchy)

정책(IAM)과 과금은 상위에서 하위로 상속됩니다.

Organization (조직)

루트 노드. 회사 전체의 정책을 관리하는 최상위 컨테이너입니다.

Folder (폴더)

부서(팀)나 환경(Dev, Prod)별로 프로젝트를 그룹화하여 격리합니다.

Project (프로젝트)

모든 리소스(VM, DB 등)가 생성되는 곳이며, 과금(Billing)의 기준입니다.

컴퓨팅 서비스 스펙트럼

구분 서비스명 특징 및 사용 사례
IaaS
직접 관리		 Compute Engine (GCE) 가상 머신(VM). OS, 커널 레벨의 튜닝이 필요하거나 레거시 앱을 그대로 옮길 때 사용합니다. [💰 초단위 과금]

컨테이너		 GKE (Kubernetes) 컨테이너 오케스트레이션의 표준. MSA(마이크로서비스) 아키텍처 구축에 최적화되어 있습니다. [💰 클러스터 관리비]

서버리스		 Cloud Run 추천. 컨테이너를 서버 관리 없이 실행. 0으로 스케일링(비용 0원)이 가능하며 웹/API 서버에 적합합니다. [💰 요청당 과금]

네트워킹: Global VPC

전 세계가 하나의 네트워크

  • 글로벌 단일 VPC: 리전마다 VPC를 만들 필요가 없습니다. VPC 하나로 전 세계 서브넷을 관리합니다.
  • 사설 통신: 서울 리전의 VM과 미국 리전의 DB가 내부 사설 IP(Private IP)로 안전하게 통신합니다.
  • 복잡성 제거: 리전 간 Peering, VPN 설정이 불필요하여 관리 포인트가 획기적으로 줄어듭니다.

보안 및 권한 관리 (IAM)

"누가(Who) 무엇을(Permission) 어디에(Resource) 할 수 있는가?"

1. Member (주체)

Google 계정, 서비스 계정(Service Account), Google 그룹 등 작업을 수행하는 주체입니다.

2. Role (역할)

권한(Permission)의 집합입니다. (예: 뷰어, 편집자, 스토리지 관리자 등)

3. Policy (정책)

Member와 Role을 리소스(프로젝트, 버킷 등)에 연결(Binding)하는 규칙입니다.

데이터 분석의 혁신

BigQuery (빅쿼리) [💰 스캔량 과금]

서버리스 데이터 웨어하우스입니다. 페타바이트급 데이터를 SQL 쿼리만으로 몇 초 만에 분석할 수 있습니다. 스토리지와 컴퓨팅이 분리되어 있어 비용 효율적입니다.

Vertex AI

데이터 준비부터 모델 학습, 배포까지 머신러닝의 전 과정을 통합 관리하는 MLOps 플랫폼입니다. Google의 최신 Gemini 모델을 손쉽게 활용할 수 있습니다.

Deep Dive: Global Load Balancing

1. Single Anycast IP
전 세계 어디서나 동일한 단 하나의 IP로 접속합니다. DNS 캐싱 문제나 전파 지연이 없습니다.
2. Edge PoP Entry
사용자는 자신과 가장 가까운 Google Edge 위치(PoP)로 진입합니다. (공용 인터넷 구간 최소화)
3. Software Defined (Maglev)
하드웨어 장비가 아닌 소프트웨어 기반 LB로, 트래픽 급증 시 워밍업 없이 즉시 확장됩니다.

Why is it superior?

기존 DNS 기반 로드밸런싱(Geo-DNS)의 한계를 극복했습니다.

  • Cross-Region Failover: 서울 리전 장애 시, 별도 설정 없이 즉시 도쿄나 미국 리전으로 트래픽을 우회시킵니다.
  • No Pre-warming: 갑작스러운 트래픽 폭주(Spike)에도 LB 용량 부족으로 인한 패킷 드랍이 발생하지 않습니다.

Deep Dive: Live Migration (Reliability)

유지보수 시에도 "서버 재부팅 없음"

GCP는 물리적 호스트 장비의 유지보수(BIOS 업데이트, 하드웨어 교체 등)가 필요할 때, 실행 중인 VM을 중단 없이 다른 호스트로 이동시킵니다.


작동 원리 (Process)

  1. 이동할 대상 호스트 준비
  2. VM 메모리 상태를 지속적으로 복사 (Pre-copy)
  3. 마지막 순간 짧은 일시정지 (Blackout < 1초)
  4. 새 호스트에서 즉시 재개
