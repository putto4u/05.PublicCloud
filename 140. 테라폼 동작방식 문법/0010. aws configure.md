## AWS CLI 초기 설정: `aws configure` 완벽 가이드

AWS CLI (Command Line Interface: 명령줄 인터페이스)[^1]를 사용하여 AWS 자원을 제어하기 위해서는 가장 먼저 사용자 인증 정보와 기본 환경 설정을 로컬 PC에 등록해야 합니다. 이때 사용하는 핵심 명령어가 바로 `aws configure`입니다.

이 명령어를 실행하면 대화형 프롬프트가 실행되며, AWS 클라우드 환경과 통신하기 위한 4가지 필수 정보를 순차적으로 요구합니다.

### 1. 4가지 필수 입력 항목 상세 분석

터미널(Terminal)에 `aws configure`를 입력하고 `Enter`를 누르면 다음과 같은 항목들을 설정하게 됩니다.

**① AWS Access Key ID (액세스 키 ID)**

* **의미:** AWS 환경에서 사용자를 식별하는 공개 형태의 고유 아이디입니다.
* **입력 예시:** `AKIAIOSFODNN7EXAMPLE`
* **발급 위치:** AWS Management Console (AWS 관리 콘솔)의 IAM (Identity and Access Management: 자원 접근 권한 관리 서비스)[^2] 메뉴에서 보안 자격 증명 탭을 통해 발급받습니다.

**② AWS Secret Access Key (비밀 액세스 키)**

* **의미:** Access Key ID와 쌍을 이루어 사용자를 인증하는 비밀번호 역할을 합니다.
* **주의사항:** 발급 시 단 한 번만 화면에 표시되므로, 반드시 안전한 곳에 저장(다운로드)해야 합니다.
* **입력 예시:** `wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY`
* *⚠️ **보안 및 과금 주의사항**: 이 키가 GitHub(깃허브) 등 퍼블릭 저장소에 노출될 경우, 해커가 악의적으로 고성능 EC2 (Elastic Compute Cloud: 가상 서버 서비스) 인스턴스를 대량으로 생성하여 **수백에서 수천만 원 이상의 막대한 요금(유료 과금)** 이 발생할 수 있습니다. 절대 코드 내에 하드코딩(Hardcoding)하지 마십시오.*

**③ Default region name (기본 리전 이름)**

* **의미:** AWS CLI 명령어가 기본적으로 실행될 지리적 데이터 센터 위치를 지정합니다.
* **입력 예시:** `ap-northeast-2` (아시아 태평양 서울 리전)
* **실무 팁:** 이전 내용에서 강조했듯 `seoul` 등의 도시 이름이 아닌, 반드시 공식 리전 코드를 기입해야 합니다.

**④ Default output format (기본 출력 형식)**

* **의미:** AWS CLI 명령어를 실행했을 때 결괏값을 화면에 보여주는 데이터 포맷을 지정합니다.
* **입력 예시:** `json`, `yaml`, `text`, `table` 중 택 1
* **권장사항:** 프로그래밍 언어나 자동화 스크립트(Script)에서 파싱(Parsing: 구문 분석)하기 가장 유리한 `json` (JavaScript Object Notation: 경량 데이터 교환 형식)[^3]을 기본값으로 사용하는 것이 업계 표준입니다.

### 2. 설정 정보의 저장 위치 및 내부 구조

`aws configure` 명령어를 통해 입력된 정보는 운영체제(OS)의 사용자 홈 디렉터리 하위에 두 개의 숨김 파일로 나뉘어 물리적으로 저장됩니다.

* **자격 증명 파일 (`~/.aws/credentials`)**
보안에 민감한 Access Key와 Secret Key가 저장됩니다.
```ini
[default]
aws_access_key_id = AKIAIOSFODNN7EXAMPLE
aws_secret_access_key = wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

```


* **설정 파일 (`~/.aws/config`)**
상대적으로 보안 민감도가 낮은 리전 및 출력 형식 정보가 저장됩니다.
```ini
[default]
region = ap-northeast-2
output = json

```



### 3. 실전 팁: 흔히 발생하는 실수 및 검증 방법

**자주 오해하거나 실수하는 부분**
새로운 자격 증명을 발급받았을 때 기존 설정을 어떻게 지우거나 수정해야 하는지 혼란스러워하는 경우가 많습니다. 별도의 삭제 명령어를 찾을 필요 없이, 터미널에서 `aws configure`를 다시 실행하여 새로운 값을 입력해 덮어쓰기(Overwrite) 하거나, `~/.aws/` 경로에 있는 위 두 파일을 `vi` 등의 텍스트 편집기로 열어 직접 수정하면 됩니다.

**설정 완료 검증하기**
모든 입력이 끝난 후 설정이 정상적으로 인가되었는지 확인하려면 다음 명령어를 통해 본인의 IAM 사용자 권한 상태를 조회해 봅니다.

```bash
aws sts get-caller-identity

```

통신이 정상적으로 이루어졌다면 Account (계정 ID), Arn (Amazon Resource Name: AWS 자원 고유 식별자)[^4], UserId (사용자 ID)가 JSON 형태로 출력됩니다.
