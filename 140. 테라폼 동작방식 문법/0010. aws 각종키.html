Phase 2. AWS 핵심 키 4대장
발급 무료

Access Key

액세스 키 아이디 & 비밀 액세스 키

프로그래밍 방식(CLI, SDK)으로 AWS API를 호출하여 클라우드 자원을 제어하는 장기 자격 증명.

주 사용처: Terraform(테라폼), CI/CD(지속적 통합/배포) 자동화 파이프라인
발급 무료

Key Pair

Amazon EC2 Key Pair (아마존 엘라스틱 컴퓨트 클라우드 키 페어)

가상 서버 운영체제에 원격으로 로그인(SSH, RDP)하기 위한 비대칭 암호화 키 (.pem / .ppk).

주의사항: 최초 1회만 다운로드 가능. 분실 시 서버 접근 불가.
유료 서비스

KMS Key

AWS Key Management Service Key (아마존 웹 서비스 키 관리 서비스 키)

DB, 스토리지 파일, 애플리케이션 데이터 등을 암/복호화하는 중앙 집중식 마스터 키.

과금 포인트: 커스텀 키 1개당 월 $1 + API 호출 1만 건당 $0.03 부과³
발급 무료

API Key

Amazon API Gateway API Key (아마존 에이피아이 게이트웨이 키)

자체 개발한 서버리스 API를 호출하는 고객의 사용량을 추적하고 제어하기 위한 식별자.

주 사용처: 고객사별 초당 호출 횟수(Throttling) 제한 및 과금 모델 구축

클라우드 해킹 원인 1위: Access Key 하드코딩

Secret Access Key(비밀 액세스 키)가 포함된 코드를 GitHub(깃허브) 퍼블릭 리포지토리에 올리는 순간, 전 세계 해킹 봇이 1분 이내에 스캔하여 최고 사양의 가상 채굴 서버 수백 대를 생성합니다. (수천만 원 요금 폭탄 발생)

절대 금지 ❌

코드 내부에 직접 키 값 입력 (하드코딩)

모범 사례 ✅

로컬 환경 변수 사용 및 .gitignore 적용.
AWS 자원 간 통신은 IAM Role(아이엠 역할) 사용.

용어 주석 및 참고 자료

1. IAM Policy(아이엠 정책): AWS 리소스에 대한 접근 권한을 매우 세밀하게 정의하는 JSON(JavaScript Object Notation) 형식의 규칙 문서입니다. 이 문서에 따라 자원의 허용/차단 여부가 결정됩니다.

2. 프로비저닝(Provisioning): IT 인프라를 설정하는 프로세스로, 사용자의 요구에 맞게 네트워크, 서버 등을 할당하고 배치하여 시스템을 즉시 사용할 수 있는 상태로 준비해 두는 과정입니다.

3. KMS 요금제: 기본 제공 관리형 키는 무료이나, 사용자 제어(Customer Managed Key) 생성 시 발생합니다. 또한, 애플리케이션 봉투 암호화(Envelope Encryption: 데이터 키 생성 후 마스터 키로 한 번 더 암호화하는 아키텍처) 적용 시 API 호출량에 따라 과금됩니다.
