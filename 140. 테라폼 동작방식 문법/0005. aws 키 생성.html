AWS Access Key 생성 및 핵심 키(Key) 관리 가이드

1. AWS Access Key(액세스 키) 생성 시 '사용 사례' 선택의 기술적 의미

AWS(Amazon Web Services, 아마존 웹 서비스) 콘솔에서 Access Key(액세스 키)를 생성할 때 마주하는 '사용 사례' 선택 화면은 많은 초보자가 혼란을 겪는 부분입니다. 결론부터 설명하면, 'AWS 외부에서 실행되는 애플리케이션' 또는 'Command Line Interface(CLI, 명령줄 인터페이스)' 등 어떤 항목을 선택하더라도 Terraform(테라폼) 및 개인 PC 환경에서 접근하고 사용하는 데 전혀 문제가 없습니다.

📌 화면의 실제 목적

이 화면에서의 선택이 생성되는 키의 기술적 특성이나 접근 권한을 물리적으로 제한하지 않습니다. 이 단계는 AWS가 사용자의 이용 목적을 통계적으로 파악하고, 그에 맞는 보안 모범 사례나 대안 서비스(예: IAM Roles Anywhere 등)를 안내해 주기 위한 단순 분류 및 가이드라인 제시 목적입니다. 어떤 항목을 선택하든 최종 발급되는 Access Key ID(액세스 키 아이디)와 Secret Access Key(비밀 액세스 키)의 통신 기능은 100% 동일합니다.

과금 및 유료 전환 안내

IAM(Identity and Access Management, 항등 및 접근 관리) 계정 생성 및 Access Key(액세스 키) 발급 자체는 전면 무료입니다. 그러나 이 키를 사용하여 Terraform(테라폼) 등으로 유료 AWS 클라우드 자원(예: EC2 등)을 실제 생성(프로비저닝[1])할 때부터 해당 자원의 요금제에 따라 과금이 발생합니다.

2. AWS 핵심 키(Key) 종류 및 사용 용도 총정리

AWS 환경에서는 자원에 접근하거나 데이터를 암호화하기 위해 다양한 형태의 키(Key)가 사용됩니다. 각 키의 목적을 정확히 구분하는 것은 클라우드 보안 아키텍처 설계의 핵심입니다.

키(Key) 명칭 영문 전체 명칭 및 한글 설명 주 사용 목적 및 대상 과금 여부
Access Key Access Key ID & Secret Access Key
(액세스 키 아이디 및 비밀 액세스 키)		 프로그래밍 방식(CLI, SDK)으로 AWS API를 호출하여 자원을 제어할 때 사용하는 장기 자격 증명 전면 무료
(생성된 자원은 과금)
Key Pair Amazon EC2 Key Pair
(아마존 엘라스틱 컴퓨트 클라우드 키 페어)		 가상 서버(인스턴스) 운영체제에 원격으로 로그인(SSH, RDP)하기 위한 비대칭 암호화 키 전면 무료
(EC2 실행 비용 별도)

(아마존 에이피아이 게이트웨이 에이피아이 키)		 자체 구축 API를 호출하는 고객의 사용량을 추적하고 제어하기 위한 식별자 키 생성 자체 무료
(호출 횟수에 따라 과금)

3. 키 종류별 상세 설명 및 추가 사용 용도

A. Access Key (액세스 키 아이디 및 비밀 액세스 키)

IAM(Identity and Access Management, 항등 및 접근 관리) 사용자에게 부여되는 영구적인 자격 증명입니다. 외부 시스템이나 스크립트에서 AWS 클라우드를 제어할 때 필수적입니다.

  • 기본 용도: PC의 CLI 환경이나 Terraform(테라폼)과 같은 IaC(Infrastructure as Code, 코드형 인프라) 도구에서 인증 수단으로 사용.
  • 추가 용도: 온프레미스 서버의 S3 백업 연동, CI/CD(지속적 통합/지속적 배포) 파이프라인(GitHub Actions 등)의 배포 인증.

💡 실전 팁 및 자주 오해하는 부분

절대 주의(해킹 사례 1위): 소스 코드를 GitHub(깃허브)에 커밋할 때 Secret Access Key가 하드코딩된 상태로 퍼블릭 리포지토리에 올라가면, 1분 이내에 해킹 봇이 스캔하여 악성 채굴 서버를 생성합니다. 수천만 원의 요금 폭탄을 맞을 수 있으므로 반드시 환경 변수(Environment Variables)로 분리하고 .gitignore에 등록해야 합니다.

모범 사례 대안: AWS 내부 자원(예: EC2)이 다른 자원에 접근할 때는 Access Key를 서버에 저장하지 마십시오. 대신 IAM Role(아이엠 역할, 항등 및 접근 관리 역할)을 부여하여 임시 자격 증명을 획득하게 하는 것이 핵심 원칙입니다.

B. Key Pair (아마존 엘라스틱 컴퓨트 클라우드 키 페어)

공개 키(Public Key) 암호화 방식을 사용하여 AWS 가상 머신에 접속할 때 패스워드를 대체하는 수단입니다.

  • 기본 용도: Linux(리눅스) 서버에 SSH(Secure Shell, 보안 셸)로 접속하거나, Windows(윈도우) 서버의 관리자 비밀번호 해독.
  • 추가 용도: Ansible(앤서블) 자동화 환경의 다중 서버 접속 인증, Bastion Host(베스천 호스트, 접근 제어용 서버) 터널링 인증.

💡 실전 팁 및 자주 오해하는 부분

프라이빗 키는 EC2 인스턴스를 생성하는 시점에 단 한 번만 다운로드할 수 있습니다. 분실 시 서버 접근이 매우 어려워지므로 관리에 유의해야 합니다. 최근에는 키 페어 없이도 안전하게 접속할 수 있는 SSM Session Manager(시스템 매니저 세션 매니저) 사용이 권장됩니다.

C. KMS Key (아마존 웹 서비스 키 관리 서비스 키)

데이터를 암호화하기 위한 마스터 키(Customer Master Key, CMK)를 생성하고 관리하는 서비스입니다.

  • 기본 용도: RDS 데이터베이스, S3 버킷 파일, EBS 볼륨 등의 '저장 데이터 암호화(Encryption at Rest)'.
  • 추가 용도: 애플리케이션 봉투 암호화(Envelope Encryption)[3], AWS Secrets Manager(시크릿 매니저) 연동 비밀 정보 보호.

D. API Key (아마존 에이피아이 게이트웨이 에이피아이 키)

서버리스 환경에서 사용자 고유의 API를 서비스할 때 사용하는 접근 제어 문자열입니다.

  • 기본 용도: 외부 개발자나 파트너사가 자체 구축 API를 호출할 때의 식별자.
  • 추가 용도: 특정 키마다 접속 트래픽을 제한하는 사용량 제어(Throttling) 및 할당량(Quota) 관리를 통한 서비스 티어 분리.

[1] 프로비저닝(Provisioning): IT 인프라를 설정하는 프로세스로, 사용자의 요구에 맞게 네트워크, 서버, 운영체제 등을 할당하고 배치하여 시스템을 즉시 사용할 수 있는 상태로 준비해 두는 일련의 과정을 의미합니다.

[2] KMS 키 요금제: AWS에서 기본적으로 제공하는 AWS 관리형 키는 무료지만, 사용자가 직접 정책을 제어하는 고객 관리형 키(Customer Managed Key)를 생성할 경우 키 1개당 매월 $1가 과금되며, 해당 키를 사용하여 데이터를 암호화/복호화하는 API 요청 10,000건당 $0.03의 요금이 별도로 부과됩니다.

[3] 봉투 암호화(Envelope Encryption): 민감 데이터를 보호하기 위해 데이터 본문을 암호화하는 '데이터 키'를 먼저 생성하고, 이 '데이터 키' 자체를 한 번 더 안전한 마스터 키로 암호화하여 봉투에 넣듯 이중으로 보호하는 고급 암호화 아키텍처입니다.
