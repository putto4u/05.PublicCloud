AWS CLI 초기 설정: aws configure 완벽 가이드

AWS CLI (Command Line Interface: 명령줄 인터페이스)[1]를 사용하여 AWS 자원을 제어하기 위해서는 가장 먼저 사용자 인증 정보와 기본 환경 설정을 로컬 PC에 등록해야 합니다. 이때 사용하는 핵심 명령어가 바로 aws configure입니다.

명령어 실행 화면

$ aws configure
AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
Default region name [None]: ap-northeast-2
Default output format [None]: json

위 명령어를 실행하면 대화형 프롬프트가 실행되며, AWS 클라우드 환경과 통신하기 위한 4가지 필수 정보를 순차적으로 요구합니다.

1. 4가지 필수 입력 항목 상세 분석

1 AWS Access Key ID

의미: AWS 환경에서 사용자를 식별하는 공개 형태의 고유 아이디입니다.

발급 위치: AWS Management Console (AWS 관리 콘솔)의 IAM (Identity and Access Management)[2] 메뉴에서 발급받습니다.

AKIAIOSFODNN7EXAMPLE

2 AWS Secret Access Key

의미: Access Key ID와 쌍을 이루어 사용자를 인증하는 비밀번호 역할을 합니다.

주의사항: 발급 시 단 한 번만 화면에 표시되므로, 반드시 안전한 곳에 저장해야 합니다.

wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

보안 및 과금 주의사항 (유료 서비스 관련)

이 키가 GitHub(깃허브) 등 퍼블릭 저장소에 노출될 경우, 해커가 악의적으로 고성능 EC2 (Elastic Compute Cloud: 가상 서버 서비스) 인스턴스를 대량으로 생성하여 수백에서 수천만 원 이상의 막대한 요금(유료 과금)이 발생할 수 있습니다. 절대 코드 내에 하드코딩(Hardcoding)하지 마십시오.

3 Default region name

의미: AWS CLI 명령어가 기본적으로 실행될 지리적 데이터 센터 위치를 지정합니다.

실무 팁: seoul 등의 도시 이름이 아닌, 반드시 공식 리전 코드를 기입해야 합니다.

ap-northeast-2

4 Default output format

의미: AWS CLI 명령어를 실행했을 때 결괏값을 화면에 보여주는 데이터 포맷을 지정합니다. (json, yaml, text, table)

권장사항: 프로그래밍 언어나 자동화 스크립트에서 구문 분석(Parsing)하기 가장 유리한 json[3]을 기본값으로 사용하는 것이 업계 표준입니다.

json

2. 설정 정보의 저장 위치 및 내부 구조

aws configure 명령어를 통해 입력된 정보는 운영체제(OS)의 사용자 홈 디렉터리 하위에 두 개의 숨김 파일로 나뉘어 물리적으로 저장됩니다.

~ (Home Directory)
.aws
credentials
config
~/.aws/credentials 보안 민감 정보
[default]
aws_access_key_id = AKIAIOSFODNN7EXAMPLE
aws_secret_access_key = wJalrXUtnFEMI/K7MDENG/bPxRfiCY...
~/.aws/config 일반 설정 정보
[default]
region = ap-northeast-2
output = json

3. 실전 팁: 흔히 발생하는 실수 및 검증 방법

자주 오해하거나 실수하는 부분

새로운 자격 증명을 발급받았을 때 기존 설정을 어떻게 지우거나 수정해야 하는지 혼란스러워하는 경우가 많습니다. 별도의 삭제 명령어를 찾을 필요 없이, 터미널에서 aws configure를 다시 실행하여 새로운 값을 입력해 덮어쓰기(Overwrite) 하거나, ~/.aws/ 경로에 있는 위 두 파일을 텍스트 편집기로 열어 직접 수정하면 됩니다.

설정 완료 검증하기

모든 입력이 끝난 후 설정이 정상적으로 인가되었는지 확인하려면 다음 명령어를 통해 본인의 IAM 사용자 권한 상태를 조회해 봅니다.

$ aws sts get-caller-identity

통신이 정상적으로 이루어졌다면 Account (계정 ID), Arn (Amazon Resource Name: AWS 자원 고유 식별자)[4], UserId (사용자 ID)가 JSON 형태로 출력됩니다.

{
    "UserId": "AIDAIOSFODNN7EXAMPLE",
    "Account": "123456789012",
    "Arn": "arn:aws:iam::123456789012:user/admin"
}

각주 및 참고자료

  • [1] CLI (Command Line Interface): 그래픽 사용자 인터페이스(GUI)가 아닌 텍스트 터미널을 통해 컴퓨터와 상호 작용하는 방식.
  • [2] IAM (Identity and Access Management): AWS 리소스에 대한 액세스를 안전하게 제어할 수 있도록 돕는 웹 서비스. 서비스 자체는 무료이나, 권한이 탈취되어 생성된 리소스에 대해서는 비용이 청구됨.
  • [3] JSON (JavaScript Object Notation): 데이터를 키-값 쌍(Key-Value Pair)으로 이루어진 텍스트 형태로 저장하는 개방형 표준 포맷.
  • [4] ARN (Amazon Resource Name): AWS 전체 영역에서 특정 자원을 명확하게 식별하기 위해 사용하는 고유한 이름 규칙.
