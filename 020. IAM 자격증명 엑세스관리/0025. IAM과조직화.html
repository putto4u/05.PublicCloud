AWS 보안 관리 시스템 비교

IAM (자격 증명 및 액세스 관리) vs AWS Organizations (조직 관리 서비스)

{/* Introduction Section */}

IAM

단일 AWS 계정 내부에서 '누가(Identity)' '무엇을(Action)' 할 수 있는지 정의합니다. 사용자와 리소스 간의 구체적인 권한을 설계하는 마이크로(Micro) 수준의 제어 도구입니다.

AWS Organizations

다수의 AWS 계정을 중앙에서 관리합니다. 전체 계정군에 적용될 보안 가드레일을 설정하고 비용을 통합하는 매크로(Macro) 수준의 거버넌스 도구입니다.

SCP (Service Control Policy, 서비스 제어 정책): 조직 전체 또는 계정 그룹(OU)에 적용되어, 개별 계정의 IAM 권한이 넘지 못하는 최대 권한 경계를 설정합니다.

{/* Best Practices & Key Insights */}

실전 모범 사례 (Best Practices)

  • •
    IAM: MFA (Multi-Factor Authentication, 다요소 인증)를 모든 사용자에게 강제하세요.
  • •
    IAM: 최소 권한의 원칙 (Principle of Least Privilege)에 따라 필요한 만큼만 권한을 부여하세요.
  • •
    Organizations: 통합 결제를 통해 전체 계정의 볼륨 할인을 활용하세요.

주의사항 (Caution)

  • •
    SCP는 권한을 부여하지 않습니다. SCP에서 허용되어도 IAM Policy가 없으면 접근이 거부됩니다.
  • •
    루트(Root) 계정은 일상 업무에 사용하지 말고 관리 전용 계정으로 보호하세요.
