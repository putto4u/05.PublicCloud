Terraform(테라폼)을 사용하여 디렉토리를 생성하고 가장 기초적인 'Hello World'를 출력하는 구성을 안내합니다. 이 과정은 테라폼의 설치 여부를 확인하고, 로컬 파일 시스템에 리소스를 생성하는 방식으로 진행됩니다.

---

## 테라폼 프로젝트 시작하기

테라폼은 별도의 복잡한 설정 없이도 **Local Provider**(로컬 프로바이더)를 사용하여 로컬 환경에 파일을 생성하는 것으로 첫 걸음을 뗄 수 있습니다.

### 1. 작업 디렉토리 생성 및 이동

먼저 테라폼 설정 파일들을 담을 전용 폴더를 생성합니다.

```bash
mkdir terraform-hello-world
cd terraform-hello-world

```

### 2. 테라폼 코드 작성 (`main.tf`)

디렉토리 안에 `main.tf` 파일을 생성하고 아래 내용을 작성합니다. 이 코드는 `hello.txt`라는 파일을 생성하고 그 안에 내용을 적는 역할을 합니다.

```hcl
# 로컬 파일 생성을 위한 리소스 정의
resource "local_file" "hello_world" {
  content  = "Hello, Terraform World!"
  filename = "${path.module}/hello.txt"
}

# 실행 후 화면에 메시지를 출력하는 출력 변수
output "greeting" {
  value = "Terraform has successfully created the file."
}

```

### 3. 테라폼 명령어 실행 (3단계 Workflow)

코드를 작성했다면 아래의 표준 절차를 따릅니다.

* **Step 1: 초기화 (Init)**
작성한 코드에 필요한 플러그인(Provider)을 다운로드합니다.
```bash
terraform init

```


* **Step 2: 계획 확인 (Plan)**
실행하기 전 어떤 변경 사항이 생길지 미리 검토합니다. **(비용 발생 없음)**
```bash
terraform plan

```


* **Step 3: 적용 (Apply)**
실제 리소스를 생성합니다. 실행 시 `yes`를 입력해야 완료됩니다.
```bash
terraform apply

```



---

## 실행 결과 확인

`apply`가 완료되면 디렉토리에 `hello.txt` 파일이 생성된 것을 볼 수 있습니다.

* **파일 내용 확인:** `cat hello.txt` 명령어를 치면 `Hello, Terraform World!`가 출력됩니다.
* **상태 파일:** 테라폼이 생성한 리소스를 관리하기 위해 `terraform.tfstate`라는 파일이 자동으로 생성됩니다.

> [!NOTE]
> **주의사항**: 위 예제는 로컬 PC의 파일 시스템을 사용하므로 별도의 클라우드 비용이 발생하지 않습니다. 하지만 AWS나 Azure 등의 리소스를 생성할 때는 **Free Tier**(프리 티어) 범위를 벗어나면 과금될 수 있으니 주의가 필요합니다.
