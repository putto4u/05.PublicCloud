Windows 환경에서 패키지 관리자인 **Choco**(Chocolatey; 초콜릿티)를 사용하여 **Terraform**(테라폼)을 설치하는 과정을 단계별로 설명합니다.

---

## 1. Chocolatey(초콜릿티) 개요

**Chocolatey**(초콜릿티)는 Windows용 소프트웨어 패키지 관리자입니다. 리눅스의 `apt`나 `yum`, macOS의 `brew`와 유사하게 명령줄 인터페이스(CLI; Command Line Interface; 명령줄 인터페이스)를 통해 소프트웨어를 설치, 업데이트, 삭제할 수 있게 해줍니다.

* **장점**: 복잡한 환경 변수(PATH; 경로 설정 변수) 설정 없이 명령어 한 줄로 도구를 설치할 수 있습니다.
* **비용**: 개인 사용자를 위한 오픈소스 버전은 **무료**입니다. (기업용 기능은 유료 과금 모델이 존재하나, 테라폼 설치는 무료 버전으로 충분합니다.)

[Chocolatey 공식 홈페이지](https://chocolatey.org/)

---

## 2. Chocolatey 설치 확인 및 설치

테라폼을 설치하기 전에 먼저 시스템에 Chocolatey가 설치되어 있는지 확인해야 합니다.

### 2.1 관리자 권한으로 PowerShell 실행

1. 시작 메뉴에서 **PowerShell**을 검색합니다.
2. 마우스 오른쪽 버튼을 클릭하여 **'관리자 권한으로 실행'**을 선택합니다.

### 2.2 설치 여부 확인

다음 명령어를 입력합니다.

```powershell
choco -v

```

버전 번호가 출력되지 않는다면 아래 명령어를 복사하여 붙여넣어 설치를 진행합니다. 긴 명령어가 잘리지 않도록 PowerShell의 줄바꿈 기호인 **` 백틱(`)** 문자를 사용하여 구분하였습니다.

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force; `
[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; `
iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

```

## 3. Terraform(테라폼) 설치하기

이제 Chocolatey를 이용하여 테라폼을 설치합니다.

### 3.1 설치 명령어 실행

관리자 권한의 PowerShell 창에서 아래 명령어를 입력합니다.

```powershell
choco install terraform

```

[이미지: Windows PowerShell에서 Chocolatey를 이용한 테라폼 설치 화면]

* **중요**: 설치 중에 스크립트 실행 여부를 묻는 메시지(`Do you want to run the script?([Y]es/[A]ll - yes to all...)`)가 나오면 **'Y'** 또는 **'A'**를 입력하고 엔터를 누릅니다.

### 3.2 설치 확인

설치가 완료되면 PowerShell을 **종료 후 다시 실행**하거나, 아래 명령어로 설치된 버전을 확인합니다.

```powershell
terraform -version

```

---

## 4. 실전 팁 및 주의사항

### 4.1 버전 고정 설치

특정 프로젝트에서 하위 호환성을 위해 구버전의 테라폼이 필요한 경우, `--version` 옵션을 사용하여 설치할 수 있습니다.

```powershell
choco install terraform --version 1.5.0

```

### 4.2 최신 버전으로 업데이트

이미 테라폼이 설치되어 있고, 최신 기능을 사용하기 위해 업데이트가 필요한 경우 아래 명령어를 사용합니다.

```powershell
choco upgrade terraform

```

### 4.3 자주 발생하는 오류 (Troubleshooting; 문제 해결)

* **'terraform' 용어가 cmdlet, 함수, 스크립트 파일 또는 실행할 수 있는 프로그램 이름으로 인식되지 않습니다**: 이 에러는 보통 설치 후 환경 변수가 현재 세션에 반영되지 않아 발생합니다. PowerShell 창을 닫고 다시 열면 해결됩니다.
* **권한 문제**: 반드시 **관리자 권한**으로 실행된 터미널에서 `choco` 명령어를 사용해야 합니다.
