03

볼륨 유형 선택 가이드

gp3 (General Purpose 3)

범용 SSD: 실무 표준. 성능(IOPS)과 용량을 독립적으로 조절 가능하여 가성비 극대화.

권장 저비용

io2 (Provisioned IOPS 2)

고성능 SSD: 지연시간(Latency)에 극도로 민감한 대규모 데이터베이스(DB) 및 SAP HANA용.

유료(고가) 최고성능

st1 (Throughput Optimized)

처리량 최적화 HDD: 빅데이터 분석, 로그 처리 등 대용량 데이터의 연속적인 작업에 적합.

저가형 대용량

sc1 (Cold HDD)

콜드 HDD: 접근 빈도가 매우 낮은 아카이브성 데이터 보관용. 가장 저렴함.

최저가 보관용
* IOPS(Input/Output Operations Per Second): 초당 입출력 횟수. 주로 작은 크기의 데이터를 자주 읽고 쓸 때 중요.
* Throughput(처리량): 초당 데이터 전송량. 대용량 파일을 한꺼번에 옮길 때 중요.

EBS 스냅샷: 백업 및 복구 프로세스

스냅샷 생성

특정 시점의 데이터를 Amazon S3에 저장

증분 백업 적용

AMI 변환

스냅샷을 기반으로 머신 이미지(AMI) 생성

Amazon Machine Image

서버 복제

AMI를 사용하여 동일 설정의 서버 즉시 배포

FSR로 빠른 복구
증분 백업 (Incremental Backup): 마지막 백업 이후 변경된 데이터 블록만 저장하여 비용과 공간을 절약하는 방식.
FSR (Fast Snapshot Restore): 지연 로딩(Lazy Loading) 없이 스냅샷에서 볼륨을 즉시 생성할 수 있게 해주는 기능 (유료).
06

실무 대응 시나리오 (Case Study)

성능 저하 발생

CloudWatch로 Burst Balance(버스트 잔량) 확인. 잔량 고갈 시 gp3로 전환하여 IOPS 강제 할당.

리전 재해 복구

DLM(Data Lifecycle Manager)을 사용하여 스냅샷을 다른 리전으로 자동 복제하여 지리적 가용성 확보.

보안 규정 준수

KMS(Key Management Service)를 연동하여 EBS 볼륨 암호화 적용. 데이터 유출 시 무단 판독 방지.

* CloudWatch: AWS 자원 모니터링 서비스 / DLM: 백업 정책 자동화 도구
