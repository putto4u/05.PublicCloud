Core Infrastructure

가상 서버의 심장:
Amazon EC2 인스턴스 이해

Amazon EC2(Elastic Compute Cloud, 탄력적 컴퓨팅 클라우드)는 AWS(Amazon Web Services, 아마존 웹 서비스)의 핵심 서비스로, 클라우드에서 가상 서버를 제공합니다. 사용자의 목적에 따라 CPU1, 메모리, 저장 공간의 비율이 최적화된 다양한 인스턴스 유형과 예산에 맞춘 요금제를 선택할 수 있습니다.

1

EC2 인스턴스 명칭 해석법 (Naming Convention)

인스턴스 이름은 일정한 규칙에 따라 명명됩니다. 이 명칭 체계만 이해하면 서버의 성능과 세대를 즉시 파악할 수 있습니다.

명칭 구조 도식화
m
패밀리 (Family)
6
세대 (Generation)
gd
추가 속성 (Attributes)
.
large
크기 (Size)

패밀리 (Family)

인스턴스의 주 목적을 나타냅니다. (C=Compute, M=Memory 등)

세대 (Generation)

숫자가 높을수록 최신 하드웨어이며 가성비가 우수합니다.

추가 속성 (Attributes)

  • g: AWS Graviton 프로세서 (가성비 최적화 ARM 칩)
  • i: Intel 프로세서 / a: AMD 프로세서
  • d: 인스턴스 저장소(로컬 NVMe SSD2) 포함

크기 (Size)

vCPU3와 메모리 용량의 규모를 나타냅니다.
(nano < micro < small < medium < large < xlarge ...)

분석 실습 1
m6gd.large
  • m 범용 (컴퓨팅과 메모리 균형)
  • 6 6세대 하드웨어
  • g AWS Graviton(ARM) 칩 (가성비 우수)
  • d 로컬 NVMe SSD 디스크 포함
  • large vCPU 2개, 메모리 8GB 사양
분석 실습 2
c7i.xlarge
  • c 컴퓨팅 최적화 (CPU 비중 높음)
  • 7 최신 7세대 하드웨어
  • i Intel 프로세서 사용
  • xlarge vCPU 4개, 메모리 16GB 사양

주요 패밀리별 상세 분류 및 용도

실무 사례를 곁들여 각 인스턴스 패밀리의 특징을 상세히 분류해 보겠습니다.

범용 (General Purpose)

T, M

컴퓨팅, 메모리, 네트워킹 리소스가 균형 잡힌 유형입니다. 어떤 용도로 써야 할지 모를 때 가장 먼저 고려하는 표준형입니다.

주요 용도: 웹 서버, 개발/테스트 환경, 소규모 DB
예: t3.micro, m6i.large

컴퓨팅 최적화

C

고성능 프로세서가 집중적으로 할당되어, 메모리 대비 CPU 성능 비율이 높게 설계되었습니다.

주요 용도: 고성능 웹 서버, 배치(Batch) 작업, 멀티플레이어 게임 서버
예: c6a.2xlarge, c7g.xlarge

메모리 최적화

R, X, z

CPU에 비해 대용량 RAM(Random Access Memory, 휘발성 메모리) 용량이 압도적으로 많습니다.

주요 용도: 메모리 내 데이터베이스(Redis 등)4, 대규모 데이터 분석
예: r6i.2xlarge, r7g.large

가속 컴퓨팅

P, G, F

GPU(그래픽 처리 장치) 또는 FPGA(현장 프로그래머블 게이트 어레이) 하드웨어 가속기를 포함합니다.

주요 용도: 딥러닝 기계학습(Machine Learning), 그래픽 렌더링
예: p4d.24xlarge, g5.xlarge

스토리지 최적화

I, D, H

로컬 스토리지의 높은 I/O(Input/Output, 입출력) 성능을 요구하는 워크로드에 설계되었습니다. 아주 빠른 디스크 읽기/쓰기 속도가 필요할 때 사용합니다.

주요 용도: 분산 파일 시스템, 대규모 데이터 웨어하우징, NoSQL DB
예: i4i.large, d3.xlarge

인스턴스 크기 확장 원리 (Scale-up)

인스턴스 크기가 한 단계 커질 때마다 일반적으로 자원(vCPU, 메모리)과 과금액이 정확히 2배씩 증가합니다.

1x
t3.medium 2 vCPU / 4 GiB
2x
t3.large 2 vCPU / 8 GiB
4x
t3.xlarge 4 vCPU / 16 GiB
8x
t3.2xlarge 8 vCPU / 32 GiB
3

EC2 구매 옵션 (요금제)

사용 기간과 예산 상황에 따라 4가지 주요 구매 옵션을 제공합니다. 비용 효율적인 인프라 운영을 위해 워크로드의 성격에 맞는 요금제를 선택해야 합니다.

기준가 0% 할인

① 온디맨드 (On-Demand)

약정 없이 사용한 만큼 초 단위로 비용을 지불합니다. 생성과 삭제가 자유로워 가장 유연합니다.

적합: 단기 프로젝트, 예측 불가능한 트래픽, 신규 애플리케이션 개발 및 테스트
최대 90% 할인

② 스팟 인스턴스 (Spot)

AWS 데이터 센터 내에 남는 잉여 자원을 매우 저렴하게 이용합니다. 단, AWS가 자원이 필요해지면 2분 전 통보 후 인스턴스를 강제 회수합니다.

적합: 중단되어도 상관없는 배치 작업, 빅데이터 분석, 상태가 없는(Stateless) 서비스
최대 72% 할인

③ 예약 인스턴스 (RI)

1년 또는 3년 기간을 약정하여 사용하는 대신 요금을 대폭 할인받는 옵션입니다. (Savings Plans 포함)

적합: 상태가 일정하고 예측 가능한 장기 워크로드, 24시간 중단 없이 가동되는 서비스

④ 전용 호스트 (Dedicated Hosts)

고객 전용의 물리적 EC2 서버를 통째로 할당받습니다. 기존에 보유한 소프트웨어 라이선스(BYOL) 규정을 준수해야 하거나, 국가/기업의 엄격한 물리적 보안 규정이 필요한 경우에 사용합니다.

실전 팁: 비용을 줄이는 하이브리드(Hybrid) 구성 전략

실제 기업의 인프라 설계 시에는 하나의 요금제만 쓰지 않고 혼합하여 비용과 안정성을 모두 잡습니다.

Base Layer (기본층)

최소한으로 유지되는 기본 트래픽은 예약 인스턴스(RI)로 구성하여 고정 비용을 대폭 절감합니다.

Burst Layer (확장층)

이벤트 등으로 갑작스러운 트래픽이 증가할 때는 온디맨드(On-Demand)로 스케일 아웃(Scale-out)하여 유연하게 대응합니다.

Batch Layer (처리층)

즉각 처리하지 않아도 되는 백그라운드 분석/로그 처리 작업은 스팟 인스턴스(Spot)로 돌려 비용을 극소화합니다.

과금 및 유료 전환 주의 (필독)

  • 프리 티어(Free Tier) 유료 전환 주의 계정 생성 후 1년간 무료로 제공되는 프리 티어를 사용하는 경우, 반드시 t2.micro 또는 t3.micro(리전별 상이) 사양만 선택해야 합니다. 실수로 m5.large와 같은 상위 사양을 생성하면 생성 즉시 시간당 약 $0.096 USD (서울 리전 기준 약 130원) 이상의 과금이 발생하기 시작합니다.
  • 스팟 인스턴스 데이터 유실 위험 저렴한 스팟 인스턴스는 AWS 자원 상황에 따라 예고 없이(2분 전 알림) 회수될 수 있습니다. 이때 메모리와 디스크의 데이터가 모두 유실될 수 있으므로, 반드시 AWS S3(Simple Storage Service)나 RDS(Relational Database Service)와 같은 외부 저장소로 실시간 데이터를 분리 및 백업하는 아키텍처 설계가 병행되어야 합니다.
  • 구형 세대 선택 주의 같은 m 패밀리라도 m4 등 이전 세대를 선택하면 최신 세대(m6, m7)에 비해 하드웨어 성능은 떨어지면서 가격은 더 비쌀 수 있습니다. 특별한 호환성 이유가 없다면 항상 최신 세대를 선택하세요.

주석 및 용어 설명

  1. CPU (Central Processing Unit, 중앙 처리 장치): 컴퓨터의 두뇌 역할을 하며 시스템 전반의 연산을 담당하는 핵심 하드웨어입니다.
  2. NVMe SSD (Non-Volatile Memory express Solid State Drive): 기존 디스크 드라이브보다 훨씬 빠른 데이터 전송 속도를 제공하는 고성능 비휘발성 저장 장치 규격입니다.
  3. vCPU (Virtual CPU, 가상 중앙 처리 장치): 물리적인 CPU 코어를 가상화 기술을 통해 논리적으로 나눈 단위로, 클라우드 환경에서 컴퓨팅 성능을 측정하는 기준이 됩니다.
  4. 인메모리 데이터베이스 (In-Memory Database): 디스크가 아닌 시스템 메모리(RAM)에 데이터를 저장하여 밀리초(ms) 단위의 초고속 데이터 응답 속도를 제공하는 데이터베이스입니다. (예: Redis, Memcached)
