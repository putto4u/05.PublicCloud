Architecture Core

안전한 인프라의 시작:
기본(Default) VPC와 사용자 정의(Custom) VPC

AWS 계정을 생성하면 각 리전(Region)마다 기본적으로 제공되는 Default VPC(기본 가상 프라이빗 클라우드)가 존재합니다. 그러나 실제 운영 환경에서는 보안과 확장성을 고려하여 Custom VPC(사용자 정의 가상 프라이빗 클라우드)를 직접 구축하는 것이 필수적입니다.

Default VPC가 있는데도 새로 만들어야 하는 이유

보안 설계 (Isolation)

Default VPC는 기본적으로 모든 서브넷이 IGW(Internet Gateway, 인터넷 게이트웨이)와 연결된 Public 서브넷(퍼블릭 하위 망)입니다. 이곳에 데이터베이스(RDS)나 백엔드 서버를 배치하면 인터넷을 통한 외부 해킹 위협에 직접적으로 노출됩니다. Custom VPC를 구축하면 외부에서 절대 접근할 수 없는 Private 서브넷(프라이빗 하위 망)을 구성하여 완벽한 보안 계층(Security Tier)을 분리할 수 있습니다.

IP 대역 관리 및 충돌 방지

실제 기업 환경에서는 클라우드와 온프레미스(IDC)1 네트워크를 연결하거나(VPN), 다른 회사의 VPC와 연결(VPC Peering)하는 하이브리드 구성이 빈번합니다. 모든 AWS 계정의 Default VPC는 동일한 172.31.0.0/16 대역을 사용하므로, 네트워크를 연결하는 순간 IP 충돌(IP Collision)이 발생하여 통신이 완전히 불가능해집니다.

인프라 관리의 명확성 (IaC 도입)

테라폼(Terraform)과 같은 IaC(Infrastructure as Code, 코드로서의 인프라)2 도구를 사용할 때, 이미 존재하는 리소스를 임포트(Import)하여 쓰는 것보다 네트워크의 기초(Base)부터 코드로 직접 정의하는 것이 관리 효율성과 재현성 면에서 훨씬 유리합니다. 특정 서비스 전용 네트워크 환경을 코드로 관리하면 생성, 변경, 삭제의 라이프사이클을 완벽하게 통제할 수 있습니다.

실습 시나리오: 언제 무엇을 쓰나?

Default VPC를 쓰는 경우

  • 단순 학습용으로 EC2(가상 서버) 한 대만 빠르게 띄워보고 싶을 때
  • 네트워크 구조나 보안에 신경 쓰지 않고 애플리케이션의 단순 동작 테스트만 수행할 때

Custom VPC를 만드는 경우 (권장)

  • 강의 및 실무 환경 구축 시 (Public + Private 표준 아키텍처 학습)
  • Amazon EKS(쿠버네티스 서비스) 클러스터를 운영할 때 (워커 노드 보호 필수)
  • ISMS(정보보호 관리체계) 등 보안 인증이 필요한 상용 서비스를 배포할 때

AWS 비용 주의사항 (VPC 구축 시)

VPC 자체를 생성하는 것은 무료입니다. 그러나 Custom VPC를 구성할 때 추가되는 다음 컴포넌트들은 비용을 발생시킵니다.

NAT Gateway (네트워크 주소 변환 게이트웨이)

Custom VPC에서 Private 서브넷 내부의 서버가 인터넷으로 나가기 위해 NAT Gateway를 생성하면 생성된 시간당 약 $0.059 (서울 리전 기준)의 고정 비용과 데이터 처리 비용이 지속적으로 발생합니다. 실습 후 반드시 삭제해야 합니다.

VPN / Direct Connect (전용선)

외부 온프레미스 네트워크와 클라우드 VPC를 안전하게 연결하기 위해 Site-to-Site VPN이나 전용선을 구성할 경우 연결 시간 및 데이터 전송량에 따른 추가 비용이 발생합니다.

"1:1 환경이라도 기본 VPC는 연습용 장난감이고,
Custom VPC는 실제 집을 짓는 기초 공사입니다."

네트워크 흐름과 보안 아키텍처를 정확히 이해하기 위해서는, 처음부터 사용자 정의 VPC를 직접 설계하고 만들어보는 과정이 반드시 선행되어야 합니다.

주석 및 용어 설명

  1. IDC (Internet Data Center) / 온프레미스(On-premises): 기업이 자체적으로 물리적 서버와 네트워크 장비를 구축하고 운영하는 전통적인 전산실 환경을 의미합니다.
  2. IaC (Infrastructure as Code, 코드로서의 인프라): 수동으로 관리 콘솔을 클릭하여 리소스를 생성하는 대신, 코드를 작성하여 인프라를 자동으로 배포하고 관리하는 현대적인 운영 방식입니다. (예: Terraform, AWS CloudFormation)
  3. VPC Peering (VPC 피어링): 서로 다른 두 개의 VPC 간에 트래픽을 라우팅할 수 있도록 연결하여, 마치 동일한 네트워크에 있는 것처럼 통신하게 해주는 네트워킹 기능입니다.
