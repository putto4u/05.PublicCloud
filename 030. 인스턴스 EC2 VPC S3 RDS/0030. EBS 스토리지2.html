Storage Services

데이터의 안전한 저장소:
Amazon EBS 및 스냅샷 실무 가이드

Amazon EBS(Elastic Block Store, 아마존 엘라스틱 블록 스토어)는 Amazon EC2(Elastic Compute Cloud, 아마존 엘라스틱 컴퓨트 클라우드) 인스턴스에 사용할 수 있는 고성능 블록 스토리지 볼륨입니다. PC의 SSD처럼 운영 체제를 설치하거나 데이터베이스를 실행하는 용도로 사용됩니다.

EC2 인스턴스를 중지하거나 종료하더라도 EBS 볼륨은 데이터를 안전하게 유지하도록 설정할 수 있습니다. 컴퓨팅과 스토리지가 분리되어 유연합니다.

네트워크 연결 방식

물리적 직접 연결이 아닌 AWS 내부 전용 네트워크를 사용합니다. EBS-optimized instances(EBS 최적화 인스턴스)를 사용하여 전용 대역폭을 확보하는 것이 실무 성능 확보의 핵심입니다.

고가용성 (High Availability)

데이터는 AZ(Availability Zone, 가용 영역)1 내에서 자동 복제되어 장애를 방어합니다. 단, 특정 AZ에 종속되므로 타 AZ 접근 시 스냅샷 복사가 필요합니다.

백업 및 가용성 전략: EBS 스냅샷 (Snapshots)

EBS 스냅샷은 특정 시점의 볼륨 상태를 내구성이 뛰어난 Amazon S3(Simple Storage Service, 아마존 심플 스토리지 서비스)에 백업하는 핵심 기능입니다.

DLM 활용

DLM(Data Lifecycle Manager, 데이터 수명 주기 관리자)을 통해 스냅샷 생성을 자동화하고 '7일 보관 후 삭제'와 같은 정책을 운영합니다.

FSR 보장

FSR(Fast Snapshot Restore, 빠른 스냅샷 복구)로 복구 시 발생하는 초기 지연 시간(Lazy Loading)을 제거하여 긴급 복구 성능을 보장합니다.

AMI 생성

안전한 스냅샷을 기반으로 AMI(Amazon Machine Image, 아마존 머신 이미지)를 생성하여 동일 설정의 서버를 대량으로 복제(Scale-out)합니다.

4

실무 검토 사례 (Case Study)

검토 항목 실제 사례 상황 적용된 실무 기준 및 해결책
비용 절감 월 $5,000의 기존 gp2 볼륨 비용 발생 최신 gp3로 무중단 전환하여 성능을 유지하면서 비용 20%($1,000) 즉시 절감
성능 장애 DB 응답 속도가 간헐적으로 심각하게 느려짐 CloudWatch(클라우드워치)로 BurstBalance(버스트 잔량) 모니터링. 크레딧 고갈 확인 후 gp3 전환 및 IOPS 추가 할당으로 해결
데이터 보안 금융권 컴플라이언스(규제 준수) 대응 필요 KMS(Key Management Service, 키 관리 서비스)를 통한 EBS 볼륨 암호화 적용 및 신규 생성 시 정책 강제화
재해 복구 (DR) 특정 리전(Region, 물리적 데이터 센터 군) 장애 대비 DLM 정책을 활용하여 주기적인 스냅샷을 타 리전(Cross-Region)으로 자동 복사하여 백업 이중화

핵심 주의사항 및 체크리스트

1. 미사용 볼륨 관리

EC2 인스턴스가 종료되어도 분리된 EBS는 남아 지속적으로 과금될 수 있습니다. (Zombie Volumes). 반드시 스냅샷 생성 후 즉시 삭제하십시오.

2. 종료 시 삭제 옵션

중요 데이터가 담긴 볼륨은 Delete on Termination(종료 시 삭제) 옵션이 활성화되어 서버 삭제 시 실수로 함께 날아가지 않도록 주의해야 합니다.

3. 볼륨 크기 조정 제약

Elastic Volumes(탄력적 볼륨) 기능을 통해 서비스 중단 없이 크기를 키우거나 유형 변경(gp2 → gp3)이 가능합니다. 단, 용량을 줄이는 것(축소)은 불가능하므로 초기 설계 시 유의하십시오.

과금 발생 주의 사항 (필독)

  • 인스턴스 중지(Stop) 상태 과금 EC2 인스턴스를 중지해 두면 컴퓨팅(CPU/RAM) 비용은 발생하지 않지만, 인스턴스에 부착된 EBS 스토리지 용량에 대한 비용은 계속 청구됩니다. 스토리지 공간을 점유하고 있기 때문입니다. 완전히 삭제하지 않는 한 스토리지 요금은 매월 부과됩니다.
  • Provisioned IOPS (io2) 과금 io2 볼륨 등에서 높은 성능을 위해 설정한 IOPS 값은 실제 해당 성능을 모두 사용했는지의 여부와 관계없이 사용자가 설정(예약)한 수치만큼 고정적으로 과금됩니다. 오버프로비저닝(불필요하게 높게 설정)하지 않도록 주의하십시오.

주석 및 용어 설명

  1. AZ (Availability Zone, 가용 영역): 하나의 리전(Region) 내에 존재하는 물리적으로 독립된 데이터 센터의 묶음입니다. 화재, 단전 등에 대비하여 독립적인 전력과 네트워킹을 갖추고 있습니다.
  2. IOPS (Input/Output Operations Per Second, 초당 입출력 횟수): 디스크가 초당 처리할 수 있는 데이터 입출력 횟수의 척도입니다. 작은 파일들을 빠르게 수없이 읽고 써야 하는 데이터베이스에 중요한 지표입니다.
  3. Throughput (처리량, 초당 데이터 전송량): 초당 전송되는 데이터의 양(MB/s)을 의미합니다. 대용량 동영상 처리, 빅데이터 연속 로드와 같이 한 번에 많은 양의 데이터를 옮길 때 중요한 지표입니다.
