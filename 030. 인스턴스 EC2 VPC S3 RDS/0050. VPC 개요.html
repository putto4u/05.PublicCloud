Networking & Content Delivery

안전한 클라우드 네트워크의 기반:
Amazon VPC 아키텍처 이해

Amazon VPC(Virtual Private Cloud, 가상 프라이빗 클라우드)는 AWS 클라우드 내에서 사용자 계정 전용으로 논리적으로 격리된 가상 네트워크 섹션을 할당하는 서비스입니다. 온프레미스(On-premises)1 환경의 전통적인 네트워크를 클라우드 환경에서 완벽하게 제어할 수 있도록 가상화하여 제공합니다.

VPC의 핵심 가치와 정의

VPC는 단순한 망 분리가 아닙니다. 클라우드 아키텍처의 보안과 가용성을 결정짓는 가장 중요한 기반 공사입니다. VPC를 통해 IP 주소 범위 선택, 서브넷(Subnet, 하위 망) 생성, 라우팅 테이블(Routing Table, 경로 테이블) 구성 등 네트워크 환경에 대한 완벽한 제어권을 가질 수 있습니다.

논리적 격리성 및 보안

다른 AWS 고객의 네트워크와 완전히 분리되어 독립적인 주소 공간과 엄격한 보안 정책을 적용받습니다. 외부의 허가되지 않은 접근을 원천 차단합니다.

높은 네트워크 제어권

사용자가 직접 서브넷을 설계하고 게이트웨이를 연결하며, 라우팅 규칙을 설정하여 트래픽의 흐름을 100% 설계하고 통제할 수 있습니다.

비용 안내: VPC 자체를 생성하고 사용하는 것은 무료입니다. 단, 외부로 나가는 데이터 전송료 및 NAT Gateway, Elastic IP(탄력적 IP) 등 부가적인 구성 요소에 대해서는 비용이 발생할 수 있습니다.

VPC 핵심 구성 요소: 주소와 서브넷

CIDR (Classless Inter-Domain Routing, 클래스 없는 도메인 간 라우팅)

VPC 내부에서 사용할 IP 주소의 범위를 정의하는 방식입니다. 클라우드 인프라의 '영토 크기'를 결정합니다.

표기 예시: 10.0.0.0/16

(/16은 약 65,536개의 IP 주소를 사용할 수 있는 넉넉한 공간을 의미합니다.)

서브넷 (Subnet, 하위 망)

VPC의 큰 영토(CIDR)를 목적에 맞게 더 작게 쪼갠 구역입니다. 외부 인터넷과의 통신 여부에 따라 두 가지로 나뉩니다.

Public Subnet (퍼블릭 서브넷)

인터넷 게이트웨이(Internet Gateway)를 통해 인터넷과 직접 양방향 통신이 가능한 영역입니다. 외부 클라이언트의 접속을 직접 받아야 하는 자원을 배치합니다.

주요 배치: 웹 서버(Web Server), 로드 밸런서(Load Balancer), 배스천 호스트(Bastion Host)

Private Subnet (프라이빗 서브넷)

외부 인터넷과 직접 연결되지 않는 완전히 폐쇄된 보안 영역입니다. 해킹의 위험으로부터 중요한 데이터를 안전하게 격리하고 보호하기 위해 사용합니다.

주요 배치: 데이터베이스(Database), 백엔드 애플리케이션(Backend App) 서버

실무 아키텍처 팁

하나의 서브넷은 반드시 하나의 AZ(Availability Zone, 가용 영역) 내에만 존재해야 합니다. 따라서 고가용성(High Availability)을 달성하기 위해서는 2개 이상의 서로 다른 AZ에 서브넷을 분산 배치하는 것이 실무 설계의 기본입니다.

VPC 연결 및 게이트웨이

IGW

IGW (Internet Gateway, 인터넷 게이트웨이)

VPC 내부의 리소스와 외부 인터넷 간의 통신을 가능하게 하는 수평 확장형 논리적 리소스입니다. 퍼블릭 서브넷이 외부로 나가는 유일한 관문 역할을 수행합니다.

RT

Route Table (라우팅 테이블, 경로 테이블)

네트워크 트래픽이 목적지 IP 주소에 따라 어디로 전달되어야 하는지 정의한 이정표(규칙 모음)입니다. 각 서브넷은 반드시 하나의 라우팅 테이블과 연결되어야 트래픽이 길을 찾을 수 있습니다.

NAT

NAT Gateway (네트워크 주소 변환 게이트웨이)

유료 서비스 과금 주의

프라이빗 서브넷에 있는 인스턴스가 보안 패치 다운로드 등을 위해 인터넷으로 나갈 수 있도록(Outbound) 돕는 서비스입니다. 동시에 외부 인터넷에서 프라이빗 서브넷 안으로 들어오는 직접적인 연결(Inbound)은 완벽히 차단하여 보안을 유지합니다.

과금 발생 주의: NAT Gateway는 생성된 시간당 고정 유지 비용과, 처리하는 데이터 처리량(GB당)에 대한 요금이 이중으로 발생합니다. 실습용으로 생성한 후 삭제하지 않으면 계속 요금이 청구되므로 사용하지 않을 때는 즉시 삭제해야 합니다.

VPC 이중 보안 체계 (SG vs NACL)

VPC는 인스턴스 수준과 서브넷 수준에서 방화벽 역할을 하는 두 가지 보안 계층을 제공하여 심층 방어(Defense in Depth)를 구현합니다.

비교 항목 SG (Security Group, 보안 그룹) NACL (Network Access Control List)
적용 범위 인스턴스(Instance) 개별 단위 적용 (1차 방어선) 서브넷(Subnet) 전체 단위 적용 (2차 방어선)
상태 유지 여부 Stateful (상태 유지)
들어오는 요청이 허용되면, 나가는 응답은 설정과 무관하게 자동으로 허용됨. 		Stateless (상태 비저장)
들어오는 요청을 허용했더라도, 나가는 응답 경로를 명시적으로 개방하지 않으면 차단됨.
규칙 적용 방식 허용(Allow) 규칙만 지정 가능. (지정되지 않은 모든 트래픽은 기본적으로 거부됨) 허용(Allow) 및 거부(Deny) 규칙 모두 명시적 지정 가능. (특정 악성 IP 차단에 유용)

VPC 설계 베스트 프랙티스 (Best Practices)

최소 권한 원칙 준수

보안 그룹 설정 시 `0.0.0.0/0`(모든 IP 허용)의 사용을 극도로 제한하고, 필요한 포트와 IP 대역만 정밀하게 개방하여 공격 표면을 최소화하십시오.

고가용성 확보

단일 데이터 센터 장애에 대비하기 위해, 서브넷과 인스턴스를 최소 2개 이상의 서로 다른 AZ(가용 영역)에 분산 구축하여 시스템의 내결함성(Fault Tolerance)을 확보하십시오.

미래를 고려한 IP 설계

VPC의 CIDR 블록은 한 번 생성하면 크기 변경이 불가능합니다. 향후 컨테이너 도입이나 서비스 확장을 고려하여 초기 설계 시 IP 범위를 넉넉하게(`/16` 등) 할당하십시오.

주석 및 용어 설명

  1. 온프레미스 (On-premises): 기업이 자체적으로 데이터 센터를 구축하고 서버, 스토리지, 네트워크 장비 등의 IT 인프라를 물리적으로 직접 소유하고 운영하는 전통적인 환경을 의미합니다.
