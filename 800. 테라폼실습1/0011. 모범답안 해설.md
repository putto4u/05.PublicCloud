### 1. Service VPC 구성 계층 (`02-service-vpc`)

Service VPC (Virtual Private Cloud, 가상 사설 클라우드)는 허브앤스포크 토폴로지에서 SPOKE에 해당되는 VPC로서 실제 애플리케이션(Nginx)이 구동되는 격리된 네트워크 환경입니다. 보안을 위해 퍼블릭 서브넷을 두지 않고 통신을 TGW (Transit Gateway, 전송 게이트웨이)로 일원화하는 것이 핵심입니다.

#### 1.1 `terragrunt.hcl` (의존성 및 변수 주입)

```hcl
include "root" {
  path = find_in_parent_folders() # [1] 공통 설정 상속: 부모 폴더의 env.hcl 등 공통 설정을 가져옵니다.
}

terraform { source = "." }        # [2] 현재 디렉터리의 main.tf를 실행합니다.

inputs = {                                                        
  vpc_name = "service-vpc"
  vpc_cidr = "10.1.0.0/16"        # [3] 독립된 네트워크 대역: Hub VPC(10.0.0.0/16)와 겹치지 않게 설정합니다.
}

```

#### 1.2 `main.tf` (리소스 정의)

```hcl
variable "vpc_name" { type = string }
variable "vpc_cidr" { type = string }

module "service_vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "5.0.0"

  name = var.vpc_name
  cidr = var.vpc_cidr

  azs             = ["ap-northeast-2a", "ap-northeast-2c"]
  private_subnets = ["10.1.1.0/24", "10.1.2.0/24"] # [1] 프라이빗 전용 구성: 외부 접근을 차단하기 위해 public_subnets는 제외합니다.

  enable_nat_gateway = false # [2] NAT 게이트웨이 비활성화: 통신은 Hub VPC의 NAT를 경유하므로 여기서는 생성하지 않아 과금을 막습니다.
}

# [3] 상호 참조용 출력: 다른 모듈(TGW 등)에서 이 VPC의 정보를 참조할 수 있도록 출력값을 정의합니다.
output "vpc_id" { value = module.service_vpc.vpc_id }
output "private_subnets" { value = module.service_vpc.private_subnets }
output "private_route_table_ids" { value = module.service_vpc.private_route_table_ids }

```

### 2. Transit Gateway 라우팅 계층 (`03-tgw`)

각기 다른 VPC를 하나의 거대한 라우터로 묶어주는 아키텍처의 중심축입니다. **(주의: TGW는 연결된 VPC 개수와 처리된 트래픽당 비용이 발생하는 유료 서비스입니다.)**

#### 2.1 `terragrunt.hcl` (의존성 및 Mock 설정)

```hcl
include "root" { path = find_in_parent_folders() }

dependency "hub" {
  config_path = "../01-hub-vpc"
  mock_outputs = { # [1] Mock 데이터: init/plan 단계에서 실제 리소스가 생성되기 전 임시로 사용할 가짜 값입니다.
    vpc_id                  = "vpc-mock-12345"
    private_subnets         = ["subnet-mock-1", "subnet-mock-2"]
    public_subnets          = ["subnet-mock-pub1", "subnet-mock-pub2"]
    private_route_table_ids = ["rtb-mock-1", "rtb-mock-2"]
    public_route_table_ids  = ["rtb-mock-pub1", "rtb-mock-pub2"]
    nat_gateway_ids         = ["nat-12345"] # [2] 이전 단계에서 충돌 수정을 위해 Hub 블록으로 이동시킨 값입니다.
  }
  mock_outputs_allowed_terraform_commands = ["init","plan", "validate"]  ## 여기서 init이 빠지면 최초 실행시 에러 발생. 
}

dependency "service" {
  config_path = "../02-service-vpc"
  mock_outputs = {
    vpc_id                  = "vpc-mock-67890"
    private_subnets         = ["subnet-mock-3", "subnet-mock-4"]
    private_route_table_ids = ["rtb-mock-3", "rtb-mock-4"]
  }
  mock_outputs_allowed_terraform_commands = ["init","plan", "validate"]
}

terraform { source = "." }

inputs = { # [3] 의존성 모듈의 출력값을 현재 모듈의 입력 변수로 주입합니다.
  hub_vpc_id                 = dependency.hub.outputs.vpc_id                  ### . . . 은 변수의 경로와 같은 역할을 합니다.
  hub_subnet_ids             = dependency.hub.outputs.private_subnets
  service_vpc_id             = dependency.service.outputs.vpc_id
  service_subnet_ids         = dependency.service.outputs.private_subnets
  hub_route_table_ids        = dependency.hub.outputs.private_route_table_ids
  hub_public_route_table_ids = dependency.hub.outputs.public_route_table_ids
  service_route_table_ids    = dependency.service.outputs.private_route_table_ids
  nat_gateway_ids            = dependency.hub.outputs.nat_gateway_ids
}

```

#### 2.2 `main.tf` (리소스 정의)

```hcl
variable "hub_vpc_id" { type = string }
variable "hub_public_route_table_ids" { type = list(string) }
variable "hub_subnet_ids" { type = list(string) }
variable "service_vpc_id" { type = string }
variable "service_subnet_ids" { type = list(string) }
variable "hub_route_table_ids" { type = list(string) }
variable "service_route_table_ids" { type = list(string) }
variable "nat_gateway_ids" { type = list(string) }

# TGW 생성 (유료 서비스)
resource "aws_ec2_transit_gateway" "main" {
  description = "Hub and Spoke TGW"
  tags = { Name = "main-tgw" }
}

# Hub VPC 연결: TGW가 Hub VPC와 통신할 수 있도록 Attachment를 생성합니다.
resource "aws_ec2_transit_gateway_vpc_attachment" "hub" {
  subnet_ids         = var.hub_subnet_ids
  transit_gateway_id = aws_ec2_transit_gateway.main.id
  vpc_id             = var.hub_vpc_id
  tags = { Name = "hub-attachment" }
}

# Service VPC 연결
resource "aws_ec2_transit_gateway_vpc_attachment" "service" {
  subnet_ids         = var.service_subnet_ids
  transit_gateway_id = aws_ec2_transit_gateway.main.id
  vpc_id             = var.service_vpc_id
  tags = { Name = "service-attachment" }
}

# [1] Service VPC -> TGW 라우팅: Service VPC에서 발생하는 외부 통신(0.0.0.0/0)을 전부 TGW로 던집니다.
resource "aws_route" "service_to_tgw" {
  count                  = length(var.service_route_table_ids)
  route_table_id         = var.service_route_table_ids[count.index]
  destination_cidr_block = "0.0.0.0/0"
  transit_gateway_id     = aws_ec2_transit_gateway.main.id
  depends_on             = [aws_ec2_transit_gateway_vpc_attachment.service]
}

# [2] Hub VPC (Private) -> Service VPC 라우팅: Hub VPC의 프라이빗 영역에서 Service VPC(10.1.0.0/16)를 찾을 때 TGW를 거치도록 설정합니다.
resource "aws_route" "hub_private_to_service" {
  count                  = length(var.hub_route_table_ids)
  route_table_id         = var.hub_route_table_ids[count.index]
  destination_cidr_block = "10.1.0.0/16"
  transit_gateway_id     = aws_ec2_transit_gateway.main.id
  depends_on             = [aws_ec2_transit_gateway_vpc_attachment.hub]
}

# [3] Hub VPC (Public) -> Service VPC 라우팅: ALB가 위치한 퍼블릭 영역에서 Nginx가 있는 Service VPC로 트래픽을 보낼 때의 경로입니다.
resource "aws_route" "hub_pub_to_service" {
  count                  = length(var.hub_public_route_table_ids)
  route_table_id         = var.hub_public_route_table_ids[count.index]
  destination_cidr_block = "10.1.0.0/16"
  transit_gateway_id     = aws_ec2_transit_gateway.main.id
  depends_on             = [aws_ec2_transit_gateway_vpc_attachment.hub]
}

# [4] TGW 기본 경로 설정: TGW 내부로 들어온 트래픽이 목적지를 모를 경우(0.0.0.0/0), 무조건 Hub VPC로 전송합니다. (이후 NAT를 거침)
resource "aws_ec2_transit_gateway_route" "tgw_default_route" {
  destination_cidr_block         = "0.0.0.0/0"
  transit_gateway_route_table_id = aws_ec2_transit_gateway.main.association_default_route_table_id
  transit_gateway_attachment_id  = aws_ec2_transit_gateway_vpc_attachment.hub.id
  depends_on                     = [aws_ec2_transit_gateway_vpc_attachment.hub]
}

# [5] 출력값: 다른 모듈에서 참조할 TGW ID 등을 내보냅니다. 주석 처리된 부분은 이전 에러 해결 시 제외한 라인입니다.
output "tgw_final_info" {
  value = {
    tgw_id            = aws_ec2_transit_gateway.main.id
    tgw_route_table   = aws_ec2_transit_gateway.main.association_default_route_table_id
    hub_attachment_id = aws_ec2_transit_gateway_vpc_attachment.hub.id

  }
}

```

### 3. Application Load Balancer 구성 계층 (`04-alb`)

클라이언트의 요청을 가장 먼저 받아 처리하는 관문입니다. **(주의: ALB는 가동 시간 및 트래픽 용량에 따라 과금됩니다.)**

#### 3.1 `terragrunt.hcl` (의존성 설정)

```hcl
include "root" { path = find_in_parent_folders() }

dependency "hub" { # [1] ALB는 Hub VPC의 퍼블릭 서브넷에 배치되어야 하므로 Hub 모듈에만 의존성을 가집니다.
  config_path = "../01-hub-vpc"
  mock_outputs = {
    vpc_id         = "vpc-mock-12345"
    public_subnets = ["subnet-mock-pub1", "subnet-mock-pub2"]
  }
  mock_outputs_allowed_terraform_commands = ["init","plan", "validate"]
}

terraform { source = "." }

inputs = {
  vpc_id         = dependency.hub.outputs.vpc_id
  public_subnets = dependency.hub.outputs.public_subnets
}

```

#### 3.2 `main.tf` (리소스 정의)

```hcl
variable "vpc_id" { type = string }
variable "public_subnets" { type = list(string) }

# ALB 보안 그룹: 외부(0.0.0.0/0)의 웹 요청(80)을 받고, TGW를 거쳐 Service VPC(10.1.0.0/16)로 전달만 하도록 제어합니다.
resource "aws_security_group" "alb_sg" {
  name        = "hub-alb-sg"
  description = "ALB Security Group"
  vpc_id      = var.vpc_id

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["10.1.0.0/16"] # [1] Service VPC 방향으로만 트래픽 유출 허용
  }
}

module "alb" {
  source  = "terraform-aws-modules/alb/aws"
  version = "~> 9.0" # [2] 9.x 최신 버전의 모듈 문법을 사용합니다.

  name               = "hub-central-alb"
  load_balancer_type = "application"
  vpc_id             = var.vpc_id
  subnets            = var.public_subnets
  security_groups    = [aws_security_group.alb_sg.id]

  listeners = { # [3] 리스너 구조: 80 포트로 들어온 요청을 nginx_tg 타겟 그룹으로 전달(Forward)합니다.
    http = {
      port     = 80
      protocol = "HTTP"
      forward = {
        target_group_key = "nginx_tg" 
      }
    }
  }
  
  target_groups = { # [4] 타겟 그룹 구조: IP 주소 기반(target_type = "ip")으로 트래픽을 분산시킬 대상을 정의합니다.
    nginx_tg = {
      name_prefix      = "ngnx-" 
      protocol         = "HTTP"
      port             = 80
      target_type      = "ip" # [5] VPC 경계를 넘어가기 위해 반드시 IP 타입으로 지정해야 합니다.
      create_attachment = false # [6] EC2 모듈에서 별도로 연결할 것이므로 여기서 자동 생성은 막습니다.
    }
  }
}

# [7] 모듈 9.x 버전에 맞춘 출력값: Map 구조에서 특정 ARN과 DNS 이름을 추출하여 제공합니다.
output "target_group_arns" {
  value = [module.alb.target_groups["nginx_tg"].arn]
}

output "alb_dns_name" {
  value = module.alb.dns_name
}
```

```hcl
    cidr_blocks = ["0.0.0.0/0"] # OS 업데이트 및 패키지 설치용 외부 통신 허용

# --- SSM (Systems Manager) 접속 권한 설정 ---
# [2] 프라이빗 서브넷에 위치하여 SSH 접속이 불가능한 EC2에 브라우저 콘솔로 안전하게 접속하기 위한 IAM 설정입니다.

  })

resource "aws_iam_instance_profile" "ssm_profile" {

  tags = { Name = "nginx-service-instance" }

  port             = 80
```
