        <article class="glass-panel rounded-2xl p-8 mb-10">
            <h2 class="text-3xl font-bold mb-6 border-b pb-4">단일 VPN 게이트웨이 다중 클러스터 연결 및 라우팅 전략</h2>
            <p class="mb-4">온프레미스(On-Premises, 자체 보유 서버 및 네트워크 환경)에 단일 VPN 게이트웨이(VPN Gateway, 가상 사설망 연결 장비 - 예: strongSwan 등)를 구축하고, 내부 네트워크의 여러 PC가 독립적인 여러 AWS 환경의 K8s(Kubernetes, 컨테이너 오케스트레이션 시스템) 클러스터에 접속하기 위해서는 고도화된 네트워크 라우팅(Routing, 데이터 패킷의 경로 설정) 설계가 필요합니다.</p>
            <p class="mb-6 font-semibold text-slate-700">결론적으로 각 PC의 기본 게이트웨이(Default Gateway, 외부 네트워크로 나가는 기본 출입구)를 변경할 필요는 없으며, 다음 방식을 통해 트래픽을 제어해야 합니다.</p>

            <div class="grid md:grid-cols-2 gap-6 mb-8">
                <div class="bg-white p-6 rounded-xl border border-slate-200 shadow-sm">
                    <h3 class="text-xl font-bold text-blue-700 mb-3">옵션 A: 메인 라우터 정적 라우트 (권장)</h3>
                    <p class="text-sm text-slate-600 mb-3">내부 네트워크의 L3 스위치(Layer 3 Switch, 라우팅 기능이 포함된 네트워크 스위치) 라우팅 테이블(Routing Table, 경로 지정 표)에 AWS 클라우드 대역으로 가는 패킷의 다음 경로(Next Hop)를 온프레미스 VPN 서버로 지정합니다.</p>
                    <ul class="text-sm list-disc pl-5 text-slate-700">
                        <li>개별 PC 설정 불필요, 중앙 집중식 관리.</li>
                        <li>타겟 대역 전체(예: <code>10.0.0.0/8</code>) 포워딩.</li>
                    </ul>
                </div>
                <div class="bg-white p-6 rounded-xl border border-slate-200 shadow-sm">
                    <h3 class="text-xl font-bold text-teal-700 mb-3">옵션 B: 각 PC 정적 라우트 개별 추가</h3>
                    <p class="text-sm text-slate-600 mb-3">메인 장비 제어가 불가능할 경우, 각 PC의 OS(Operating System, 운영체제)에 목적지 대역으로 가는 경로만 VPN으로 지정하는 분할 터널링(Split Tunneling)을 구현합니다.</p>
                    <div class="bg-slate-800 text-slate-200 p-3 rounded-lg text-xs font-mono mb-2 overflow-x-auto">
                        C:\> route add 10.1.0.0 mask 255.255.0.0 &lt;VPN_IP&gt; -p
                    </div>
                </div>
            </div>
            
            <p class="text-sm text-slate-500 italic">* 기본 게이트웨이를 VPN으로 통째로 변경하는 방식(옵션 C)은 일반 인터넷 트래픽까지 터널을 거치게 되어 대역폭 저하 및 NAT(Network Address Translation, 네트워크 주소 변환) 구성 복잡성을 초래하므로 지양합니다.</p>

        <!-- Section: CIDR & Octet Design -->
        <article class="glass-panel rounded-2xl p-8 mb-10">
            <h2 class="text-3xl font-bold mb-6 border-b pb-4">CIDR 설계 및 네트워크 대역 중복(Routing Conflict) 방지</h2>
            
            <p class="mb-4">다수의 접속 환경이 동일한 <code>10.0.0.0/16</code> 대역의 VPC(Virtual Private Cloud, 가상 사설 클라우드)를 사용할 경우, 중앙 VPN 라우터는 패킷의 목적지를 식별할 수 없어 치명적인 라우팅 충돌(Routing Conflict)이 발생합니다. <span class="highlight">서브넷 크기(/16)가 아닌 주소 체계(Network ID) 자체가 달라야 합니다.</span></p>

            <div class="bg-slate-50 p-6 rounded-xl mb-6 border border-slate-200">
                <h3 class="text-lg font-bold text-slate-800 mb-3">옥텟(Octet) 분할 아키텍처 원리</h3>
                <p class="text-sm mb-4">IPv4(Internet Protocol version 4) 주소 체계에서 두 번째 옥텟(8비트 데이터 블록)을 사용자별로 고유하게 할당하여 완벽한 격리를 구현합니다.</p>
                <ul class="list-none space-y-2 text-sm">
                    <li class="flex items-center"><span class="w-2 h-2 rounded-full bg-blue-500 mr-3"></span><strong>환경 A:</strong> <code>10.1.0.0/16</code> (사용 범위: 10.1.0.0 ~ 10.1.255.255)</li>
                    <li class="flex items-center"><span class="w-2 h-2 rounded-full bg-teal-500 mr-3"></span><strong>환경 B:</strong> <code>10.2.0.0/16</code> (사용 범위: 10.2.0.0 ~ 10.2.255.255)</li>
                    <li class="flex items-center"><span class="w-2 h-2 rounded-full bg-indigo-500 mr-3"></span><strong>환경 C:</strong> <code>10.3.0.0/16</code> (사용 범위: 10.3.0.0 ~ 10.3.255.255)</li>
                </ul>
            </div>

            <div class="tip-box p-5 rounded-lg mb-6 shadow-sm">
                <h4 class="font-bold text-blue-800 mb-2 flex items-center">
                    <svg class="w-5 h-5 mr-2" fill="none" stroke="currentColor" viewBox="0 0 24 24" xmlns="http://www.w3.org/2000/svg"><path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M13 16h-1v-4h-1m1-4h.01M21 12a9 9 0 11-18 0 9 9 0 0118 0z"></path></svg>
                    실전 팁 및 자주 오해하는 부분
                </h4>
                <ul class="text-sm list-disc pl-5 space-y-2 text-slate-700">
                    <li><strong>기본(Default) VPC 사용 금지:</strong> AWS 콘솔이 제공하는 <code>172.31.0.0/16</code> 기본 대역을 그대로 사용하면 모든 환경이 중복됩니다. 반드시 고유 CIDR(Classless Inter-Domain Routing, IP 주소 할당을 위한 블록 지정 방식)을 지닌 새 VPC를 생성해야 합니다.</li>
                    <li><strong>EKS CNI 대역 충돌:</strong> AWS EKS는 VPC CNI(Container Network Interface, 컨테이너 네트워크 인터페이스)를 사용하므로 파드(Pod) IP가 VPC 실제 서브넷 IP와 동일합니다. 온프레미스 망(예: <code>192.168.x.x</code>)과 클라우드 대역이 겹치지 않는지 교차 검증해야 합니다.</li>
                </ul>
            </div>

        <!-- Section: Windows VPN Setup -->
        <article class="glass-panel rounded-2xl p-8 mb-10">
            <h2 class="text-3xl font-bold mb-6 border-b pb-4">Windows 환경 strongSwan (IKEv2) 클라이언트 구축 가이드</h2>
            <p class="mb-4">최신 Windows(윈도우)는 내장된 Agile VPN(애자일 VPN, 마이크로소프트의 내장 네트워크 모듈)을 통해 strongSwan의 핵심인 IKEv2(Internet Key Exchange version 2) 프로토콜을 네이티브로 지원합니다. 별도의 프로그램 설치 없이 OS 내장 기능만으로 엔터프라이즈급 암호화 터널을 구성할 수 있습니다.</p>

            <div class="space-y-6">
                <!-- Step 1 -->
                <div class="flex items-start">
                    <div class="flex-shrink-0 flex items-center justify-center w-8 h-8 rounded-full bg-blue-600 text-white font-bold mr-4">1</div>
                    <div>
                        <h4 class="text-lg font-bold text-slate-800 mb-2">보안 인증서(Certificate) 등록</h4>
                        <p class="text-sm text-slate-600 mb-2">PC가 서버를 신뢰하도록 Root CA(Root Certificate Authority, 최상위 인증 기관) 인증서를 등록합니다.</p>
                        <ul class="text-sm list-disc pl-5 text-slate-700 bg-slate-50 p-4 rounded border border-slate-200">
                            <li><code>ca-cert.pem</code> 파일을 실행하여 <strong>'인증서 설치'</strong> 선택</li>
                            <li>저장소 위치: <span class="font-semibold text-red-600">반드시 '로컬 컴퓨터(Local Machine)' 선택</span> (관리자 권한 필요)</li>
                            <li>저장소: <strong>'신뢰할 수 있는 루트 인증 기관(Trusted Root Certification Authorities)'</strong> 폴더 지정</li>
                        </ul>
                    </div>
                </div>

                <!-- Step 2 -->
                <div class="flex items-start">
                    <div class="flex-shrink-0 flex items-center justify-center w-8 h-8 rounded-full bg-blue-600 text-white font-bold mr-4">2</div>
                    <div>
                        <h4 class="text-lg font-bold text-slate-800 mb-2">윈도우 내장 VPN 프로필 생성</h4>
                        <p class="text-sm text-slate-600 mb-2">설정 > 네트워크 및 인터넷 > VPN 메뉴에서 프로필을 추가합니다.</p>
                        <div class="text-sm bg-slate-50 p-4 rounded border border-slate-200 grid grid-cols-1 md:grid-cols-2 gap-2">
                            <div><strong>VPN 공급자:</strong> Windows(기본 제공)</div>
                            <div><strong>연결 이름:</strong> AWS EKS VPN (임의)</div>
                            <div><strong>서버 이름/주소:</strong> vpn.domain.com</div>
                            <div><strong>VPN 종류:</strong> <span class="font-semibold text-blue-700">IKEv2</span></div>
                            <div class="md:col-span-2 text-xs text-slate-500 mt-2">* IKEv2는 MOBIKE(이동성 및 다중 홈 프로토콜)를 지원하여 망 전환 시에도 연결을 유지합니다.</div>
                        </div>
                    </div>
                </div>

                <!-- Step 3 -->
                <div class="flex items-start">
                    <div class="flex-shrink-0 flex items-center justify-center w-8 h-8 rounded-full bg-blue-600 text-white font-bold mr-4">3</div>
                    <div>
                        <h4 class="text-lg font-bold text-slate-800 mb-2">고급 어댑터 보안 격상 설정 (필수)</h4>
                        <p class="text-sm text-slate-600 mb-2">제어판의 네트워크 어댑터 속성에서 구형 암호화를 차단하고 강력한 ESP(Encapsulating Security Payload, 캡슐화 보안 페이로드)를 강제합니다.</p>
                        <ul class="text-sm list-disc pl-5 text-slate-700 bg-slate-50 p-4 rounded border border-slate-200">
                            <li><strong>보안 탭 > 데이터 암호화:</strong> <code>최대 강도 암호화(서버에서 거절하면 연결 끊기)</code> 선택</li>
                            <li><strong>보안 탭 > 인증:</strong> <code>EAP(확장명 인증 프로토콜) 사용</code> 선택 후 <code>Microsoft: 보안 암호(EAP-MSCHAP v2)</code> 지정</li>
                        </ul>
                    </div>
                </div>
            </div>

            <div class="tip-box p-5 rounded-lg mt-6 shadow-sm">
                <h4 class="font-bold text-blue-800 mb-2">방화벽 UDP 포트 점검</h4>
                <p class="text-sm text-slate-700">IKEv2는 통신 초기화에 <code>UDP 500</code>, 암호화 통신에 <code>UDP 4500</code> 포트를 사용합니다. PC 설정이 완벽함에도 접속 에러가 난다면 현재 연결된 사내망 또는 공용 Wi-Fi의 방화벽(Firewall)에서 해당 포트를 차단했는지 의심해야 합니다.</p>
            </div>

                    <li><strong>다중 VPN 연결 및 Transit Gateway:</strong> VPC별로 독립적인 Site-to-Site VPN 터널을 맺거나 AWS Transit Gateway(전송 게이트웨이)로 허브 앤 스포크(Hub & Spoke)를 구성할 경우, 연결된 Attachment 개수 및 처리량에 비례하여 유료 과금이 발생합니다.</li>
