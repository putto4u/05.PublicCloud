단일 온프레미스 VPN 기반
다중 AWS EKS 클러스터 연결 아키텍처

초격차 인프라 설계를 위한 라우팅 전략 및 클라이언트 구성 가이드

단일 VPN 게이트웨이 다중 클러스터 연결 및 라우팅 전략

온프레미스(On-Premises, 자체 보유 서버 및 네트워크 환경)에 단일 VPN 게이트웨이(VPN Gateway, 가상 사설망 연결 장비 - 예: strongSwan 등)를 구축하고, 내부 네트워크의 여러 PC가 독립적인 여러 AWS 환경의 K8s(Kubernetes, 컨테이너 오케스트레이션 시스템) 클러스터에 접속하기 위해서는 고도화된 네트워크 라우팅(Routing, 데이터 패킷의 경로 설정) 설계가 필요합니다.

결론적으로 각 PC의 기본 게이트웨이(Default Gateway, 외부 네트워크로 나가는 기본 출입구)를 변경할 필요는 없으며, 다음 방식을 통해 트래픽을 제어해야 합니다.

옵션 A: 메인 라우터 정적 라우트 (권장)

내부 네트워크의 L3 스위치(Layer 3 Switch, 라우팅 기능이 포함된 네트워크 스위치) 라우팅 테이블(Routing Table, 경로 지정 표)에 AWS 클라우드 대역으로 가는 패킷의 다음 경로(Next Hop)를 온프레미스 VPN 서버로 지정합니다.

  • 개별 PC 설정 불필요, 중앙 집중식 관리.
  • 타겟 대역 전체(예: 10.0.0.0/8) 포워딩.

옵션 B: 각 PC 정적 라우트 개별 추가

메인 장비 제어가 불가능할 경우, 각 PC의 OS(Operating System, 운영체제)에 목적지 대역으로 가는 경로만 VPN으로 지정하는 분할 터널링(Split Tunneling)을 구현합니다.

C:\> route add 10.1.0.0 mask 255.255.0.0 <VPN_IP> -p

* 기본 게이트웨이를 VPN으로 통째로 변경하는 방식(옵션 C)은 일반 인터넷 트래픽까지 터널을 거치게 되어 대역폭 저하 및 NAT(Network Address Translation, 네트워크 주소 변환) 구성 복잡성을 초래하므로 지양합니다.

CIDR 설계 및 네트워크 대역 중복(Routing Conflict) 방지

다수의 접속 환경이 동일한 10.0.0.0/16 대역의 VPC(Virtual Private Cloud, 가상 사설 클라우드)를 사용할 경우, 중앙 VPN 라우터는 패킷의 목적지를 식별할 수 없어 치명적인 라우팅 충돌(Routing Conflict)이 발생합니다. 서브넷 크기(/16)가 아닌 주소 체계(Network ID) 자체가 달라야 합니다.

옥텟(Octet) 분할 아키텍처 원리

IPv4(Internet Protocol version 4) 주소 체계에서 두 번째 옥텟(8비트 데이터 블록)을 사용자별로 고유하게 할당하여 완벽한 격리를 구현합니다.

  • 환경 A: 10.1.0.0/16 (사용 범위: 10.1.0.0 ~ 10.1.255.255)
  • 환경 B: 10.2.0.0/16 (사용 범위: 10.2.0.0 ~ 10.2.255.255)
  • 환경 C: 10.3.0.0/16 (사용 범위: 10.3.0.0 ~ 10.3.255.255)

실전 팁 및 자주 오해하는 부분

  • 기본(Default) VPC 사용 금지: AWS 콘솔이 제공하는 172.31.0.0/16 기본 대역을 그대로 사용하면 모든 환경이 중복됩니다. 반드시 고유 CIDR(Classless Inter-Domain Routing, IP 주소 할당을 위한 블록 지정 방식)을 지닌 새 VPC를 생성해야 합니다.
  • EKS CNI 대역 충돌: AWS EKS는 VPC CNI(Container Network Interface, 컨테이너 네트워크 인터페이스)를 사용하므로 파드(Pod) IP가 VPC 실제 서브넷 IP와 동일합니다. 온프레미스 망(예: 192.168.x.x)과 클라우드 대역이 겹치지 않는지 교차 검증해야 합니다.

strongSwan Windows 공식 접속 가이드 (내장 Agile VPN 활용)

최신 strongSwan 공식 문서(Official Documentation)에 따르면, Windows(윈도우) 환경에서는 별도의 외부 클라이언트 소프트웨어를 다운로드하거나 설치할 필요가 없습니다. 과거 실험적으로 제공되던 UWP(Universal Windows Platform, 범용 윈도우 플랫폼) 앱이나 MSI 패키지는 현재 공식 지원이 중단되었거나 제한적이므로, 운영체제에 기본 내장된 Agile VPN(애자일 VPN, 마이크로소프트의 내장 네트워크 모듈)을 활용하여 IKEv2(Internet Key Exchange version 2, 인터넷 키 교환 버전 2) 터널을 구성하는 것이 가장 무결성 높고 공식적인 글로벌 표준(Global Standard)입니다.

1

루트 인증서(Root CA) 수동 설치 (필수 보안 검증)

클라이언트 PC가 게이트웨이를 신뢰할 수 있도록 인프라 관리자가 제공한 인증서를 OS(Operating System, 운영체제) 깊숙이 등록해야 합니다.

  1. 제공받은 ca-cert.pem 또는 ca-cert.crt 파일을 더블 클릭하여 '인증서 설치(Install Certificate)' 마법사를 실행합니다.
  2. 저장소 위치를 반드시 '로컬 컴퓨터(Local Machine)'로 선택하고 다음(Next)을 클릭합니다. (관리자 권한 승인 필요)
  3. '모든 인증서를 다음 저장소에 저장'을 선택하고 찾아보기(Browse)를 누릅니다.
  4. '신뢰할 수 있는 루트 인증 기관(Trusted Root Certification Authorities)' 폴더를 정확히 지정한 후 완료합니다.
2

Windows 내장 VPN 프로필 생성 절차

별도 프로그램 없이 윈도우 네트워크 설정 메뉴를 통해 직접 접속 프로필을 생성합니다.

  1. 설정 메뉴 진입: 윈도우 작업 표시줄 우측 하단의 네트워크 아이콘 우클릭 ➔ '네트워크 및 인터넷 설정(Network & Internet settings)' ➔ 'VPN' 메뉴 선택 ➔ 'VPN 연결 추가(Add a VPN connection)' 클릭.
  2. 세부 정보 입력:
    • ▶ VPN 공급자(VPN provider): Windows(기본 제공) (Windows (built-in)) 선택
    • ▶ 연결 이름(Connection name): AWS EKS VPN 등 식별하기 쉬운 명칭 기입
    • ▶ 서버 이름 또는 주소(Server name or address): vpn.domain.com (또는 인프라 팀에서 부여한 공인 IP)
    • ▶ VPN 종류(VPN type): 반드시 IKEv2 선택
    • ▶ 로그인 정보 유형(Type of sign-in info): 서버 정책에 따라 인증서(Certificate) 또는 사용자 이름 및 암호(User name and password) 선택
  3. 모든 정보를 입력한 후 하단의 '저장(Save)' 버튼을 클릭합니다.
3

고급 어댑터 암호화 설정 (필수 튜닝 및 트러블슈팅)

윈도우 기본 설정은 하위 호환성을 위해 취약한 암호화를 허용하므로, 이를 차단하고 최고 수준의 ESP(Encapsulating Security Payload, 캡슐화 보안 페이로드)를 강제해야 서버 접속 거절(Policy Match Error)을 막을 수 있습니다.

  1. 제어판의 '네트워크 연결(Network Connections)' (또는 어댑터 옵션 변경) 창을 엽니다.
  2. 앞서 생성한 VPN 가상 어댑터(예: AWS EKS VPN)를 우클릭하고 '속성(Properties)'을 선택합니다.
  3. 보안(Security) 탭으로 이동합니다.
  4. 데이터 암호화(Data encryption): 드롭다운에서 최대 강도 암호화(서버에서 거절하면 연결 끊기) (Maximum strength encryption (disconnect if server declines))로 변경합니다.
  5. 인증(Authentication): EAP(확장명 인증 프로토콜) 사용(Use Extensible Authentication Protocol (EAP)) 라디오 버튼을 체크하고, 하위 메뉴에서 Microsoft: 보안 암호(EAP-MSCHAP v2)를 명시적으로 선택한 후 확인(OK)을 눌러 최종 저장합니다.

* 주석: 서드파티 앱을 별도로 다운로드하여 설치하지 않고 OS 커널 레벨의 내장 IKEv2 스택을 활용하면, 네트워크 오버헤드가 최소화되고 엔터프라이즈 환경의 MDM(Mobile Device Management, 모바일 기기 관리) 정책과 완벽하게 호환되는 아키텍처적 장점이 있습니다.

strongSwan 전용 클라이언트 앱 설치 및 설정 (선택/대안 방식)

OS(Operating System, 운영체제) 내장 기능을 사용하지 않거나 로컬 인증서 관리가 번거로울 경우, 연결 프로필과 인증서를 앱 내부에서 직관적으로 관리할 수 있는 strongSwan 공식 UWP(Universal Windows Platform, 범용 윈도우 플랫폼) 앱을 대안으로 사용할 수 있습니다.

1

공식 앱 다운로드 및 설치 (URL 우회)

웹 브라우저의 다운로드 링크가 차단되거나 연결되지 않는 문제를 완벽히 우회하기 위해, 윈도우 기본 스토어 앱을 통해 직접 설치합니다.

  • Windows 작업 표시줄의 돋보기 아이콘(검색창)에서 'Microsoft Store(마이크로소프트 스토어)'를 타이핑하여 실행합니다.
  • 스토어 최상단 검색창에 strongSwan VPN Client를 정확히 입력하여 검색합니다.
  • 'strongSwan Project'에서 배포한 공식 앱 아이콘(백조 모양)을 확인한 후 '다운로드(Get)' 또는 '설치(Install)' 버튼을 클릭하여 PC에 자동 설치를 진행합니다.
2

연결 프로필 메뉴 설정 및 인증서 매핑

설치된 앱을 실행하고, GUI(Graphical User Interface, 그래픽 사용자 인터페이스) 환경에서 다음 절차대로 접속 정보를 기입합니다.

  1. 앱 실행 후 메인 화면 우측 상단의 'Add connection(연결 추가)' 버튼을 클릭합니다.
  2. Server(서버): 인프라 관리자가 제공한 VPN 게이트웨이의 공인 IP 또는 도메인(예: vpn.domain.com)을 입력합니다.
  3. VPN Type(VPN 종류): 인증 정책에 따라 IKEv2 EAP (Username/Password) 또는 IKEv2 Certificate를 선택합니다.
  4. Username / Password(사용자 이름 및 암호): EAP 방식을 선택한 경우, 부여받은 접속 계정 정보를 기입합니다.
  5. CA certificate(인증 기관 인증서) 지정: 기본값인 'Select automatically(자동 선택)'를 해제하고, 'Select(선택)' 버튼을 눌러 로컬 PC에 저장해둔 ca-cert.pem 파일을 직접 선택하여 등록합니다.
  6. Profile name(프로필 이름): AWS EKS 실습망 등 식별용 이름을 기입하고 우측 상단의 'Save(저장)'를 클릭합니다.
  7. 메인 화면에 생성된 프로필 목록을 클릭하여 암호화 터널 접속을 개시합니다.

* 주석: 전용 앱을 사용할 경우 윈도우 OS의 복잡한 로컬 컴퓨터 인증서 저장소(Certificate Store)를 직접 건드릴 필요 없이, 앱 내부에서 인증서를 직접 매핑(Mapping, 연결)하여 보안 협상을 수행하므로 설정 오류 및 권한 문제가 대폭 줄어드는 장점이 있습니다.

클라우드 인프라 과금 (Billing) 및 유료 전환 주의 사항

본 아키텍처를 구현할 때 다음 서비스들에서 과금이 발생하거나 비용이 증폭될 수 있으므로 철저한 예산 통제가 필요합니다.

  • AWS Data Transfer Out (데이터 송신 비용): PC에서 VPN 터널을 통해 K8s 파드의 대용량 로그나 데이터를 다운로드할 경우, 외부 인터넷 아웃바운드와 동일하게 GB당 과금이 발생합니다.
  • NAT Gateway & Elastic IP 유지비: 독립된 VPC마다 외부 통신을 위한 NAT 게이트웨이를 구성하면 인스턴스 시간당 요금과 트래픽 처리 요금이 이중 부과됩니다. 실습 후 삭제 시 EIP(탄력적 공인 IP)까지 반드시 반납(Release)해야 페널티 과금을 막을 수 있습니다.
  • 다중 VPN 연결 및 Transit Gateway: VPC별로 독립적인 Site-to-Site VPN 터널을 맺거나 AWS Transit Gateway(전송 게이트웨이)로 허브 앤 스포크(Hub & Spoke)를 구성할 경우, 연결된 Attachment 개수 및 처리량에 비례하여 유료 과금이 발생합니다.

본 교재는 IP 직업훈련 전문가 과정 아키텍처 설계를 위해 작성되었습니다.
