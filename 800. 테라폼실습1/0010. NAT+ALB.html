8. 트러블슈팅: 초기 배포 시 의존성 출력값 에러 (Cold Start Dependency Error)

인프라를 처음 배포하기 위해 terragrunt run-all plan을 실행할 때 아래와 같이 "detected no outputs (출력값이 감지되지 않음)" 에러가 발생하며 실행이 중단되는 현상은 Terragrunt (테라그런트) 환경에서 발생하는 가장 대표적인 콜드 스타트(Cold Start) 문제입니다. 

time=2026-03-10T05:31:05+09:00 level=error msg=Module C:/.../04-alb has finished with an error: 
.../01-hub-vpc/terragrunt.hcl is a dependency of .../04-alb/terragrunt.hcl but detected no outputs. 
Either the target module has not been applied yet, or the module has no outputs.

원인 분석 및 해결 방안

하위 디렉터리(예: 04-alb)는 상위 디렉터리(예: 01-hub-vpc)가 생성한 vpc_id 값을 참조해야 합니다. 하지만 아직 실제 인프라가 배포(Apply)되지 않았기 때문에 State File (상태 파일)에 해당 결과값이 존재하지 않아 Terragrunt가 Plan (계획) 단계를 진행하지 못하는 것입니다.

이를 해결하기 위해서는 dependency 블록 내에 mock_outputs (모의 출력값) 속성을 명시하여, 실제 자원이 생성되기 전인 plan 단계에서는 테라폼이 임시 가짜 값을 사용하여 유효성 검사를 통과하도록 아키텍처 코드를 보완해야 합니다.

8.1 의존성 코드 보완 (Mock Outputs 적용)

에러가 발생한 하위 디렉터리들의 terragrunt.hcl 파일 내 dependency 블록을 아래와 같이 수정합니다. (사용자가 별도로 생성한 03-2-routtb 등의 디렉터리가 있다면 동일한 방식으로 적용해야 합니다.)

NatAlb/prod/03-tgw/terragrunt.hcl (수정)
include "root" { path = find_in_parent_folders() }

dependency "hub" { 
  config_path = "../01-hub-vpc" 
  mock_outputs = {
    vpc_id                  = "vpc-mock-12345"
    private_subnets         = ["subnet-mock-1", "subnet-mock-2"]
    public_subnets          = ["subnet-mock-pub1", "subnet-mock-pub2"]
    private_route_table_ids = ["rtb-mock-1", "rtb-mock-2"]
  }
  mock_outputs_allowed_terraform_commands = ["plan", "validate"]
}

dependency "service" { 
  config_path = "../02-service-vpc" 
  mock_outputs = {
    vpc_id                  = "vpc-mock-67890"
    private_subnets         = ["subnet-mock-3", "subnet-mock-4"]
    private_route_table_ids = ["rtb-mock-3", "rtb-mock-4"]
  }
  mock_outputs_allowed_terraform_commands = ["plan", "validate"]
}

terraform { source = "." }

inputs = {
  hub_vpc_id              = dependency.hub.outputs.vpc_id
  hub_subnet_ids          = dependency.hub.outputs.private_subnets
  service_vpc_id          = dependency.service.outputs.vpc_id
  service_subnet_ids      = dependency.service.outputs.private_subnets
  hub_route_table_ids     = dependency.hub.outputs.private_route_table_ids
  service_route_table_ids = dependency.service.outputs.private_route_table_ids
}

실전 팁: mock_outputs_allowed_terraform_commands

mock_outputs_allowed_terraform_commands = ["plan", "validate"] 옵션은 매우 중요합니다. 이 옵션이 없으면 실제 인프라를 배포하는 apply 단계에서도 가짜 값(Mock)을 AWS API (Application Programming Interface, 애플리케이션 프로그래밍 인터페이스)로 전송하여 치명적인 배포 실패를 유발할 수 있습니다. 위 옵션을 통해 오직 검증(Plan/Validate) 단계에서만 Mock 값을 사용하도록 안전장치를 걸어둡니다. 수정 완료 후 다시 terragrunt run-all plan을 실행하면 에러 없이 정상 통과됩니다.
