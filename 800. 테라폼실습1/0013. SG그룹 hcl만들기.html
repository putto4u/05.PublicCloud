테라그런트 기반 Hub and Spoke 네트워크 아키텍처 구축 v2

IP 직업훈련 전문가과정 - 고급 클라우드 인프라 자동화 실습

아키텍처 개요 및 목표

본 실습 예제는 대규모 엔터프라이즈 클라우드 환경에서 표준으로 채택되는 Hub and Spoke (중앙 허브 및 분기) 네트워크 아키텍처를 테라그런트(Terragrunt)를 통해 모듈화하여 구축합니다. 중앙의 Hub VPC (Virtual Private Cloud, 가상 사설 클라우드)가 인터넷 연결점 역할을 수행하며, Service VPC는 인터넷과 직접 연결되지 않고 TGW (Transit Gateway, 전송 게이트웨이)를 통해 Hub VPC를 거쳐 안전하게 외부와 통신하게 됩니다.

[필독] AWS (Amazon Web Services, 아마존 웹 서비스) 유료 과금 주의

본 실습에 포함된 NAT Gateway (Network Address Translation Gateway, 네트워크 주소 변환 게이트웨이)와 Transit Gateway는 생성되는 순간부터 시간당 높은 비용이 발생하는 유료 서비스입니다. 실습이 종료된 후에는 반드시 terragrunt run-all destroy 명령어를 통해 리소스를 삭제하여 불필요한 과금을 방지해야 합니다.

[v2 업데이트] SG (Security Group, 보안 그룹) 중앙 분리 아키텍처 적용

기존에 각 애플리케이션 리소스(ALB, EC2 등)에 종속되어 있던 보안 그룹을 04-sgrp 디렉터리로 완전히 분리하였습니다. 이는 IaC (Infrastructure as Code, 코드형 인프라) 환경에서 발생하기 쉬운 순환 참조 (Circular Dependency) 문제를 원천 차단하고, 전사적인 보안 정책을 중앙에서 통제하기 위한 엔터프라이즈 모범 사례(Best Practice)입니다.

핵심 구성 파일 상세 분석

1 루트 설정 파일

전체 프로젝트에 적용되는 AWS Provider (클라우드 공급자) 설정과 Terraform State (인프라 상태) 파일의 로컬 저장 방식을 동적으로 생성합니다.

labterra1/terragrunt.hcl HCL 
# 로컬 변수(Local Variables)를 통해 하위 디렉터리의 환경 설정 파일(env.hcl)을 읽어옵니다.
locals {
  env_vars = read_terragrunt_config(find_in_parent_folders("env.hcl"))
  aws_region = local.env_vars.locals.aws_region
}

# Provider (클라우드 공급자) 자동 생성 설정
# 이 블록은 각 하위 디렉터리에 'provider.tf' 파일을 동적으로 생성해 줍니다.
generate "provider" {
  path      = "provider.tf"
  if_exists = "overwrite_terragrunt"
  contents  = <<EOF
provider "aws" {
  region = "${local.aws_region}"
}
EOF
}

# PC (Personal Computer, 개인용 컴퓨터)의 현재 디렉터리를 활용한 로컬 상태 저장소(Local State Backend) 설정
# 실무에서는 S3와 DynamoDB를 조합하여 사용하지만, 본 실습에서는 로컬 환경에 저장합니다.
remote_state {
  backend = "local"
  generate = {
    path      = "backend.tf"
    if_exists = "overwrite_terragrunt"
  }
  config = {
    path = "${get_terragrunt_dir()}/terraform.tfstate"
  }
}

2 환경 변수 설정 파일

labterra1/prod/env.hcl HCL 
# 환경별(운영, 개발 등) 공통 변수를 정의합니다.
# [v2 아키텍처 개선] 하드코딩되어 있던 CIDR (Classless Inter-Domain Routing, 클래스 없는 도메인 간 라우팅), AZ (Availability Zone, 가용 영역), 라우팅 대역을 모두 중앙 집중화합니다.
# 이를 통해 인프라 코드의 재사용성을 높이고 Human Error (작업자 실수)를 방지합니다.
locals {
  environment              = "prod"        # 프로덕션(운영) 환경 지정
  aws_region               = "us-east-1"   # 리소스가 배포될 AWS 리전 (미국 동부 - 버지니아 북부)

  # Hub VPC (Virtual Private Cloud, 가상 사설 클라우드) 네트워크 변수
  hub_vpc_cidr             = "10.250.0.0/16"
  hub_public_subnets       = ["10.250.21.0/24", "10.250.22.0/24"]
  hub_private_subnets      = ["10.250.101.0/24", "10.250.102.0/24"]
  
  # Service VPC 네트워크 변수
  service_vpc_cidr         = "10.251.0.0/16"
  service_private_subnets  = ["10.251.101.0/24", "10.251.102.0/24"]
}

3 Hub VPC 구성 (01-hub-vpc)

외부 인터넷과의 통신을 전담하는 중앙 허브 네트워크입니다. Public/Private Subnet(서브넷, 부분망)을 모두 포함하며 NAT Gateway (유료 과금 주의)가 위치합니다.

main.tf Terraform 
variable "vpc_name" { type = string }
variable "vpc_cidr" { type = string }

# [v2 아키텍처 개선] 동적 할당을 위해 리스트 타입의 변수를 추가 선언합니다.
variable "azs" { type = list(string) }
variable "public_subnets" { type = list(string) }
variable "private_subnets" { type = list(string) }

module "hub_vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "5.0.0"

  name = var.vpc_name
  cidr = var.vpc_cidr

  # [v2 아키텍처 개선] 하드코딩된 배열을 제거하고 변수로 매핑합니다.
  azs             = var.azs
  public_subnets  = var.public_subnets
  private_subnets = var.private_subnets

  # NAT Gateway 활성화 (프라이빗 서브넷의 아웃바운드 인터넷 통신용)
  enable_nat_gateway = true
  single_nat_gateway = true # 실습 비용 절감을 위한 단일 NAT 구성
}

# TGW 연동을 위해 필요한 정보들을 출력값(Output)으로 내보냅니다.
output "vpc_id" { value = module.hub_vpc.vpc_id }
output "public_subnets" { value = module.hub_vpc.public_subnets }
output "private_subnets" { value = module.hub_vpc.private_subnets }
output "private_route_table_ids" { value = module.hub_vpc.private_route_table_ids }
output "public_route_table_ids" { value = module.hub_vpc.public_route_table_ids }  
output "nat_gateway_ids" { value = module.hub_vpc.natgw_ids }
terragrunt.hcl HCL 
# [v2 아키텍처 개선] 상위 폴더의 env.hcl을 읽어와서 local 변수로 캐싱(저장)합니다.
locals {
  env_vars = read_terragrunt_config(find_in_parent_folders("env.hcl"))
}

include "root" {
  path = find_in_parent_folders()
}

terraform {
  source = "." # 현재 디렉터리의 .tf 파일들을 실행
}

# [v2 아키텍처 개선] main.tf에 선언된 변수들에 env.hcl의 중앙값을 동적으로 주입합니다.
inputs = {
  vpc_name        = "HUB"
  vpc_cidr        = local.env_vars.locals.hub_vpc_cidr
  azs             = local.env_vars.locals.azs
  public_subnets  = local.env_vars.locals.hub_public_subnets
  private_subnets = local.env_vars.locals.hub_private_subnets
}

4 Service VPC 구성 (02-service-vpc)

실제 애플리케이션 서비스가 배포되는 내부망입니다. 보안 강화를 위해 Public Subnet과 NAT Gateway를 생성하지 않고 폐쇄적으로 구성됩니다.

main.tf Terraform 
variable "vpc_name" { type = string }
variable "vpc_cidr" { type = string }

# [v2 아키텍처 개선] Service VPC용 동적 변수 선언
variable "azs" { type = list(string) }
variable "private_subnets" { type = list(string) }

module "service_vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "5.0.0"

  name = var.vpc_name
  cidr = var.vpc_cidr

  # [v2 아키텍처 개선] 하드코딩 제거 및 변수화
  azs             = var.azs
  private_subnets = var.private_subnets

  enable_nat_gateway = false # NAT 없음 (Hub VPC 에 의존하여 인터넷 통신)
}

output "vpc_id" { value = module.service_vpc.vpc_id }
output "private_subnets" { value = module.service_vpc.private_subnets }
output "private_route_table_ids" { value = module.service_vpc.private_route_table_ids }
terragrunt.hcl HCL 
# [v2 아키텍처 개선] env.hcl 로드
locals {
  env_vars = read_terragrunt_config(find_in_parent_folders("env.hcl"))
}

include "root" {
  path = find_in_parent_folders()
}

terraform { source = "." }

# [v2 아키텍처 개선] env.hcl에서 Service VPC 대역폭 및 AZ 주입
inputs = {
  vpc_name        = "service-vpc"
  vpc_cidr        = local.env_vars.locals.service_vpc_cidr
  azs             = local.env_vars.locals.azs
  private_subnets = local.env_vars.locals.service_private_subnets
}

5 Transit Gateway (TGW) 구성 (03-tgw)

분리된 여러 VPC를 중앙에서 연결하고 라우팅을 통제하는 TGW (유료 과금 주의) 구성 파일입니다. Terragrunt의 강력한 기능인 `dependency` 블록을 사용하여 앞서 생성한 VPC들의 출력값을 안전하게 가져옵니다.

terragrunt.hcl (의존성 및 값 주입) HCL 
# [v2 아키텍처 개선] 라우팅을 위한 env.hcl 로드
locals {
  env_vars = read_terragrunt_config(find_in_parent_folders("env.hcl"))
}

include "root" { path = find_in_parent_folders() }

# 의존성 설정: Hub VPC 모듈이 먼저 실행되어야 합니다.
# mock_outputs는 'terragrunt plan' 시 실제 인프라가 배포되기 전 가짜 데이터를 제공하여 에러를 방지합니다.
dependency "hub" { 
  config_path = "../01-hub-vpc" 
  mock_outputs = {
    vpc_id                  = "vpc-mock-12345"
    private_subnets         = ["subnet-mock-1", "subnet-mock-2"]
    public_subnets          = ["subnet-mock-pub1", "subnet-mock-pub2"]
    private_route_table_ids = ["rtb-mock-1", "rtb-mock-2"]
    public_route_table_ids  = ["rtb-mock-pub1", "rtb-mock-pub2"]
    nat_gateway_ids         = ["nat-12345"]
  }
  mock_outputs_allowed_terraform_commands = ["init","plan", "validate"]
}

dependency "service" { 
  config_path = "../02-service-vpc" 
  mock_outputs = {
    vpc_id                  = "vpc-mock-67890"
    private_subnets         = ["subnet-mock-3", "subnet-mock-4"]
    private_route_table_ids = ["rtb-mock-3", "rtb-mock-4"]
  }
  mock_outputs_allowed_terraform_commands = ["init","plan", "validate"]
}

terraform { source = "." }

# 종속성 모듈에서 추출한 Output 값 및 env.hcl의 중앙값을 현재 모듈의 Input으로 매핑합니다.
inputs = {
  hub_vpc_id                 = dependency.hub.outputs.vpc_id
  hub_subnet_ids             = dependency.hub.outputs.private_subnets
  service_vpc_id             = dependency.service.outputs.vpc_id
  service_subnet_ids         = dependency.service.outputs.private_subnets
  hub_route_table_ids        = dependency.hub.outputs.private_route_table_ids
  hub_public_route_table_ids = dependency.hub.outputs.public_route_table_ids
  service_route_table_ids    = dependency.service.outputs.private_route_table_ids
  nat_gateway_ids            = dependency.hub.outputs.nat_gateway_ids
  
  # [v2 아키텍처 개선] 라우팅 대역폭 변수 주입
  all_ips                    = local.env_vars.locals.all_ips
  service_vpc_cidr           = local.env_vars.locals.service_vpc_cidr
}
main.tf (라우팅 핵심 논리) Terraform 
variable "hub_vpc_id" { type = string }
variable "hub_public_route_table_ids" { type = list(string) }
variable "hub_subnet_ids" { type = list(string) }
variable "service_vpc_id" { type = string }
variable "service_subnet_ids" { type = list(string) }
variable "hub_route_table_ids" { type = list(string) }
variable "service_route_table_ids" { type = list(string) }
variable "nat_gateway_ids" { type = list(string) }

# [v2 아키텍처 개선] 라우팅 타겟 CIDR 변수
variable "all_ips" { type = string }
variable "service_vpc_cidr" { type = string }

# TGW 생성 (유료 인프라)
resource "aws_ec2_transit_gateway" "main" {
  description = "Hub and Spoke TGW"
  tags = { Name = "main-tgw" }
}

# TGW에 Hub VPC 네트워크 어태치먼트(Attachment, 연결점) 생성
resource "aws_ec2_transit_gateway_vpc_attachment" "hub" {
  subnet_ids         = var.hub_subnet_ids
  transit_gateway_id = aws_ec2_transit_gateway.main.id
  vpc_id             = var.hub_vpc_id
  tags = { Name = "hub-attachment" }
}

# TGW에 Service VPC 네트워크 어태치먼트 생성
resource "aws_ec2_transit_gateway_vpc_attachment" "service" {
  subnet_ids         = var.service_subnet_ids
  transit_gateway_id = aws_ec2_transit_gateway.main.id
  vpc_id             = var.service_vpc_id
  tags = { Name = "service-attachment" }
}

# [핵심 라우팅 1] Service VPC에서 외부로 가는 모든 트래픽을 TGW로 강제 전송
resource "aws_route" "service_to_tgw" {
  count                  = length(var.service_route_table_ids)
  route_table_id         = var.service_route_table_ids[count.index]
  destination_cidr_block = var.all_ips # [v2 아키텍처 개선] 0.0.0.0/0 변수화
  transit_gateway_id     = aws_ec2_transit_gateway.main.id
  depends_on             = [aws_ec2_transit_gateway_vpc_attachment.service]
}

# [핵심 라우팅 2] Hub VPC의 Private Subnet에서 Service VPC 대역으로 가는 트래픽을 TGW로 전송
resource "aws_route" "hub_private_to_service" {
  count                  = length(var.hub_route_table_ids)
  route_table_id         = var.hub_route_table_ids[count.index]
  destination_cidr_block = var.service_vpc_cidr # [v2 아키텍처 개선] 10.251.0.0/16 변수화
  transit_gateway_id     = aws_ec2_transit_gateway.main.id
  depends_on             = [aws_ec2_transit_gateway_vpc_attachment.hub]
}

# [핵심 라우팅 3] Hub VPC의 Public Subnet에서 Service VPC 대역으로 가는 트래픽을 TGW로 전송
resource "aws_route" "hub_pub_to_service" {
  count                  = length(var.hub_public_route_table_ids)
  route_table_id         = var.hub_public_route_table_ids[count.index]
  destination_cidr_block = var.service_vpc_cidr # [v2 아키텍처 개선] 10.251.0.0/16 변수화
  transit_gateway_id     = aws_ec2_transit_gateway.main.id
  depends_on             = [aws_ec2_transit_gateway_vpc_attachment.hub]
}

# [핵심 라우팅 4] TGW 내부 라우팅 테이블 설정 - TGW가 수신한 외부 인터넷 향 트래픽을 Hub VPC 연결점으로 보냄
resource "aws_ec2_transit_gateway_route" "tgw_default_route" {
  destination_cidr_block         = var.all_ips # [v2 아키텍처 개선] 0.0.0.0/0 변수화
  transit_gateway_route_table_id = aws_ec2_transit_gateway.main.association_default_route_table_id
  transit_gateway_attachment_id  = aws_ec2_transit_gateway_vpc_attachment.hub.id
  depends_on                     = [aws_ec2_transit_gateway_vpc_attachment.hub]
}

# 검증을 위한 식별자 출력
output "tgw_final_info" {
  value = {
    tgw_id            = aws_ec2_transit_gateway.main.id
    tgw_route_table   = aws_ec2_transit_gateway.main.association_default_route_table_id
    hub_attachment_id = aws_ec2_transit_gateway_vpc_attachment.hub.id
  }
}

6 SG (Security Group, 보안 그룹) 전용 모듈 구성 (04-sgrp)

시스템 전체의 네트워크 방화벽 역할을 하는 SG (Security Group, 보안 그룹)를 한곳에서 관리합니다. Hub VPC와 Service VPC 각각에 필요한 정책을 일괄 배포하여 인프라의 복잡성을 낮추고 관리를 용이하게 합니다.

main.tf (통합 보안 정책) Terraform 
variable "hub_vpc_id" { type = string }

7 ALB (Application Load Balancer, 애플리케이션 로드 밸런서) 구성 (05-alb)

외부 클라이언트의 HTTP 트래픽을 안전하게 받아 내부 서비스로 전달하는 핵심 모듈입니다. 앞서 04-sgrp에서 중앙 생성한 보안 그룹 ID를 참조하여 연결합니다.
[AWS 과금 주의] ALB는 프로비저닝된 시간과 처리된 트래픽 양에 따라 지속적인 유료 요금이 발생합니다.

main.tf (ALB 배포) Terraform 
variable "vpc_id" { type = string }
variable "public_subnets" { type = list(string) }

# [v2 아키텍처 개선] 분리된 보안 그룹 모듈로부터 ID 리스트를 전달받습니다.
variable "security_group_ids" { type = list(string) }

module "alb" {
  source  = "terraform-aws-modules/alb/aws"
  version = "~> 9.0"

  name               = "hub-alb"
  load_balancer_type = "application"
  vpc_id             = var.vpc_id
  subnets            = var.public_subnets
  
  # 보안 그룹 직접 생성 코드가 제거되고, 전달받은 변수를 활용합니다.
  security_groups    = var.security_group_ids

  listeners = {
    http = {
      port     = 80
      protocol = "HTTP"
      forward = {
        target_group_key = "nginx_tg" 
      }
    }
  }
  
  target_groups = {
    nginx_tg = {
      name_prefix      = "ngnx-" 
      protocol         = "HTTP"
      port             = 80
      target_type      = "ip"
      create_attachment = false
    }
  }
}

output "target_group_arns" { 
  value = [module.alb.target_groups["nginx_tg"].arn] 
}

output "alb_dns_name" { 
  value = module.alb.dns_name 
}
