테라그런트 기반 멀티 환경(Prod/Dev) Transit Gateway 라우팅 아키텍처 구축

IP 직업훈련 전문가과정 - 고급 클라우드 인프라 자동화 실습

아키텍처 개요 및 목표

본 실습에서는 독립적인 두 개의 Spoke VPC (Virtual Private Cloud, 가상 사설 클라우드) 환경인 운영(Prod: 10.251.x.x) 환경과 개발(Dev: 10.252.x.x) 환경을 구성합니다. 각 VPC는 고가용성을 위해 Public Subnet(공용 서브넷)과 Private Subnet(사설 서브넷)을 모두 포함하며, 두 환경 간의 안전한 내부 네트워크 통신을 위해 TGW (Transit Gateway, 전송 게이트웨이)를 중앙 라우터로 배치하여 연결하는 엔터프라이즈급 네트워크 토폴로지를 구현합니다.

[필독] AWS (Amazon Web Services, 아마존 웹 서비스) 유료 과금 주의

본 실습에 포함된 각 VPC 내의 NAT Gateway (Network Address Translation Gateway, 네트워크 주소 변환 게이트웨이) 2개와 중앙의 Transit Gateway는 배포 즉시 시간당 높은 비용이 발생하는 유료 서비스입니다. 실습 완료 후 반드시 terragrunt run-all destroy 명령어를 실행하여 리소스를 모두 삭제하시기 바랍니다.

핵심 구성 파일 상세 분석

1 중앙 환경 변수 설정 (env.hcl)

Prod와 Dev 두 환경의 CIDR (Classless Inter-Domain Routing, 클래스 없는 도메인 간 라우팅) 대역을 명확히 분리하여 겹치지 않게(Non-overlapping) 설계합니다.

labterra2/multi-env/env.hcl HCL 
locals {
  aws_region = "ap-northeast-2" # 서울 리전
  azs        = ["ap-northeast-2a", "ap-northeast-2c"]
  
  # 운영(Prod) Spoke VPC 변수
  prod_vpc_cidr     = "10.251.0.0/16"
  prod_public_subs  = ["10.251.10.0/24", "10.251.20.0/24"]
  prod_private_subs = ["10.251.110.0/24", "10.251.120.0/24"]
  
  # 개발(Dev) Spoke VPC 변수
  dev_vpc_cidr      = "10.252.0.0/16"
  dev_public_subs   = ["10.252.10.0/24", "10.252.20.0/24"]
  dev_private_subs  = ["10.252.110.0/24", "10.252.120.0/24"]
}

2 운영 환경 VPC 구성 (01-prod-vpc)

운영 환경을 위한 VPC입니다. 자체적인 외부 인터넷 통신을 위해 NAT Gateway를 구성합니다.

main.tf Terraform 
variable "vpc_name" { type = string }
variable "vpc_cidr" { type = string }
variable "azs" { type = list(string) }
variable "public_subnets" { type = list(string) }
variable "private_subnets" { type = list(string) }

module "prod_vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "5.0.0"

  name = var.vpc_name
  cidr = var.vpc_cidr
  azs  = var.azs
  
  public_subnets  = var.public_subnets
  private_subnets = var.private_subnets

  # [AWS 과금 주의] 프라이빗 서브넷의 통신을 위한 NAT Gateway 생성
  enable_nat_gateway = true
  single_nat_gateway = true
}

# TGW 연동을 위한 식별자(ID) 출력
output "vpc_id" { value = module.prod_vpc.vpc_id }
output "private_subnets" { value = module.prod_vpc.private_subnets }
output "public_route_table_ids" { value = module.prod_vpc.public_route_table_ids }
output "private_route_table_ids" { value = module.prod_vpc.private_route_table_ids }
terragrunt.hcl HCL 
locals {
  env_vars = read_terragrunt_config(find_in_parent_folders("env.hcl"))
}

include "root" { path = find_in_parent_folders() }

terraform { source = "." }

inputs = {
  vpc_name        = "PROD-VPC"
  vpc_cidr        = local.env_vars.locals.prod_vpc_cidr
  azs             = local.env_vars.locals.azs
  public_subnets  = local.env_vars.locals.prod_public_subs
  private_subnets = local.env_vars.locals.prod_private_subs
}

4 Transit Gateway 다중 Spoke 라우팅 구성 (03-tgw)

생성된 Prod와 Dev VPC를 TGW (유료 과금)에 연결(Attachment)하고, 서로 간의 트래픽이 TGW를 통과하도록 각 VPC의 라우팅 테이블(Route Table)을 동적으로 업데이트합니다.

terragrunt.hcl (의존성 및 값 주입) HCL 
locals {
  env_vars = read_terragrunt_config(find_in_parent_folders("env.hcl"))
}

include "root" { path = find_in_parent_folders() }

# Prod 인프라 종속성 (완료 후 실행)
dependency "prod" { 
  config_path = "../01-prod-vpc" 
  mock_outputs = {
    vpc_id                  = "vpc-mock-prod"
    private_subnets         = ["subnet-prod-1", "subnet-prod-2"]
    private_route_table_ids = ["rtb-prod-priv1", "rtb-prod-priv2"]
    public_route_table_ids  = ["rtb-prod-pub1", "rtb-prod-pub2"]
  }
  mock_outputs_allowed_terraform_commands = ["init","plan","validate"]
}

# Dev 인프라 종속성 (완료 후 실행)
dependency "dev" { 
  config_path = "../02-dev-vpc" 
  mock_outputs = {
    vpc_id                  = "vpc-mock-dev"
    private_subnets         = ["subnet-dev-1", "subnet-dev-2"]
    private_route_table_ids = ["rtb-dev-priv1", "rtb-dev-priv2"]
    public_route_table_ids  = ["rtb-dev-pub1", "rtb-dev-pub2"]
  }
  mock_outputs_allowed_terraform_commands = ["init","plan","validate"]
}

terraform { source = "." }

inputs = {
  # Prod 자원 매핑
  prod_vpc_id           = dependency.prod.outputs.vpc_id
  prod_subnet_ids       = dependency.prod.outputs.private_subnets
  prod_route_table_ids  = concat(dependency.prod.outputs.private_route_table_ids, dependency.prod.outputs.public_route_table_ids)
  prod_cidr             = local.env_vars.locals.prod_vpc_cidr

  # Dev 자원 매핑
  dev_vpc_id            = dependency.dev.outputs.vpc_id
  dev_subnet_ids        = dependency.dev.outputs.private_subnets
  dev_route_table_ids   = concat(dependency.dev.outputs.private_route_table_ids, dependency.dev.outputs.public_route_table_ids)
  dev_cidr              = local.env_vars.locals.dev_vpc_cidr
}
main.tf (TGW 생성 및 상호 라우팅) Terraform 
variable "prod_vpc_id" { type = string }
variable "prod_subnet_ids" { type = list(string) }
variable "prod_route_table_ids" { type = list(string) }
variable "prod_cidr" { type = string }

variable "dev_vpc_id" { type = string }
variable "dev_subnet_ids" { type = list(string) }
variable "dev_route_table_ids" { type = list(string) }
variable "dev_cidr" { type = string }

# 1. 중앙 Transit Gateway 생성
resource "aws_ec2_transit_gateway" "multi_env_tgw" {
  description = "Prod and Dev Communication Router"
  tags = { Name = "multi-env-tgw" }
}

# 2. Prod VPC를 TGW에 연결(Attachment)
resource "aws_ec2_transit_gateway_vpc_attachment" "prod_attach" {
  subnet_ids         = var.prod_subnet_ids
  transit_gateway_id = aws_ec2_transit_gateway.multi_env_tgw.id
  vpc_id             = var.prod_vpc_id
  tags = { Name = "prod-tgw-attachment" }
}

# 3. Dev VPC를 TGW에 연결(Attachment)
resource "aws_ec2_transit_gateway_vpc_attachment" "dev_attach" {
  subnet_ids         = var.dev_subnet_ids
  transit_gateway_id = aws_ec2_transit_gateway.multi_env_tgw.id
  vpc_id             = var.dev_vpc_id
  tags = { Name = "dev-tgw-attachment" }
}

# 4. 라우팅: Prod VPC 내부에서 Dev 대역(10.252.0.0/16)으로 가는 트래픽을 TGW로 보냄
resource "aws_route" "prod_to_dev" {
  count                  = length(var.prod_route_table_ids)
  route_table_id         = var.prod_route_table_ids[count.index]
  destination_cidr_block = var.dev_cidr
  transit_gateway_id     = aws_ec2_transit_gateway.multi_env_tgw.id
  depends_on             = [aws_ec2_transit_gateway_vpc_attachment.prod_attach]
}

# 5. 라우팅: Dev VPC 내부에서 Prod 대역(10.251.0.0/16)으로 가는 트래픽을 TGW로 보냄
resource "aws_route" "dev_to_prod" {
  count                  = length(var.dev_route_table_ids)
  route_table_id         = var.dev_route_table_ids[count.index]
  destination_cidr_block = var.prod_cidr
  transit_gateway_id     = aws_ec2_transit_gateway.multi_env_tgw.id
  depends_on             = [aws_ec2_transit_gateway_vpc_attachment.dev_attach]
}
