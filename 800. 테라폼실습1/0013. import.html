VPN 전용 보안 그룹(SG) 구성 및 코드화

AWS Console 설정부터 Terragrunt(테라그란트) Import 및 IaC 통합까지

1. AWS 콘솔 기반 보안 그룹 생성

VPN(Virtual Private Network, 가상 사설망)을 통해 온프레미스(On-premise, 사내 구축형) 서버와 AWS VPC(Virtual Private Cloud, 가상 사설 클라우드)가 통신할 때, 인스턴스의 입구 역할을 하는 SG(Security Group, 보안 그룹) 설정이 필수적입니다.

생성 절차 (Step-by-Step)

  1. AWS 콘솔에서 VPC 서비스 접속
  2. 좌측 메뉴의 Security Groups(보안 그룹) 선택
  3. Create security group(보안 그룹 생성) 클릭
  4. Name tag(이름 태그): SG-VPN-Common 입력
  5. VPC: VPN이 연결된 해당 VPC 선택

핵심 인바운드 규칙(Inbound Rules) 설정

Type (유형) Protocol (프로토콜) Port (포트) Source (소스)
All ICMP - IPv4 ICMP All 10.0.0.0/16
Custom TCP TCP 22 (SSH) 10.0.0.0/16
Custom TCP TCP 3389 (RDP) 10.0.0.0/16

* Source(소스)의 CIDR(Classless Inter-Domain Routing, 무클래스 간 도메인 라우팅)은 실제 본사(On-premise) 대역으로 지정해야 보안이 유지됩니다.

과금 및 주의사항

보안 그룹 생성 자체는 무료입니다. 하지만 이와 연결된 Site-to-Site VPN 서비스는 연결 시간당 약 $0.05(리전별 상이)의 비용이 발생하며, 전송되는 데이터 양에 따른 데이터 전송 비용이 추가됩니다.

2. 기존 자원을 Terragrunt로 Import 하기

이미 콘솔에서 생성한 보안 그룹을 관리 코드(IaC, Infrastructure as Code) 상태로 가져오려면 Import(임포트, 불러오기) 과정이 필요합니다.

1단계: Terraform(테라폼) 리소스 선언

먼저 `main.tf` 파일에 콘솔 설정과 동일한 뼈대 코드를 작성합니다.

resource "aws_security_group" "vpn_sg" { # 콘솔에서 생성한 이름과 동일하게 설정 name = "SG-VPN-Common" description = "Security group for VPN traffic" vpc_id = "vpc-0abc123456789def0" # 실제 VPC ID }

2단계: Terragrunt Import 실행

터미널에서 실제 AWS 상의 보안 그룹 ID(예: sg-0123...)를 이용해 실행합니다.

# 실행 위치: terragrunt.hcl 파일이 있는 디렉토리 terragrunt import aws_security_group.vpn_sg sg-0123456789abcdef0

이 명령을 실행하면 AWS 실물 자원과 테라폼의 State(상태) 파일이 동기화됩니다. 이후 terragrunt plan을 통해 코드와 실제 자원의 일치 여부를 확인하세요.

3. IaC 코드 내 규칙 자동화

보안 그룹만 선언하는 것이 아니라, 인바운드 규칙(Inbound Rule, 들어오는 규칙)을 코드 내에서 동적으로 관리하는 완성형 예제입니다.

locals { on_prem_cidr = "10.0.0.0/16" # 본사 대역 변수화 } resource "aws_security_group" "vpn_sg" { name = "SG-VPN-Common" vpc_id = var.vpc_id # 인바운드 규칙: On-premise로부터의 ICMP 허용 ingress { from_port = -1 to_port = -1 protocol = "icmp" cidr_blocks = [local.on_prem_cidr] description = "Allow Ping from On-premise" } # 아웃바운드 규칙: 모든 트래픽 허용 (기본값) egress { from_port = 0 to_port = 0 protocol = "-1" cidr_blocks = ["0.0.0.0/0"] } tags = { Name = "SG-VPN-Common" Environment = "Production" } }

이렇게 작성된 코드는 terragrunt.hcl의 inputs 블록을 통해 리전별, 환경별로 VPC ID 등을 주입받아 재사용될 수 있습니다.
