Hub & Spoke 기반
AWS Client VPN 엔터프라이즈 실무 명세

본 문서는 AWS 관리 콘솔의 최신 업데이트 항목을 반영하여, 개별 사용자가 Mutual TLS (상호 전송 계층 보안) 인증을 통해 프라이빗 네트워크에 접속하는 전 과정을 상세히 기술합니다.

Phase 0. 인증 인프라 및 ACM(AWS Certificate Manager) 준비

0-1 Easy-RSA를 이용한 인증서(Certificate) 및 키(Key) 생성

Client VPN은 Server(서버)와 Client(클라이언트)가 서로를 검증하는 상호 인증 방식을 사용합니다. 이를 위해 사설 CA(인증 기관)를 구축합니다. 

# 1. 환경 초기화
git clone https://github.com/OpenVPN/easy-rsa.git
cd easy-rsa/easyrsa3
./easyrsa init-pki

# 2. CA(Certificate Authority, 인증 기관) 구축
./easyrsa build-ca nopass

# 3. 서버 인증서 및 개인키 생성 (AWS 콘솔 등록용)
./easyrsa build-server-full server nopass

# 4. 클라이언트 인증서 및 개인키 생성 (사용자 PC 배포용)
./easyrsa build-client-full user01.domain.tld nopass

0-2 ACM(AWS Certificate Manager, 인증서 관리자) 업로드

Console ➔ AWS Certificate Manager ➔ Import a certificate (인증서 가져오기)

SERVER CERT

  • Body: server.crt
  • Key: server.key
  • Chain: ca.crt

CLIENT CERT

  • Body: user01.domain.tld.crt
  • Key: user01.domain.tld.key
  • Chain: ca.crt

Phase 3. 활성화 및 정책 적용 (Post-Setup)

Step 1

Target network associations

생성된 엔드포인트를 Hub-VPC의 Private Subnet에 연결합니다. 이 과정이 완료(Available)되어야 ENI가 생성됩니다.

Step 2

Authorization rules

Destination network를 10.251.0.0/16 (Spoke 대역)으로 설정하여 사용자의 접근 권한을 부여합니다.

Step 3

Route table (Endpoint)

목적지 10.251.0.0/16으로 가기 위한 타겟 서브넷 인터페이스를 경로 정보에 등록합니다.

🚨 구축 핵심 요약 (Cheat Sheet)

  • 1. 인증서 교차 검증: ACM 업로드 시 Chain(ca.crt)을 빠뜨리면 접속 오류가 발생합니다.
  • 2. 스플릿 터널: 사용자 로컬 인터넷 속도 보장과 AWS 과금 방지를 위한 필수 설정입니다.
  • 3. TGW 라우팅: Spoke VPC의 Route Table에서도 172.16.x.x 대역을 TGW로 돌려주는 반환 경로가 반드시 필요합니다.
