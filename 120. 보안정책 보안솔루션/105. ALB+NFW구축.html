NFW 기반 Hub-and-Spoke 인라인 아키텍처 라우팅 가이드

NFW(Network Firewall, 네트워크 방화벽) 환경에서의 서브넷별 경로 제어 및 아키텍처 설계 전문가 과정

아키텍처 개요 (Architecture Overview)

본 가이드는 NFW(Network Firewall, 네트워크 방화벽)를 Hub VPC(중심 가상 사설 클라우드)에 배치하여, Spoke VPC(연결 가상 사설 클라우드)와 외부 인터넷 간의 모든 트래픽을 인라인으로 검사하는 구조를 다룹니다. 특히 NAT(Network Address Translation, 네트워크 주소 변환), ALB(Application Load Balancer, 애플리케이션 로드 밸런서), NFW가 각각 Hub-Public-1, 2, 3 서브넷으로 분리된 환경을 기준으로 합니다.

1. 아웃바운드 경로 (Outbound Path): 내부 서버 ➔ 인터넷

내부 서버가 업데이트 패치 등을 받기 위해 밖으로 나가는 트래픽 흐름입니다. 패킷이 방화벽을 반드시 거치도록 설계되었습니다.

순서 이동 구간 (패킷 흐름) 라우팅 테이블(RT) 작성 장소 목적지 (Destination) 대상 (Target)
1 Spoke 서버 ➔ TGW Spoke 프라이빗 서브넷 RT 0.0.0.0/0 TGW-ID
2 TGW ➔ Hub 프라이빗 TGW 전용 프라이빗 서브넷 RT
(Hub 내 TGW 접점)		 0.0.0.0/0 NAT-Gateway-ID
3 NAT ➔ NFW Hub-Public-1 (NAT 서브넷) RT 0.0.0.0/0 NFW-VPCE-ID
4 NFW ➔ IGW Hub-Public-3 (NFW 서브넷) RT 0.0.0.0/0 IGW-ID

2. 인바운드 경로 (Inbound Path): 외부 유저 ➔ 내부 서버

외부 고객이 서비스를 이용하기 위해 ALB로 접속할 때의 흐름입니다. 엣지 라우팅을 통해 유입 시점부터 검사를 수행합니다.

순서 이동 구간 (패킷 흐름) 라우팅 테이블(RT) 작성 장소 목적지 (Destination) 대상 (Target)
1 IGW ➔ NFW IGW 엣지 라우팅 테이블
(Edge Association)		 Hub-Public-2 (ALB CIDR) NFW-VPCE-ID
2 NFW ➔ ALB Hub-Public-3 (NFW 서브넷) RT VPC 내부 대역 (전체) Local (자동)
3 ALB ➔ TGW Hub-Public-2 (ALB 서브넷) RT Spoke-VPC-CIDR TGW-ID
4 TGW ➔ Spoke 서버 TGW 라우팅 테이블 (TGW-RT) Spoke-VPC-CIDR Spoke-Attachment

3. 리턴 트래픽 (Return Path): ALB ➔ 외부 유저

ALB가 응답을 보낼 때 다시 NFW를 거치게 하여 대칭 라우팅(Symmetric Routing)을 완성하는 경로입니다.

순서 이동 구간 (패킷 흐름) 라우팅 테이블(RT) 작성 장소 목적지 (Destination) 대상 (Target)
1 ALB ➔ NFW Hub-Public-2 (ALB 서브넷) RT 0.0.0.0/0 NFW-VPCE-ID
2 NFW ➔ IGW Hub-Public-3 (NFW 서브넷) RT 0.0.0.0/0 IGW-ID

클라우드 과금 및 유료 서비스 주의사항 (Billing Warnings)

  • NFW 데이터 처리 비용 중복 발생 (유료): 모든 인바운드 및 아웃바운드 패킷이 NFW를 경유하므로, 동일 데이터에 대해 GB당 처리 비용이 양방향으로 각각 합산 청구됩니다.
  • TGW 및 NAT 비용 중첩 (유료): 아웃바운드 트래픽은 TGW, NAT, NFW 세 가지 관리형 서비스를 모두 통과하므로, 각 서비스의 데이터 처리 요금이 누적되어 발생합니다.
  • VPCE 유지 요금 (유료): NFW 엔드포인트는 가용 영역(AZ)별로 생성되며, 가동 중인 엔드포인트당 시간당 고정 비용이 별도로 청구됩니다.

아키텍트 실전 팁 (Architect's Insight)

NFW 서브넷(Hub-Public-3)의 라우팅 테이블에서 Local(로컬) 규칙은 VPC 전체 CIDR을 포괄합니다. 따라서 NAT 대역이나 ALB 대역으로 향하는 응답 패킷에 대해 개별적인 정적 경로를 생성할 필요가 없습니다. 하지만 가용 영역(AZ) 간 통신 시 발생하는 Cross-AZ 비용을 방지하기 위해, NAT/ALB/NFW 엔드포인트를 동일한 AZ 내에 배치하는 AZ Alignment(가용 영역 정렬) 설계가 필수적입니다.

핵심 네트워크 용어 및 기술 주석

NFW (Network Firewall, 네트워크 방화벽): 트래픽 필터링 및 침입 방지 기능을 제공하는 관리형 네트워크 보안 서비스입니다.
IGW (Internet Gateway, 인터넷 게이트웨이): VPC 리소스와 공인 인터넷 간의 통신을 가능하게 하는 게이트웨이입니다.
ALB (Application Load Balancer, 애플리케이션 로드 밸런서): HTTP/HTTPS 트래픽을 처리하며 타겟으로 요청을 분산하는 로드 밸런서입니다.
VPCE (VPC Endpoint, VPC 엔드포인트): NFW와 같은 AWS 서비스를 VPC 내부 서브넷에 연결하기 위한 가상 네트워크 접점입니다.
Edge Association (엣지 연결): 라우팅 테이블을 서브넷이 아닌 IGW에 직접 연결하여 유입 트래픽을 제어하는 기술입니다.
Symmetric Routing (대칭 라우팅): 요청과 응답의 경로를 일치시켜 세션 기반 방화벽의 연결 끊김을 방지하는 설계 방식입니다.
TGW (Transit Gateway, 트랜짓 게이트웨이): 수천 개의 VPC와 온프레미스 망을 중앙에서 연결하는 클라우드 라우터입니다.
NAT (Network Address Translation, 네트워크 주소 변환): 사설 IP를 공인 IP로 변환하여 외부 인터넷 통신을 지원하는 기술입니다.
