베스천 서버(Bastion Host) 최적 배치 및 통합 라우팅 설계

제로 트러스트 아키텍처 기반의 프라이빗 서브넷 접근 통제 및 트래픽 포워딩 마스터 과정

아키텍처 개요 (Architecture Overview)

최신 클라우드 보안 표준에서는 베스천 서버를 퍼블릭 서브넷(Public Subnet, 공용 서브넷)에 배치하여 공인 IP(Public IP)를 통해 SSH(Secure Shell, 시큐어 셸) 포트를 직접 개방하는 방식을 엄격히 금지합니다.

본 아키텍처는 베스천 서버를 Hub VPC(Virtual Private Cloud, 가상 사설 클라우드)의 프라이빗 서브넷(Private Subnet, 사설 서브넷)에 안전하게 격리 배치하고, SSM(Systems Manager, 시스템스 매니저)의 세션 매니저(Session Manager)를 활용하여 외부 인터넷 노출 없이 내부 자원에 접근하는 엔터프라이즈 모범 사례(Best Practice)를 구현합니다.

1. 베스천 서버 출발 라우팅 (Outbound / Forward)

프라이빗 서브넷에 위치한 베스천 서버(Hub-Private-Bastion)가 목적지를 향해 세션을 시작할 때 참조하는 라우팅 테이블(RT) 규칙입니다.

통신 목적 이동 구간 (패킷 흐름) 라우팅 테이블(RT) 작성 장소 목적지 (Destination) 대상 (Target)
Spoke 서버 접속 베스천 ➔ TGW Hub-Private-Bastion RT Spoke VPC 전체 대역
(예: 10.10.0.0/16)		 TGW-ID
인터넷 / SSM 연결 베스천 ➔ NAT Hub-Private-Bastion RT 0.0.0.0/0 NAT-Gateway-ID
Hub 퍼블릭 자원 접근 베스천 ➔ Hub 퍼블릭 (NAT/ALB/NFW) Hub-Private-Bastion RT Hub VPC 전체 대역
(예: 10.0.0.0/16)		 Local (자동 적용)

2. 베스천 서버 복귀 라우팅 (Inbound / Return)

Spoke 망 내부의 서버나 Hub 망의 퍼블릭 자원들이 베스천 서버의 요청에 대해 응답 패킷을 되돌려 보낼 때 사용하는 경로입니다.

통신 목적 이동 구간 (패킷 흐름) 라우팅 테이블(RT) 작성 장소 목적지 (Destination) 대상 (Target)
Spoke 서버 응답 Spoke ➔ TGW Spoke 프라이빗 서브넷 RT Hub VPC 전체 대역
또는 0.0.0.0/0		 TGW-ID
TGW 전달 TGW ➔ Hub 프라이빗 TGW 라우팅 테이블 Hub VPC 전체 대역 Hub-VPC-Attachment
Hub 퍼블릭 자원 응답 Hub 퍼블릭 ➔ 베스천 Hub-Public RT (NAT/ALB/NFW) Hub VPC 전체 대역 Local (자동 적용)

아키텍처 설계 핵심 포인트 (Architect's Insight)

  • Local(로컬) 라우팅의 완결성: 베스천 서버가 Hub VPC 내의 퍼블릭 서브넷(NAT, ALB, NFW)과 통신할 때는 라우팅 테이블에 기본적으로 하드코딩되어 있는 VPC CIDR ➔ Local 규칙을 사용합니다. 이 규칙은 삭제 및 수정이 불가능하며, VPC 내부 통신을 위해 별도의 규칙을 수동으로 추가할 필요가 없습니다.
  • TGW(Transit Gateway) 경로의 명시적 선언: 베스천 서버가 타망(Spoke VPC)으로 진입하기 위해서는 베스천이 속한 서브넷의 라우팅 테이블에 반드시 Spoke 대역 ➔ TGW-ID 경로가 명시되어 있어야만 패킷이 라우터(TGW)로 정확히 전달됩니다.

클라우드 과금 및 유료 서비스 주의사항 (Billing Warnings)

  • TGW 데이터 처리 요금 (유료): 베스천 서버를 통해 Spoke VPC 내부의 서버로 덤프 파일이나 대용량 로그를 전송할 경우, 트래픽이 TGW를 횡단하므로 GB당 양방향 데이터 처리 요금이 부과됩니다.
  • NAT Gateway 데이터 요금 (유료): 프라이빗 서브넷의 베스천 서버가 AWS Systems Manager(SSM) 컨트롤 플레인과 통신하기 위해 인터넷으로 나갈 때 NAT 게이트웨이를 통과하며 GB당 데이터 처리 비용이 지속적으로 청구됩니다.
  • VPC Endpoint 대안 (비용 최적화): NAT 트래픽 비용을 근본적으로 절감하기 위해, SSM 통신 전용으로 Interface VPC Endpoint(인터페이스 VPC 엔드포인트, 시간당 프로비저닝 요금 부과)를 프라이빗 서브넷에 직접 생성하여 내부망으로만 통신하도록 구성하는 것이 엔터프라이즈의 표준입니다.

핵심 네트워크 용어 및 기술 주석

Bastion Host (베스천 호스트, 요새 호스트): 외부에서 내부 프라이빗 네트워크 인프라에 접근하기 위해 유일하게 열어두는 단일 진입점 역할의 강력한 보안 경유 서버입니다.
SSM (Systems Manager, 시스템스 매니저): AWS 리소스를 가시화하고 제어하는 관리 서비스로, 인바운드 포트(예: SSH 22번)를 열지 않고도 안전한 터미널 접속을 제공하는 세션 매니저를 포함합니다.
TGW (Transit Gateway, 트랜짓 게이트웨이): 다수의 VPC와 온프레미스(사내 구축형) 네트워크를 단일 지점에서 중앙 집중식으로 연결하는 클라우드 라우터입니다.
SSH (Secure Shell, 시큐어 셸): 네트워크 상의 다른 컴퓨터에 암호화된 방식으로 원격 로그인하여 명령을 실행하기 위한 프로토콜입니다.
NAT (Network Address Translation, 네트워크 주소 변환): 프라이빗 망의 리소스가 외부 공인 망(인터넷)과 통신할 수 있도록 사설 IP를 공인 IP로 변환하는 서비스입니다.
Interface VPC Endpoint (인터페이스 VPC 엔드포인트): AWS 서비스로 향하는 트래픽이 인터넷 구간을 타지 않고 AWS 프라이빗 네트워크를 통해 직접 연결되도록 지원하는 ENI 기반 접점입니다.
