        <h2>1. 자격 증명 및 접근 관리 (Identity & Access)</h2>
        <div class="summary-text">
            모든 보안의 시작점입니다. <span class="term">IAM (Identity and Access Management, 자격 증명 및 접근 관리)</span>을 통해 '누가(Identity)' '무엇에(Resource)' '어떻게(Action)' 접근할 수 있는지 세밀하게 통제합니다.
        </div>

        <h3>AWS IAM (Identity and Access Management)</h3>
        <div class="billing-box">
            <span class="billing-icon">💰</span>
            <div>
                <strong>과금 안내:</strong> <strong>무료</strong>입니다. 사용자, 그룹, 역할, 정책을 무제한으로 생성하고 관리할 수 있으며 추가 비용이 발생하지 않습니다.
            </div>
        </div>
        <p>
            IAM은 AWS 리소스에 대한 접근을 안전하게 제어하는 글로벌 서비스입니다. 최소 권한의 원칙(Principle of Least Privilege)을 적용하여, 사용자가 업무를 수행하는 데 필요한 <strong>정확히 그 권한만</strong> 부여해야 합니다.
        </p>
        <ul>
            <li><strong>User (사용자):</strong> 실제 사람이나 애플리케이션을 의미합니다. 영구적인 자격 증명(비밀번호 또는 액세스 키)을 가집니다.</li>
            <li><strong>Role (역할):</strong> 특정 권한을 가진 자격 증명이지만, 특정 사용자나 그룹에 영구적으로 연결되지 않습니다. EC2 인스턴스나 Lambda 함수가 임시 보안 자격 증명(Temporary Security Credentials)을 얻기 위해 'Assume(수임)'하는 데 주로 사용됩니다.</li>
            <li><strong>Policy (정책):</strong> 권한을 정의하는 <span class="term">JSON (JavaScript Object Notation, 자바스크립트 객체 표기법)</span> 형식의 문서입니다.</li>
        </ul>

        <div class="code-block">
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::my-company-confidential-bucket/*"
        }
    ]
}
        </div>

        <div class="tip-box">
            <span class="tip-title">아키텍트의 조언: IAM Role(역할)의 적극적인 활용</span>
            <p>
                실무에서 애플리케이션 코드 내부에 AWS Access Key를 하드코딩하는 것은 최악의 보안 프랙티스입니다. EC2 인스턴스에서 S3 버킷에 접근해야 한다면, 액세스 키를 발급받는 대신 <strong>EC2 인스턴스 프로파일(Instance Profile)에 IAM Role을 연결</strong>하십시오. AWS가 백그라운드에서 키를 자동으로 회전시키고 관리하므로 크레덴셜 유출 위험이 사라집니다.
            </p>
        </div>

        <h2>2. 탐지 및 모니터링 (Detective Controls)</h2>
        <div class="summary-text">
            환경 내에서 발생하는 모든 활동을 기록하고, 머신 러닝 알고리즘을 통해 비정상적인 위협을 자동으로 탐지하며, 전체적인 보안 상태를 대시보드에서 통합 관리합니다.
        </div>

        <h3>AWS CloudTrail (클라우드트레일, API 호출 로깅)</h3>
        <div class="billing-box">
            <span class="billing-icon">💰</span>
            <div>
                <strong>과금 안내:</strong> 최근 90일간의 계정 활동 내역(이벤트 기록)을 보는 것은 <strong>무료</strong>입니다. 하지만 로깅을 S3 버킷으로 영구적으로 내보내어 저장하거나(추적 생성), 여러 리전에 걸쳐 로깅을 활성화할 경우 데이터 이벤트 100,000건당 요금 및 S3 저장소 <strong>비용이 발생</strong>합니다.
            </div>
        </div>
        <p>
            누가, 언제, 어디서, 어떤 IP로 AWS 인프라를 변경했는지에 대한 '블랙박스' 역할을 합니다. AWS Management Console 로그인, CLI, SDK를 통한 모든 <span class="term">API (Application Programming Interface, 애플리케이션 프로그래밍 인터페이스)</span> 호출이 로깅됩니다. 보안 사고 발생 시 <span class="term">Forensics (포렌식, 디지털 증거 분석)</span>의 가장 기초적이고 핵심적인 자료가 됩니다.
        </p>

        <h3>Amazon GuardDuty (아마존 가드듀티, 지능형 위협 탐지)</h3>
        <div class="billing-box">
            <span class="billing-icon">💰</span>
            <div>
                <strong>과금 안내:</strong> <strong>유료 (30일 무료 평가판 제공)</strong>. CloudTrail 이벤트, VPC Flow Logs(VPC 흐름 로그), DNS 로그 등 분석된 데이터의 기가바이트(GB) 단위로 비용이 청구됩니다.
            </div>
        </div>
        <p>
            인프라에 대한 악의적 활동이나 무단 동작을 지속적으로 모니터링하는 위협 탐지 서비스입니다. 머신 러닝과 위협 인텔리전스(알려진 악성 IP 목록 등)를 결합하여 암호화폐 채굴, 비정상적인 리전에서의 인스턴스 배포, 탈취된 자격 증명 사용 등을 즉각적으로 탐지해 냅니다.
        </p>

        <h3>AWS Security Hub (AWS 시큐리티 허브, 통합 보안 상태 관리)</h3>
        <p>
            GuardDuty, Macie, Inspector 등 다양한 AWS 보안 서비스의 경고(Findings)를 한 곳으로 모아주는 중앙 집중식 대시보드입니다. <span class="term">CIS (Center for Internet Security, 인터넷 보안 센터)</span> AWS Foundations Benchmark와 같은 산업 표준 모범 사례를 기반으로 계정의 보안 상태를 자동으로 평가해 줍니다. <strong>(유료 서비스)</strong>
        </p>

        <hr class="my-12 border-t-2 border-slate-200">

        <h2>3. 인프라 및 네트워크 보호 (Infrastructure Protection)</h2>
        <div class="summary-text">
            네트워크 경계에서 발생하는 악의적인 트래픽(DDoS 공격, SQL 인젝션 등)을 필터링하고 내부 리소스를 안전하게 격리합니다.
        </div>

        <h3>AWS WAF (Web Application Firewall, 웹 애플리케이션 방화벽)</h3>
        <div class="billing-box">
            <span class="billing-icon">💰</span>
            <div>
                <strong>과금 안내:</strong> <strong>유료</strong>. 생성한 <span class="term">ACL (Access Control List, 접근 제어 목록)</span> 개수, 추가한 규칙(Rule)의 개수, 그리고 검사하는 웹 요청 100만 건당 비용이 청구됩니다.
            </div>
        </div>
        <p>
            웹 애플리케이션의 취약점을 노리는 공격으로부터 서비스를 보호합니다. <span class="term">SQL Injection (SQL 인젝션, 악의적 SQL 구문 삽입 공격)</span>, <span class="term">XSS (Cross-Site Scripting, 교차 사이트 스크립팅)</span> 등의 공격 패턴을 차단할 수 있습니다. Amazon CloudFront(CDN), Application Load Balancer(ALB), API Gateway에 배포하여 트래픽이 서버에 도달하기 전 최전선에서 방어합니다.
        </p>

        <h3>AWS Shield (AWS 쉴드, DDoS 보호)</h3>
        <div class="billing-box">
            <span class="billing-icon">💰</span>
            <div>
                <strong>과금 안내:</strong><br>
                - <strong>Shield Standard:</strong> <strong>무료</strong> (모든 AWS 고객에게 기본 제공). L3/L4 네트워크 계층의 일반적인 DDoS 공격을 방어합니다.<br>
                - <strong>Shield Advanced:</strong> <strong>유료 (월 $3,000의 기본 요금 + 데이터 전송 요금)</strong>. 대규모 봇 공격 방어, 24/7 DDoS 대응팀(DRT) 지원, WAF 무료 사용 등의 엔터프라이즈급 혜택을 제공합니다.
            </div>
        </div>
        <p>
            <span class="term">DDoS (Distributed Denial of Service, 분산 서비스 거부)</span> 공격으로부터 애플리케이션의 가용성을 보호하는 관리형 서비스입니다.
        </p>

        <hr class="my-12 border-t-2 border-slate-200">

        <h2>4. 데이터 보호 (Data Protection)</h2>
        <div class="summary-text">
            저장된 데이터(Data at Rest)와 전송 중인 데이터(Data in Transit)를 강력한 암호화 알고리즘으로 보호하고, 민감 정보의 유출을 방지합니다.
        </div>

        <h3>AWS KMS (Key Management Service, 키 관리 서비스)</h3>
        <div class="billing-box">
            <span class="billing-icon">💰</span>
            <div>
                <strong>과금 안내:</strong> <strong>유료</strong>. 생성한 고객 관리형 키(CMK) 1개당 월 $1가 청구되며, 암/복호화 API 요청 10,000건당 0.03달러의 비용이 발생합니다. (단, AWS 관리형 키는 무료입니다.)
            </div>
        </div>
        <p>
            데이터를 암호화하는 데 사용되는 <strong>암호화 키(Encryption Keys)</strong>를 쉽게 생성하고 제어할 수 있게 해주는 관리형 서비스입니다. <span class="term">FIPS 140-2 (Federal Information Processing Standard, 연방 정보 처리 표준)</span> 검증을 받은 하드웨어 보안 모듈(HSM)을 사용하여 키의 안전성을 보장합니다. EBS 볼륨, S3 버킷, RDS 데이터베이스 등 대부분의 AWS 서비스와 기본적으로 통합되어 클릭 한 번으로 손쉽게 암호화를 적용할 수 있습니다.
        </p>

        <h3>Amazon Macie (아마존 메이시, 민감 데이터 검색 및 보호)</h3>
        <div class="billing-box">
            <span class="billing-icon">💰</span>
            <div>
                <strong>과금 안내:</strong> <strong>유료</strong>. S3 버킷 평가 비용 및 민감한 데이터 검색을 위해 검사(스캔)한 데이터의 GB 용량 기준으로 비용이 청구됩니다. (초기 스캔 비용이 크게 나올 수 있으므로 타겟 버킷 지정에 주의해야 합니다.)
            </div>
        </div>
        <p>
            머신 러닝 및 패턴 일치를 활용하여 S3 버킷 내에 저장된 <span class="term">PII (Personally Identifiable Information, 개인 식별 정보)</span> (예: 주민등록번호, 신용카드 번호, 여권 번호 등)를 자동으로 검색, 식별 및 보호하는 데이터 보안 서비스입니다.
        </p>

        <hr class="my-12 border-t-2 border-slate-200">

        <h2>실무 아키텍처 사례 (Real-world Use Cases)</h2>

        <h3>실무 사례 1: 무중단 웹 애플리케이션 보안 아키텍처 (Edge Security)</h3>
        <p>
            글로벌 이커머스 서비스는 해커들의 지속적인 타겟이 됩니다. 실무에서는 서버 자원(EC2) 앞단에 다중 계층의 방어벽을 구축합니다.
        </p>
        <ol class="list-decimal pl-6 space-y-2 mb-6">
            <li><strong>AWS Shield Standard:</strong> 인프라 라우팅 단계에서 대규모 SYN Flood와 같은 L3/L4 DDoS 공격을 자동 완화합니다.</li>
            <li><strong>Amazon CloudFront (CDN):</strong> 엣지(Edge) 로케이션에서 트래픽을 분산 수용하여 오리진 서버의 부하를 막습니다.</li>
            <li><strong>AWS WAF:</strong> CloudFront에 WAF를 연결하여 국가 기반 <span class="term">Geo-blocking (지리적 차단)</span>, 악성 IP 블록, SQL 인젝션 패턴을 필터링합니다. 정상적인 요청만 다음 단계로 넘깁니다.</li>
            <li><strong>Application Load Balancer (ALB):</strong> SSL/TLS 인증서를 ACM(AWS Certificate Manager)으로 관리하여 <strong>전송 중 데이터(Data in Transit)</strong>를 암호화합니다.</li>
            <li><strong>Security Group & NACL:</strong> ALB에서 들어오는 트래픽만 EC2가 수신하도록(Inbound Rule) 설정하여 직접적인 외부 접근을 차단합니다.</li>
        </ol>

        <h3>실무 사례 2: 멀티 계정 환경의 중앙 집중식 보안 모니터링</h3>
        <p>
            엔터프라이즈 환경에서는 부서별/프로젝트별로 수십 개의 AWS 계정을 사용합니다. 보안팀은 이 모든 계정의 위협 상황을 한곳에서 모니터링해야 합니다.
        </p>
        <ol class="list-decimal pl-6 space-y-2">
            <li><strong>AWS Organizations:</strong> 모든 계정을 하나의 조직으로 묶습니다.</li>
            <li><strong>위임된 관리자(Delegated Administrator) 설정:</strong> '보안 전용 계정(Security Account)'을 생성하고, GuardDuty와 Security Hub의 관리 권한을 위임합니다.</li>
            <li><strong>자동 활성화:</strong> 조직 내에 새로운 멤버 계정이 생성될 때마다 GuardDuty와 Security Hub가 자동으로 활성화되도록 구성합니다.</li>
            <li><strong>중앙 집중화:</strong> 모든 멤버 계정의 위협 탐지 결과(Findings)가 '보안 전용 계정'의 Security Hub로 실시간 집계되어, 보안팀이 단일 대시보드에서 전사적 위협 대응을 수행합니다.</li>
        </ol>

        <hr class="my-12 border-t-2 border-slate-200">

        <h2>현업 엔지니어의 팁 및 자주 하는 실수 (Troubleshooting & Tips)</h2>

        <div class="warning-box">
            <span class="warning-title">치명적 실수 1: IAM 정책에서의 와일드카드(*) 남용</span>
            <p>
                개발 편의를 위해 자격 증명 정책에 <code>"Action": "s3:*"</code>, <code>"Resource": "*"</code> 와 같은 정책을 부여하는 경우가 많습니다. 이는 권한 탈취 시 전체 S3 데이터베이스의 삭제나 유출로 이어지는 가장 흔하고 치명적인 실수입니다. 반드시 <strong>최소 권한의 원칙</strong>을 준수하고, IAM Access Analyzer를 사용하여 과도한 권한을 식별 및 축소해야 합니다.
            </p>
        </div>

        <div class="warning-box">
            <span class="warning-title">치명적 실수 2: KMS 고객 관리형 키(CMK)의 섣부른 삭제</span>
            <p>
                비용 절감을 위해 사용하지 않는다고 판단한 KMS 키를 즉시 삭제해 버리는 경우가 있습니다. <strong>만약 해당 키로 암호화된 S3 데이터나 EBS 스냅샷이 남아있었다면, 키가 삭제되는 순간 그 데이터는 영원히 복구할 수 없는 쓰레기 데이터가 됩니다 (Crypto-shredding).</strong><br>
                AWS는 이러한 사고를 방지하기 위해 키 삭제 시 최소 7일에서 최대 30일의 <strong>대기 기간(Pending Deletion)</strong>을 강제합니다. 삭제 전 해당 키가 어디에 사용 중인지 CloudTrail을 통해 철저히 검증하십시오.
            </p>
        </div>

        <div class="tip-box">
            <span class="tip-title">실전 팁: CloudTrail 글로벌 로깅 비용 최적화</span>
            <p>
                모든 리전에서 CloudTrail 추적을 S3로 보내면 로깅 비용이 상승합니다. <span class="term">IAM</span>, <span class="term">STS (Security Token Service, 보안 토큰 서비스)</span>와 같은 '글로벌 서비스 이벤트'는 기본적으로 로깅에 포함됩니다. 여러 개의 추적(Trail)을 생성할 때, <strong>'글로벌 서비스 이벤트 로깅' 옵션을 단 하나의 추적에서만 활성화</strong>하여 중복 과금을 방지하십시오.
            </p>
        </div>
