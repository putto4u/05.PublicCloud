아니요, **Transit Gateway (TGW, 트랜짓 게이트웨이)**는 특정 VPC(Virtual Private Cloud, 가상 프라이빗 클라우드) 내부에 생성되는 리소스가 아닙니다.

TGW는 AWS 지역(Region) 단위로 생성되는 **네트워크 허브(Hub)** 역할을 하는 독립적인 엔터티입니다. 여러 VPC나 온프레미스 네트워크를 중앙에서 연결하기 위한 "가상 라우터"라고 이해하시면 됩니다.

---

## Transit Gateway의 구조적 특징

TGW를 사용할 때 가장 혼동하기 쉬운 부분은 **연결 방식**입니다. TGW 자체는 VPC 밖에 존재하지만, VPC와 통신하기 위해 각 VPC의 서브넷에 **Transit Gateway Attachment (TGW 어태치먼트, 트랜짓 게이트웨이 연결)**라는 접점을 생성합니다.

### 1. 독립적 위치

TGW는 특정 VPC의 하위 구성 요소가 아닙니다. AWS 계정 내의 리전 레벨에서 관리되며, 동일 리전 내의 수천 개 VPC를 하나의 TGW에 연결할 수 있습니다.

### 2. VPC 어태치먼트 (VPC Attachment)

VPC를 TGW에 연결하려면, 해당 VPC 내의 가용 영역(AZ)별로 최소 하나의 서브넷을 지정하여 어태치먼트를 생성해야 합니다. 이때 해당 서브넷 안에 **Elastic Network Interface (ENI, 탄력적 네트워크 인터페이스)**가 생성되어 TGW와 VPC 간의 통로 역할을 합니다.

### 3. 라우팅 테이블 (Routing Table)

* **VPC 라우팅 테이블:** 대상 네트워크(다른 VPC 등)로 가는 트래픽의 Next Hop(다음 홉)을 **TGW ID**로 지정해야 합니다.
* **TGW 라우팅 테이블:** TGW 자체에도 라우팅 테이블이 있어, 들어온 패킷을 어떤 어태치먼트(VPC나 VPN 등)로 보낼지 결정합니다.

## 과금 요금 (Pricing)

TGW는 사용 시 다음과 같은 비용이 발생하므로 설계 시 주의가 필요합니다.

* **AWS Transit Gateway Attachment 시간당 요금:** 각 VPC, VPN 또는 Direct Connect(다이렉트 커넥트)가 TGW에 연결된 시간만큼 과금됩니다 (약 $0.05/hour, 리전별 상이).
* **데이터 처리 요금 (Data Processing):** TGW를 통해 전송되는 데이터 1GB당 요금이 부과됩니다 (약 $0.02/GB, 리전별 상이).

> **주석 (Footnote):**
> * **Next Hop (다음 홉):** 패킷이 목적지로 가기 위해 거쳐야 하는 다음 경로의 장치나 게이트웨이를 의미합니다.
> * **Elastic Network Interface (ENI, 탄력적 네트워크 인터페이스):** VPC 내의 가상 네트워크 카드로, IP 주소를 할당받아 통신을 가능하게 합니다.
